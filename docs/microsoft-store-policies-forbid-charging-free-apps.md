# 微软商店政策将禁止粗略的开发者对免费应用程序收费

> 原文：<https://www.xda-developers.com/microsoft-store-policies-forbid-charging-free-apps/>

微软将从下周开始对开发者的微软商店政策进行一些修改。更新后的微软商店政策将禁止开发者对开源应用或互联网上其他地方通常免费提供的其他应用收费。目标是防止特定个人从这些应用程序中获利，这些应用程序有时在微软商店中不可用，可能会导致消费者认为这些应用程序只有付费版本。

一个通常免费但在微软商店需要付费的应用程序的例子是 Paint.NET，它实际上是由最初的开发者发布的。收费是作为支持开发的一种方式，而那些从开发者网站下载的人被要求(但不是必须的)捐款。这项新政策的措辞表明，这将不再可能做到这一点，但事实并非如此。微软的 Giorgio Sardio 在一条推特中澄清说，重点是保护用户免受误导性列表的影响，有些人可能会利用他人开发的开源软件通过误导客户来赚钱。微软已经承诺更新措辞以明确这一意图。

此外，微软希望限制开发者收取相对于其应用提供的功能“不合理的高”价格。微软没有具体说明什么是不合理的高价，但如果你今天在微软商店搜索像媒体播放器这样简单的东西，你会发现高达 16.99 美元的选项，这是基于快速搜索的。

另一个潜在的重大更新是应用安装程序通过微软商店外的直接链接分发(在。exe 或。msi 格式)将需要使用属于 Microsoft 可信根程序中包含的证书颁发机构的代码签名证书进行数字签名。这似乎是一种确保不能在微软商店直接获得的应用程序的安全性的方法。当[微软宣布将在商店](https://www.xda-developers.com/microsoft-store-apps-windows-11-updated-externally/)中提供外部应用时，这是一些用户可能会担心的问题之一，所以这一变化应该有助于缓解这些担忧。

另一方面，你可能会认为这破坏了微软商店的开放性，因为它限制了可以发布的应用程序。不过，这是一种平衡的行为，进行某种安全检查总是会在某种程度上对自由产生影响。

微软商店政策的最新修订版还有其他一些变化，包括禁止提供新闻和信息的应用程序传播虚假信息。其他变化影响稍小，主要是增加了更清晰的语言和更小的调整。所有这些变化都将于 7 月 16 日生效，比最初宣布的日期晚一个月。

**【UPDATE 7/6/2022 @ 3:05PM EST】**微软已经澄清，它不打算阻止开发者对自己的应用程序收费，而只是为了打击那些从开源或免费软件中获利的误导性列表。我们相应地更新了第二段。

## 更新:微软正在删除有争议的政策

鉴于对阻止开源应用在微软商店销售的政策的担忧，该公司将它从修订版中删除，直到语言得到澄清。虽然这个版本的文件的其余政策将生效，但在可预见的未来，开发者将继续能够销售他们的开源应用程序，而不用担心。

在将来的某个时候，Microsoft Store 政策的新更新中应该会出现一个语言更加准确的修订版政策。
