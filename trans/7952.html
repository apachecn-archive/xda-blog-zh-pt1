如果你使用谷歌浏览器，门户网站的零日漏洞意味着你应该立即更新

谷歌 Chrome 门户网站中的零日漏洞意味着你应该立即更新，因为它已被广泛利用。

如果你使用谷歌浏览器，你应该立即更新。作为 Chrome 94.0.4606.61 的一部分，零日安全漏洞得到了修复，该版本是作为 Windows、Mac 和 Linux 的紧急更新发布的。该漏洞已被指定为 CVE ID CVE-2021-37973，尽管该公司在大多数用户更新之前一直保留有关该漏洞的信息。更新正在稳定频道上推出，用户应该尽快更新。要检查您的 Chrome 版本，请单击右上角的溢出菜单，进入“更多”，然后单击“帮助”。它会显示你已经安装的 Chrome 版本，还会安装最新的版本。

在该公司发布的(通过)的安全建议中，它说“谷歌意识到 CVE-2021-37973 存在漏洞。”谷歌表示，这是在门户中的一次“释放后使用”攻击，这意味着门户中的一个漏洞允许已经释放的内存仍然被引用。这可能导致意外行为，并可能导致攻击者在理想条件下利用浏览器。门户是该公司在 2019 年开始测试的一项功能，用于页面之间的嵌入和无缝过渡。

今天修复的零日安全漏洞是在 9 月 21 日第一个谷歌 Chrome 94 稳定版发布的当天报告的。它是由来自 Google TAG 的 Clément Lecigne 在 Sergei Glazunov 和来自 Google Project Zero 的 Mark Brand 的帮助下发现的。Project Zero 是谷歌雇佣的一个安全部门，成立于 2014 年。该团队的主要任务是发现零日漏洞，也就是说，应该对其缓解感兴趣的一方未知(或未解决)的漏洞。“Heartbleed”就是这样一个零日漏洞，由两个独立的安全团队私下向 OpenSSL 报告。其中一个安全团队隶属于谷歌，并最终促成了“零计划”的诞生。

随着谷歌披露这一漏洞，2021 年谷歌 Chrome 中发现的零日漏洞总数达到 11 个。
