第一次 Windows 11 更新在这里，它带有重大修复

原文：https://www.xda-developers.com/first-windows-11-update-ton-of-fixes/#0001-01-01

Windows 11 已经推出一周了，但它已经开始了第一轮修复。如果你想知道为什么它更新得这么快，那么让我第一个欢迎你来到补丁星期二的精彩世界。每个月的第二个星期二，所有受支持的微软产品都会获得更新，这也包括 Windows 11。

距离微软向 Beta 通道发布 Windows 11 build 22000.194 已经过去了近一个月，随后它在一周后的 9 月 23 日到达了发布预览通道。换句话说，尽管 Windows 11 在一周前开始全面上市，但它实际上已经有一段时间没有进行适当的更新了。

这是 Windows 11 在生产中收到的第一个更新，测试版和发布预览版用户正在同时获得它。奇怪的是，修复列表相对较短，所以可能有一些不在变更日志中的变化。

更新是 KB5006674 ，它带来了 22000.258 的内部版本号。你可以在这里手动下载。亮点只是它更新了安全性，但修复列表更详细一些。

  • 解决了一些英特尔“黑仔”和“智能字节”网络软件与 Windows 11(原始版本)之间的已知兼容性问题。在某些情况下，安装了受影响软件的设备可能会丢弃用户数据报协议(UDP)数据包。这给基于 UDP 的协议带来了性能和其他问题。例如，在受影响的设备上，一些网站的加载速度可能比其他网站慢，这可能会导致某些分辨率下的视频流速度变慢。基于 UDP 的 VPN 解决方案也可能较慢。

微软表示，更新中没有任何已知问题，但这并不意味着 Windows 11 中没有任何已知问题。该公司在这里保留了一个单独的已知问题列表，在这三个问题中，只有一个在此次更新中得到修复。

展望未来， Windows 11 更新与 Windows 10 不会有任何不同。你仍然会在每周二看到另一个强制性的累积更新，如果你不主动安装，这些会自动安装。还会有可选的 C 周和 D 周更新，不会自动安装。
