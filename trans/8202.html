微软表示，它缓解了一次 2.4 Tbps 的 DDoS 攻击，这是有史以来最大的 DDoS 攻击之一

微软表示，它缓解了 Azure 网络上的 2.4 Tbps DDoS 攻击，这是有史以来最大的 DDoS 攻击之一。

微软表示，它在 8 月份缓解了对其 Azure 网络的 2.4 Tbps 分布式拒绝服务(DDoS)攻击。DDoS 攻击通常是一种用来关闭网站或系统的工具，经常向它们注入服务器无法处理的流量。这明显高于 2020 年对 Azure 网络的攻击，当时的峰值流量速度为 1 Tbps。这种 2.4 Tbps 的攻击比以前在 Azure 上检测到的任何网络容量事件都要高。

微软表示攻击持续时间略长于 10 分钟，具有非常短暂的爆发，每一次都在几秒钟内上升到万亿比特量。总共有三个峰值；第一个速率为 2.4 Tbps，第二个速率为 0.55 Tbps，第三个速率为 1.7 Tbps。由于 Azure 的 DDoS 保护平台(建立在分布式检测和缓解平台上)，客户完全不受影响。此外，微软表示，其 DDoS 保护平台可以承受数十万亿比特流量的冲击。这次攻击接近有史以来最大的 DDoS 攻击——谷歌在 2018 年受到 2.54 Tbps 流量的打击。

对 Azure 网络的攻击来自多个国家的大约 70，000 个来源，如马来西亚、越南、台湾、日本和中国，以及美国。通常，DDoS 攻击是由攻击者控制的僵尸网络发起的。僵尸网络通常由被入侵的机器组成，物联网设备是攻击者的主要目标。Mirai 未来组合僵尸网络是这方面的一个主要例子，它感染路由器和 IP 摄像头等物联网设备，以便稍后用于关闭网站。这一特定攻击的确切性质尚不清楚。

Azure 在 8 月初分享了 DDoS 攻击趋势，显示 2021 年上半年的攻击数量比 2020 年第四季度增长了 25%。然而，平均吞吐量(攻击规模)在同一时间段内从每秒 1tb 下降到 625 Mbps。
