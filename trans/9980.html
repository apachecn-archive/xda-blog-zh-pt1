Windows 11 build 22572 带有家庭安全和 Clipchamp 应用程序

原文：https://www.xda-developers.com/windows-11-build-22572-family-safety-clipchamp/#0001-01-01

像时钟一样，微软向 Dev 频道注册的 Insider 发布了 Windows 11 的每周版本。这一次，我们将获得 Windows 11 build 22572，它附带了一些开箱即用的新应用程序。新的应用程序是 Family 和 Clipchamp。

Family 是一款允许您管理 Microsoft 帐户的家庭设置的应用程序。如果您有一个家庭群组，您可以在这里看到您的家庭成员，以及管理儿童的屏幕时间，甚至可以实时定位家庭成员。这个应用只包含在 Windows 11 家庭版中，所以专业用户不需要担心额外的应用。

至于 Clipchamp，它是微软去年收购的一个视频编辑工具。当 Windows 11 启动时，新安装的程序中包含了一个 clipcamp 图标，它将作为 web 应用程序安装，因为 clipcamp 是一个基于 web 的视频编辑器。现在，它是开箱即用的，所以你不需要等待它下载和安装。不过，这仍然是一个网络应用。

虽然它今天还不可用，但微软也宣布将从下周初开始推出新的 Windows 搜索体验。这项新功能附带了一个叫做“搜索亮点”的东西，它根据日期显示琐事和热门搜索。你可以看到诸如是否是假日，那天发生了什么，等等。如果你已经使用工作或学校帐户登录，搜索将显示来自你的组织的相关信息，如人员和文件。这种体验也会降临到 Windows 10 上。

回到今天可用的内容，Windows 11 build 22572 做了一些更多的改进。打印队列已经用新的设计语言进行了更新，该语言与 Windows 11 的其余部分保持一致，并且它现在支持深色主题。Quick Assist 应用程序和 Windows Sandbox 现在也有一个流畅的设计图标，微软已经更新了通知中心图标，以指示何时启用勿扰模式。此外，您现在可以按住 Shift ，同时右键单击桌面上或文件资源管理器中的项目，以立即打开传统的“显示更多选项”上下文菜单。

其他较小的变化包括改变 Windows 终端的名称，现在只是在开始菜单中称为终端；Windows Media Player 被重命名为 Windows Media Player Legacy(因为 Windows 11 中有一个新的媒体播放器应用程序)；以及对设置应用程序的一些小改动。

除了这些变化之外，Windows 11 build 22572 还包括了对以前版本中已知问题的一些修复。完整列表如下:

Windows 11 内部版本 22572 中的修复

【常规】

  • 修复了企业版设备在设置体验(OOBE)时第一次尝试跳过网络添加屏幕的问题。
  • 解决了一个问题，该问题导致一些内部人员在以前的版本中尝试执行各种操作(包括登录、打开 outlook 和访问文件资源管理器中的网络共享)时看到带有 CRITICAL _ PROCESS _ DIED 错误的错误检查。
  • 缓解了之前飞行中导致音频断断续续和触摸板问题的潜在问题。
  • 解决了内部人员在最近的版本中尝试使用 Windows 混合现实时看到的 explorer.exe 崩溃问题。
  • 修复了滚动条在某些 Win32 应用程序上无法正常显示的问题。
  • 修正了另一个用户听不到启动声音的问题。
  • 当使用带有阿拉伯语或希伯来语显示语言的锁定屏幕幻灯片时，图像不应再向后显示。

【任务栏】

  • 我们修复了当你右击开始按钮或使用 WIN + X 时导致 explorer.exe 崩溃的问题
  • 缓解了在微软团队中共享一个窗口时，与任务栏中的视觉指示器相关的最近几次飞行中可能发生的 explorer.exe 挂起。
  • 调整了灵敏度，使你的鼠标快速滑过窗口图标到一个打开的弹出菜单或菜单时不再使其消失。
  • 进行了更改，以帮助解决重复打开和关闭通知中心时 Shell 体验主机中的内存泄漏问题。
  • 解决了如果在某些应用程序上打开通知中心，它可能不会自动关闭的情况。
  • 修复了当显示比例大于 100%时导致任务栏中的拖放意外显示不可用的潜在问题。

【平板电脑优化任务栏】

注意:这些修复将仅在您的设备上启用平板电脑优化任务栏的情况下显示，该任务栏开始向 Windows Insiders 推出，但尚未向所有人提供。

  • 所有设备类型上都恢复了触摸手势，因为我们解决了在非平板设备上破坏作为 Build 22557 的一部分宣布的几个新触摸手势的问题。
  • 提高了启动应用程序后任务栏自动折叠的可靠性。
  • 修复了几个与断开或重新连接键盘时任务栏状态变化相关的错误。
  • 解决了使用阿拉伯语或希伯来语显示语言时隐藏图标按钮和弹出按钮的定位问题。

【开始菜单】

  • 改进了“固定”标题名称在阿拉伯语显示语言中的呈现方式。

【文件浏览器】

  • 对上下文菜单调用性能做了更多的改进。
  • 现在使用快速访问中的过滤器选项应该可以处理图像了。
  • 修复了一个潜在的问题，当你最大化文件浏览器窗口时，文件浏览器中的标题栏在着色时不会占据它应该占据的全部区域。
  • 修复了一个问题，如果您在导航窗格中选择了一个文件夹，然后右键单击另一个文件夹，像在 Windows 终端中打开这样的选项会意外地打开第一个文件夹。
  • 当在快速访问的最近文件中选择一个文件时，命令栏中的“锁定到快速访问”现在可用。

【输入】

  • 缓解了用触摸或笔点击 RichTextBlocks 中的超链接时不起作用的问题。
  • 解决了一个问题，自定义鼠标指针可能会恢复到默认的 Windows 重启后。
  • 修正了一个问题，语音打字可能会显示“有一个连接问题”，当这不是错误的原因。
  • 解决了解锁电脑后语音输入可能无法启动的问题。
  • 修正了当 PC 上有多个账户时，登录屏幕上可能发生的触摸键盘崩溃。
  • 缓解了最近飞行中影响笔灵敏度的问题。
  • 修正了一个渲染问题，导致在最近的航班中，当试图在某些应用程序中使用墨水时，屏幕出现小的冻结。
  • 做了一些改进以帮助改进触摸板上缩放手势的检测。
  • 解决了导致 IME 工具栏选项在某些情况下对简体中文输入法无响应的问题。

【搜索】

  • 做了一些工作来解决高命中率搜索崩溃。

【设置】

  • 修复了试图访问个性化 > 任务栏 > 任务栏行为时导致挂起的问题。
  • 解决了当启用对比模式时，个性化>字体下的字体预览可能不可见的问题。
  • 改进了在应用>已安装应用的不同可用视图之间切换时的性能。
  • 通过 Windows Update >更新历史>卸载更新在继续之前应该显示一个确认。

【开窗】

  • 如果你在与屏幕顶部的对齐布局交互时按下 WIN + D，亚克力区域不会再出现在屏幕上。
  • 修复了一个问题，如果您使用捕捉助手捕捉已最小化的窗口，它可能不会填满整个可用空间。
  • 修正了一些在屏幕上使用三指手势时动画不流畅的问题。
  • 解决了在屏幕上看不到窗口的情况下尝试使用三指屏幕手势时可能发生的 explorer.exe 崩溃问题。
  • 解决了调用任务视图时可能影响动画和性能的潜在问题。
  • 缓解了导致您在某些情况下尝试在任务视图中重新排列桌面时意外看到 X 的潜在问题。
  • 调整了逻辑，以帮助进一步减少连接和断开显示器和坞站时听到的声音数量。
  • 缓解了最近的一个问题，即最小化应用程序的 windows 在取消对接和重新对接时无法在预期的显示器上启动。
  • 当最大化某些应用程序时，标题栏将不再显示在辅助显示器上。
  • 修正了一个问题，导致某些窗口显示一个意想不到的白色矩形从下面戳出来，横跨窗口的顶部。
  • 更新了新的全屏手爪用于边缘手势，以使用亚克力。

【旁白】

  • 修正了一个问题，阻止讲述人在扫描模式下正确阅读运行对话框。
  • 扫描模式下的讲述人现在可以在 Visual Studio 中正确读取错误列表中的错误信息。
  • 在设备设置(OOBE)中设置 PIN 时出现的错误现在将由讲述人读出。
  • 旁白自然声音的音高范围已被调整，以提供更多级别的控制。
  • 解决了一个潜在的问题，这是导致音高不变时，调整叙述者的速度在 0 至 5 和 15 至 20 范围内的叙述者设置。

【任务管理器】

  • 启用对比模式后，内容的可视性得到改善。
  • 修复了关键系统进程意外显示为后台进程的问题。
  • 解决了在使用黑暗模式时导致进程列表中的随机行意外写入黑色文本的问题。
  • 右键单击系统托盘中的任务管理器图标，现在应该实际显示上下文菜单，而不是一个空白框。
  • 按 Esc 应该不会再意外关闭任务管理器。

当然，在这个版本中也有一些已知的问题。以下是您需要注意的事项:

Windows 11 内部版本 22572 中的已知问题

【常规】

  • 运行 Windows 10 的用户尝试在 Dev 通道中直接升级到 build 22563 或更高版本可能会遇到安装失败，错误代码为 0x8007007f。要绕过，请加入测试版频道，在那里安装提供的 Windows 11 build，然后切换到 Dev 频道接收当前的 Dev 频道更新。此问题已被理解，并将在即将到来的版本中得到解决。

【平板电脑优化任务栏】

  • 在 2 合 1 设备上启动应用程序或点击扩展的任务栏外部后，任务栏并不总是自动折叠。
  • 操作系统的某些区域尚未跟踪 2 合 1 设备上扩展任务栏的高度，因此您可能会看到重叠的组件，例如与任务栏重叠的小工具。

【文件浏览器】

  • 在文件资源管理器的搜索框中输入搜索词时，可能无法打开显示的建议结果。
  • 我们正在修复显示 OneDrive 存储的弹出菜单中有关图标大小、视觉错误和文本剪辑的问题。

【微件】

  • 有时，当从提要中锁定时，锁定的小部件被放在顶部，而不是其他锁定的小部件的下面。如果发生这种情况，这将在 30 分钟内自动更正，将最近锁定的小工具移动到预期的默认位置。或者你可以退出你的插件板，然后立即重新登录，应该可以解决这个问题。
  • 在 widgets 面板中重新排列 widgets 后，一些用户会遇到固定部分中的 widgets 呈现不正确的问题。如果发生这种情况，退出你的插件板并立即重新登录应该可以解决问题。

【聚焦】

  • 在应用程序中配置焦点会话时，时钟应用程序尚未更新 Windows 焦点状态。这将在未来的应用程序更新中解决。

【旁白】

  • 讲述人自然声音听起来混乱，无法准确读出文本。这种情况发生在快速导航、打字或改变音调时。作为一个解决办法，你可以切换回老的声音，如微软大卫，马克或齐拉。

【直播字幕】

  • 全屏模式下的某些应用程序(如视频播放器)会阻止显示实时字幕。
  • 位于屏幕顶部附近并在运行实时字幕前关闭的某些应用程序将在位于顶部的实时字幕窗口后重新启动。当应用程序有焦点时，使用系统菜单(ALT +空格键)将应用程序的窗口进一步下移。
  • 当实时字幕位于顶部时，触摸无法到达最大化应用的最顶部(例如标题栏窗口管理按钮)。

【任务管理器】

  • 当任务管理器配置为黑暗模式时，一些工具提示不能准确显示首选视觉设置。

如果你注册了开发频道，你可以在 Windows Update 下的设置应用中检查更新，以获得 Windows 11 build 22572。你也可以查看我们的对 Windows Insider 程序的解释，了解更多关于它是如何工作的。
