Windows 11 上的 Android 应用程序现在可以在开发频道中使用

原文：https://www.xda-developers.com/android-apps-windows-11-dev-channel/#0001-01-01

微软正在向注册开发者频道的内部人士推出在 Windows 11 上安装和运行安卓应用的能力。微软最初在 6 月 24 日宣布了 Windows 11 的 Android 应用程序以及操作系统本身，但该功能错过了最初的 Windows 11 版本。

奇怪的是，微软开始在测试频道而不是开发频道与内部人员测试 Android 应用支持。据该公司称，这是因为它想优先测试 Windows 11 的稳定版本，这是目前在测试版渠道中可用的版本。这可能是因为 Android 应用支持预计将与完整功能更新分开提供，因此大多数用户将在 Windows 11 的初始版本上运行它们。

现在，Dev 频道的内部人士可以正式测试这项功能，尽管你以前也不是不能调整 Android 应用程序。正如我们在指南中解释的那样，任何人都可以为 Android 安装 Windows 子系统，并在 Windows 11 上运行 Android 应用程序，甚至包括非业内人士。但是，这确实可以省去您从不太可靠的第三方来源下载安装文件的麻烦。

开箱即用，你将被限制安装来自亚马逊应用商店的应用程序，其中只有 50 个是官方可用的。不仅如此，只有美国的用户才能在 Windows 11 上访问亚马逊 Appstore。然而，你可以相对容易地在 Windows 11 上下载 Android 应用。微软商店甚至有一个新的粉丝制作的名为 WSATools 的应用程序，让你只需打开 APK 文件就可以轻松安装安卓应用程序，无需通过命令行工具使用安卓调试桥(ADB)。这个应用程序使得安装 Android 应用程序的体验更加无缝。

当然，如果你是第一次进入这个领域，有些事情你需要知道。首先，系统要求有点高。你需要 8GB 内存，第八代英特尔酷睿 i3 处理器/AMD 锐龙 3000 系列/高通骁龙 8c 或更高版本，以及固态硬盘。这也有很好的理由，因为 Android 仿真需要相当多的功率，性能不会是最好的。

另一件要记住的事情是，谷歌服务不是正式可用的，因此，许多应用程序将无法工作。不过，如果你想修修补补，你可以将谷歌服务添加到 Android 的 Windows 子系统中，让一切正常运行。如果你在 Windows 11 的测试或开发渠道，你可以从微软商店下载 Android 的 Windows 子系统来开始。
