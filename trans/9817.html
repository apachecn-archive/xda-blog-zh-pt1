Windows 11 获得了针对平板电脑的改进任务栏，构建版本为 22563

原文：https://www.xda-developers.com/windows-11-build-22563/#0001-01-01

在上周发布了 Windows 11 build 22557 的大规模更新后，微软在开发频道为 Windows 内部人员推出了一个新版本。这一次，我们将获得 Windows 11 build 22563，由于过去的这个星期一是美国假期，所以比平时晚了一点。自从上次构建以来，我们只有一个星期的时间，所以这里没有太多的新内容，但它仍然包含一些好东西。对于使用 ARM64 PCs 的用户，现在将提供该版本，因此您也将首次获得上周的所有新功能。

Windows 11 build 22563 继续专注于平板电脑的改进。上周，我们获得了一系列新的手势，使触摸交互更加自然，现在，又有了新的任务栏。当使用平板电脑时，或者如果您将键盘从 2 合 1 PC 上断开，任务栏现在将折叠成简化的视图，仅显示核心系统图标，如电池、互联网和音量指示器。如果你想看到所有打开的应用程序，你可以从屏幕底部向上滑动，以显示带有应用程序图标的完整任务栏。这仅适用于平板电脑，因此台式机和笔记本电脑用户不会看到这一变化。

微软也对 widgets 面板做了一些调整，所以新的内容和你的个人 Widgets 现在融合在一起了。这样一来，一次可以看到更多的内容，如果你没有很多固定的窗口小部件，新闻将更容易访问。不过，您仍然可以将小部件固定在面板的顶部。

如果你使用 Edge，这个版本还附带了一些多任务消息。使用“吸附辅助”时，您现在可以将特定的边缘选项卡作为吸附建议来查看，因此您可以轻松地将选项卡组织到单独的窗口中，并一次查看所有选项卡。

对于表情符号粉丝来说，这个版本还增加了对表情符号 14.0 的支持，这是 Unicode 标准的最新版本，增加了 37 个新的独特表情符号，其中一些包括肤色变化。在这一点上，握手表情符号现在可以让你选择双手的肤色，当你包括肤色变化时，总共有 112 个新的表情符号。

此版本中的其他调整包括 Windows Update 通知的新组策略设置。同时，当从快速访问页面进行搜索时，文件资源管理器中的搜索栏现在包括 OneDrive、Downloads 文件夹和其他索引位置。此外，快速访问中未保存在本地的文件现在可以显示缩略图，并且支持 OneDrive 共享。

除了所有的新特性和改进之外，这个版本还包括一些错误修复，您可以在下面看到完整的细节。

Windows 11 内部版本 22563 中的修复

【常规】

  • 改进了某个人有一个非常大的临时文件夹的特定情况下的登录性能。
  • 修正了一些影响 explorer.exe 可靠性的问题。
  • 更新了使用键盘上的硬件键时的音量弹出按钮，以解决没有工作音频的情况。

【任务栏】

  • 当您使用任务视图切换桌面时，自动隐藏任务栏将保持展开状态。
  • 当拖动文件或应用程序到任务栏时，我们已经更新了鼠标旁边的消息，显示现在支持和不支持黑暗模式和对比主题。
  • 修复了使用法语(加拿大)显示语言的人发生的周期性 explorer.exe 崩溃，与电池图标工具提示有关。
  • 更新了任务栏中的静音图标，现在工具提示会给出快捷键的提示(WIN + Alt + K)。

【平板电脑优化任务栏】

注意:这些修复仅在您的设备上启用了平板电脑优化任务栏时才会显示。请参见上面关于平板电脑优化任务栏的详细信息，该任务栏开始向 Windows Insiders 推出，但尚未向所有人提供。

  • 任务栏的顶部边框线现在延伸到整个任务栏，而不是停在系统托盘区域之前。
  • 隐藏图标弹出按钮的样式现在与系统托盘按钮的设计保持一致。
  • 当切换输入法时，或者在添加和删除系统托盘图标时，任务栏不再闪烁。

【开始菜单】

  • 修正了当你从屏幕底部向上滑动时，开始菜单不能与新的触摸手势一致打开的问题。
  • 当新添加或删除推荐项目时，它们的图标将会以动画形式淡入淡出。
  • 电源菜单中的睡眠图标不再错误地显示为铃声。
  • 修复了导致电源按钮工具提示中的撇号不能正确显示的问题。
  • 在“开始”中打开文件夹时，文件夹中的应用程序名称不再短暂模糊。
  • 修复了一些人在尝试共享所有应用列表中的应用时遇到的崩溃。

【文件浏览器】

  • 修复了 OneDrive 图标有时不应显示在文件浏览器命令栏中的问题。
  • 移除了在黑暗模式下右击文件浏览器标题栏时出现在上下文菜单中的随机白点。

【输入】

  • 根据反馈对表情符号搜索结果进行了一些改进，包括通常在搜索结果中更倾向于与表情符号名称完全匹配的表情符号(例如，在搜索气球时首先搜索气球表情符号)，而不是针对这些关键词返回的各种其他表情符号。
  • 修正了一个导致内部人员在之前的飞行中看不到更新的韩国 IME 设计的问题。
  • 修正了几个 IME 候选窗口的着色问题。
  • 缓解了升级后尝试访问设置中的个性化>文本输入时发生的崩溃，如果您自定义了触摸键盘外观。
  • 解决了切换输入法时可能发生的 explorer.exe 崩溃问题。
  • 改进了首次启动输入切换器的性能。
  • 如果您在“颜色设置”中启用了“在开始和任务栏上显示强调色”,输入切换器的背景颜色将遵循该首选项。
  • 修复了触摸键盘设置选项中的键入链接名称，使其与打开的设置页面相匹配。
  • “设置”中的“系统”>“剪贴板”现在解释了为什么“跨设备同步”选项会变灰。
  • 向表情面板添加文本，以帮助解释不支持表情搜索结果的某些情况。

【搜索】

  • 我们做了更多的工作来减少最近搜索弹出按钮卡在屏幕上的机会。
  • 我们已经在最近的开发频道版本中解决了一个问题，该问题导致一些内部人员遇到了一个问题，即搜索可以打开，但只能在窗口中显示一个放大镜，而不能显示任何结果。如果您继续遇到问题，请在桌面环境>搜索下提交反馈，详细说明您到底看到了什么。

【设置】

  • 解决了一个问题，弹出对话框(例如，当添加一个可选功能，或当配置您的 IP 地址)在设置启动左对齐，而不是居中。
  • 解决了一个问题，RSAT: DNS 服务器工具将显示它是安装在可选功能，如果你试图安装它，但不会实际安装。
  • 修正了当你的电脑处于纵向模式时，显示设置可能不显示改变刷新率的选项的问题。
  • 当您的用户帐户控制设置为“总是通知”时，尝试通过“应用程序”>“已安装的应用程序”卸载应用程序时，设置不应再崩溃。
  • Windows Update >更新历史应再次显示此版本的功能更新历史。
  • 缓解了与 Windows Update 页面相关的可能导致设置挂起的问题。
  • 点击设置 > 系统 > 通知中的反馈链接，您将直接进入反馈中心的通知反馈部分。
  • 修正了一个问题，阻止你添加蓝牙或演员到快速设置弹出。
  • 快速设置上方的媒体控制现在应该在您打开媒体时显示，或者在您关闭媒体时隐藏，这样更可靠。
  • 修正了一个问题，当你使用你的硬件键时，弹出的音量窗口中的数字与快速设置中的音量不匹配。
  • 改进了登录界面上按钮上方的辅助弹出按钮的位置。
  • 修复了偶尔导致某些蓝牙设备在开机或从睡眠/休眠状态恢复后无法自动连接的问题。

【微件】

  • 任务栏左对齐，现在显示温度等信息。
  • 修正了一个小工具天气信息可能在多显示器设置中显示不同信息的问题。

【任务管理器】

  • 做了一些工作，以帮助解决一个问题，即当您从最小化状态启动任务管理器时，更新的任务管理器设计中的导航栏显示黑色小故障。

【开窗】

  • 修复了一个问题，该问题阻止您在最大化的应用程序(和其他场景)上使用最小化、最大化和关闭按钮，因为通知中心保持焦点。
  • 做了一些工作，以提高动画的流畅度时，调用任务视图使用 3 个手指在屏幕上的手势。
  • 改进了 ALT + Tab 的总体可靠性，有助于解决它可能被卡住的情况。
  • 解决了在最近的版本中使用 ALT + Tab、任务视图或 snap assist 时影响性能的问题。
  • 现在，当使用带有阿拉伯语或希伯来语显示语言的窗口时，快照组标题应该会正确显示。
  • 对将窗口移动到屏幕顶部时的快照布局调用体验做了一些改进。
  • 修复了当“捕捉助手”处于活动状态时拖动捕捉的窗口会包含被拖动窗口的缩略图的问题。
  • 当使用向下箭头在 snap assist 中浏览应用缩略图时，当您到达底部行时，它应该会循环到顶部。
  • 修复了在辅助显示器或纵向模式显示器上使用对齐布局时可能发生的两个崩溃。
  • 增加了屏幕左右两边的边距。
  • 在“开始”菜单打开时使用快捷键切换桌面不会再导致关闭“开始”后桌面意外切换回来。

【网络】

  • 添加了一些文字，以显示在快速设置的 VPN 部分点击断开按钮后，如果工作正在进行中，断开通信。
  • 修复了指定了 IPv4 路由的托管 VPN 配置文件可能会频繁断开连接的问题。

【旁白】

  • 在使用“权力”或“帐户”菜单后,“讲述人”将不再跳到“开始”中的搜索框。
  • 切换到“开始”菜单的“所有应用程序”部分后，讲述人现在将导航到并正确阅读您的应用程序信息。
  • Word online 中的讲述人现在可以在段落之间上下移动箭头时正确阅读新行。
  • 修复了在 Microsoft Word 中输入表格时导致讲述人停止阅读或无法移动光标的问题。
  • 改进了讲述人在登录屏幕上的辅助功能弹出按钮中阅读内容的方式。

【其他】

  • 修复了使用对比主题时导致某些应用程序登录窗口中的后退按钮不可见的问题。

当然，还有一些已知的问题需要警惕。这就是你对内部版本的期望。

Windows 11 内部版本 22563 中的已知问题

【常规】

  • 运行 Windows 10 的用户试图在 Dev 频道直接升级到 Build 22563，会在 Windows Update 上遇到“0%下载”的挂起。要绕过，请加入测试版频道，在那里安装提供的 Windows 11 build，然后切换到 Dev 频道接收 Build 22563 更新。此问题已被理解，并将在即将到来的版本中得到解决。
  • 在企业版上体验设备设置(OOBE)时，第一次尝试时会跳过网络添加屏幕。作为一种解决方法，当用户看到“命名您的计算机”选项时，请重新启动并重新启动 OOBE。现在，网络添加屏幕将按预期出现。
  • 许多用户现在听不到启动声音。

【任务栏】

  • 如果你右击这个版本中的开始图标或者按 WIN + X，explorer.exe 很可能会崩溃。如果使用此菜单启动任务管理器，请暂时使用 CTRL + Shift + Esc。

【平板电脑优化任务栏】

  • 启用了该功能的 Windows 内部人员将无法使用在 Build 22557 中宣布的从屏幕底部开始的新触摸手势，直到他们首先移除或折叠他们的键盘。拥有笔记本电脑等不能用作平板电脑的个人电脑的内部人员，以及启用了该功能的人员，将无法使用屏幕底部开始的触摸手势。这将在未来的飞行中得到解决。
  • 在 2 合 1 设备上启动应用程序或点击扩展的任务栏外部后，任务栏并不总是自动折叠。
  • 操作系统的某些区域尚未跟踪 2 合 1 设备上扩展任务栏的高度，因此您可能会看到重叠的组件，例如与任务栏重叠的小工具。
  • 2 合 1 设备上展开和折叠任务栏之间的过渡动画仍在调整中。

【文件浏览器】

  • 键入和单击建议时，文件资源管理器搜索框可能不起作用。
  • 我们正在修复显示 OneDrive 存储的弹出菜单中有关图标大小、视觉错误和文本剪辑的问题。

【微件】

  • 有时，当从提要中锁定时，锁定的小部件被放在顶部，而不是其他锁定的小部件的下面。如果发生这种情况，这将在 30 分钟内自动更正，将最近锁定的小工具移动到预期的默认位置。或者你可以退出你的插件板，然后立即重新登录，应该可以解决这个问题。
  • 在 widgets 面板中重新排列 widgets 后，一些用户会遇到固定部分中的 widgets 呈现不正确的问题。如果发生这种情况，退出你的插件板并立即重新登录应该可以解决问题。

【聚焦】

  • 任务栏图标和工具提示可能与焦点状态不匹配。
  • 在应用程序中配置焦点会话时，时钟应用程序尚未更新 Windows 焦点状态。这将在未来的应用程序更新中解决。

【直播字幕】

  • 全屏模式下的某些应用程序(如视频播放器)会阻止显示实时字幕。
  • 位于屏幕顶部附近并在运行实时字幕前关闭的某些应用程序将在位于顶部的实时字幕窗口后重新启动。当应用程序有焦点时，使用系统菜单(ALT +空格键)将应用程序的窗口进一步下移。
  • 当实时字幕位于顶部时，触摸无法到达最大化应用的最顶部(例如标题栏窗口管理按钮)。

【任务管理器】

  • 我们意识到，一些子流程上缺少效率模式图标。
  • 一些设置选项不会保留。
  • 我们正在解决有关图标大小和视觉缺陷的问题，在大多数情况下，解决方法是调整任务管理器的大小或从“设置”>“显示”>“显示分辨率”中降低分辨率设置。

微软最近分享了一些关于 2022 年你可以从 Windows Insider 计划中期待什么的见解，并提到你很快就会有一个窗口切换到 Beta 通道，以避免接收太多实验性功能的构建。由于我们还不知道这将在何时发生，这似乎表明所有这些改进都将成为 Windows 11 下一次功能更新的一部分，我们预计将在今年晚些时候进行。然而，我们必须等待确认。作为标准实践，Dev 频道中的特性永远不能保证能够稳定构建。
