Linux 的 Windows 子系统获得了一个新图标，并在其最新更新中更新了 Linux 内核

Windows Subsystem for Linux 的最新预发布版本带有一个全新的图标、一个更新的 Linux 内核等等。请继续阅读！

在尽可能本机地在 Windows 下运行 Linux 应用程序时，Windows Subsystem for Linux (WSL)兼容性层无需介绍。虽然微软仍然将 WSL 作为 Windows 10 和 Windows 11 的可选功能，但该公司已经逐渐开始将其与核心操作系统分离。模块化设计的第一步是将整个 Linux 子系统包装成一个应用程序，并通过微软商店分发。为了进一步巩固它，微软现在推出了一个新的预发布版本的 Windows 子系统，它带来了一个全新的图标，一个更新的 Linux 内核和一系列改进。

新版本将 Windows Subsystem for Linux app 的版本号提升至 0.50.2 。更新后你注意到的第一件事是新的企鹅标志，它取代了通用的应用程序存根图标。在引擎盖下，你可以找到 Linux 内核的版本【5.10.74.3】,其中有大量与 DirectX 图形内核子系统相关的修复。最后但同样重要的是，如果你在开始菜单中搜索“Windows Subsystem for Linux”并点击它，它现在将在你的 Linux 主目录而不是 Windows 实例的 System32 文件夹中启动你。
