Raspberry Pi 操作系统现已推出 64 位版本，提高了应用程序的兼容性

原文：https://www.xda-developers.com/raspberry-pi-os-64-bit/#0001-01-01

Raspberry Pi 系列单板计算机凭借其出色的多功能性和低成本，多年来取得了巨大成功。从为 DIY 电子项目供电到充当学习编程的廉价 PC，Raspberry Pi 系列可以做到这一切，现在另一个重大升级已经到来:默认操作系统的 64 位版本。

有一些不同的操作系统可供 Pi 板使用，包括 Android 的一些尝试，但是大多数人使用的默认操作系统是 T2 的树莓 Pi 操作系统。以前被称为 Raspbian，它是一个基于 Debian 的 Linux 桌面，专门为 Pi 家族构建。即使一些较新的 Pi 计算机有 64 位 ARM CPUs，除了测试版，Raspberry Pi 操作系统仍然只有 32 位。

Raspberry Pi 在的一篇博客文章(viaArs Technica)中表示，“我们一直在 32 位 Raspberry Pi 平台上构建我们的 Raspberry Pi 操作系统版本，旨在最大限度地提高设备之间的兼容性，避免客户混淆。[...]但我们逐渐意识到，有理由选择 64 位操作系统而不是 32 位操作系统。兼容性是一个关键问题:许多闭源应用程序只适用于 arm64，而开源应用程序并没有针对 armhf 端口进行充分优化。除此之外，A64 指令集还有一些固有的性能优势:今天，这些优势在基准测试中最为明显，但假设这些优势将在未来渗透到现实世界的应用性能中。”

切换到 64 位意味着更多的应用程序和服务可以访问更高层 Raspberry Pi 板上的更大内存量(如 8GB Raspberry Pi 4)，大多数人应该会看到性能的提升。目前唯一的问题是 64 位 Chromium 不支持 Widevine DRM ，因此需要 DRM 的网站(如迪士尼+或网飞)无法工作——你需要安装 32 位 Chromium。

您还需要一个 64 位的 Raspberry Pi 板来使用新的 64 位 Raspberry Pi 操作系统，该操作系统目前包括 Pi Zero 2、Pi 3 和 Pi 4。Pi 2、Pi 1 和最初的 Pi Zero 拥有不兼容的旧芯片组，这些设备将保留在 32 位操作系统上。

似乎现有 32 位 Raspberry Pi 操作系统的用户不会自动升级到 64 位版本，这可能是一个好主意。如果你想尝试新版本，去下载页面创建一个可启动的 USB 或 SD 卡。
