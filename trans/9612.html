Chrome 99 beta 测试了新的下载快捷方式，增加了手写 API 等等

原文：https://www.xda-developers.com/chrome-99-beta-features/#0001-01-01

谷歌刚刚在测试版频道发布了 Chrome 99，它带来了几个新的改进和增强，包括新的下载工作流程、手写识别 API、广告拦截器工作方式的改变等等。

在 Chrome 桌面上访问正在进行的下载或下载的文件有点麻烦。这是因为下载选项卡隐藏在溢出菜单中。但在 Chrome 99 测试版中，谷歌正在测试右上角工具栏中的简单下载快捷方式 (via 【安卓警察】 )。这种快捷方式是在 12 月份的 Chrome 99 Canary 中首次发现的。这与我们在微软 Edge 中拥有的非常相似。当您正在下载某个项目时，下载图标会变成蓝色，将来甚至可能会有一个环，一旦项目下载完毕，它会变回灰色。请注意，下载快捷方式在默认情况下是不可用的，所以我们不太确定它是否能及时为即将到来的稳定版本做好准备。

接下来，Chrome 99 beta 增加了对手写识别 API 的支持。谷歌从 Chrome 91 开始就一直在试验这个 API，现在它已经在最新的测试版中上线了。该 API 将使开发人员更容易构建跨不同平台无缝工作的笔记和绘图应用程序，而不必使用第三方集成。

一个用于 web 应用程序的 API，利用高级手写识别服务(例如操作系统上的服务)实时识别手写绘图(墨迹)中的文本。在这种情况下，手写绘图意味着用于描述人类手写过程的时间和位置信息。

此外，Chrome 99 beta 还引入了与广告拦截器相关的变化。从 Chrome 99 开始，谷歌不再允许新的广告拦截器扩展使用清单 V2。新的广告拦截器现在必须使用 Manifest V3，许多人认为这将使广告拦截器不那么有效，并损害创新。然而，现有的扩展将能够使用 V2 清单，直到明年。

最后，web 应用程序现在可以使用 color_scheme_dark 字段为黑暗模式指定不同的主题颜色和背景颜色，并强制浏览器以编程方式打开日期选择器。

Chrome 99 测试版现已在 Windows、Mac、Android 和 iOS 上发布，稳定版也将紧随其后。
