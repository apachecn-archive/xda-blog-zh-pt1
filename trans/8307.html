如何在任何没有 Windows Insider 的 Windows 11 PC 上运行 Android 应用程序

原文：https://www.xda-developers.com/how-to-run-android-apps-on-any-windows-11-pc/#0001-01-01

距离微软为 Windows 11 发布期待已久的 Android Windows 子系统仅仅一天。该公司还发布了大量文档，这些文档与亚马逊应用商店的集成以及使用原生 Android 子系统在 Windows 上开发 Android 应用程序有关。然而，与典型的前沿功能不同，Windows 11 的开发频道没有被官方允许访问 Android 应用的初始预览。现在，你不仅需要注册测试频道，而且你的电脑的地区应该设置为美国。除了这些“软”限制，你目前只能运行一小部分发布到亚马逊 Appstore 并得到微软和亚马逊批准的 Android 应用。

如果你在 Windows 11 的稳定或开发频道，并且你不想仅仅为了运行 Android 应用程序而改变更新频道，那么你会很高兴地知道，Android 的 Windows 子系统的当前版本可以很容易地安装和运行在不支持的版本以及 T2 的传统平台 T3 和 T4 上，而无需任何修改 T5。还可以使用 ADB 从下载安卓应用 ，这意味着你可以享受所有的附加功能，而不必局限于亚马逊应用商店的产品。

注意:在我们进入 Android 的 Windows 子系统的手动安装过程之前，请记住，当前版本远未达到一个稳定的里程碑。微软通过记住性能参数来设置限制，这就是为什么你可能会在不支持的平台上偶尔遇到故障(甚至崩溃)。在修补 Windows 11 之前，创建当前 Windows 11 安装的完整磁盘映像总是一个好主意，这样可以快速恢复到工作状态。

如何在任何 Windows 11 PC 上运行 Android 应用程序

步骤 1-获取 Android 的 Windows 子系统的安装包

就像用于 Linux 的 Windows 子系统(WSL) 一样，用于 Android 的 Windows 子系统也通过微软商店分发，用于运行 Windows 11 的设备。由于微软不提供从其商店获取应用程序包直接下载链接的官方方式，我们将选择第三方网站来通过这一障碍。

  1. 点击此处通过 @rgadguard 打开微软商店在线链接生成器网站。
  2. Android 应用的 Windows 子系统的产品 id 是 9P3395VX91NR ，所以你必须在第一个下拉菜单中选择“产品 ID”后在搜索框中输入相同的内容。或者，你可以直接把https://www.microsoft.com/en-us/p/windows-subsystem-for-android/9p3395vx91nr作为“URL(链接)”。接下来，从第二个下拉菜单中选择“慢速”频道。
  3. 从列表中找到最大的文件。它应该是一个 MSIX 包(扩展名为。msixbundle)大小略大于 1 GB。这是 Android 的 Windows 子系统的主安装包。点击文件名下载文件。
    • 在撰写本文时，Android 包的 Windows 子系统的版本号是 1.7.32815.0 。我们还镜像了这个特殊的版本，以便于下载。这个版本的 MSIX 包的 SHA-1 散列是0c4b3430bcd37be562f54322c79fb14f11ce1348。

步骤 2–安装 Android 版 Windows 子系统软件包

MSIX 包是一种包含多个 MSIX 包的打包格式——每个包可以支持一个特定的系统架构。这就是为什么你既可以在传统的 x64 电脑上安装这个软件包，也可以在兼容的 ARM64 设备上安装。

  1. 以管理员身份打开 Windows 终端并启动 PowerShell 实例。为此，右击开始按钮并选择 Windows 终端(管理)。默认情况下，它应该用 PowerShell 打开，但是如果没有打开，就用 PowerShell 调用一个新的标签页。
  2. 执行以下命令来安装您下载的 MSIX 软件包:
     Add-AppxPackage -Path <path_to_msixbundle_file>
  3. 如果一切顺利，那么这个包将被部署，你将在你的开始菜单中看到一个新的图标。
