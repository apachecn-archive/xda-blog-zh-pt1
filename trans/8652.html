以下是 Android 12L 将给你的智能手机带来的变化

原文：https://www.xda-developers.com/android-12l-smartphone-changes/#0001-01-01

谷歌在 10 月份首次发布了 Android 12L，与大多数 Android 更新不同，它几乎完全专注于大屏幕设备。平板电脑、可折叠手机和 Chromebooks 是最令人兴奋的变化将到来的地方，但这并不意味着普通智能手机完全失去了乐趣。现在，我们对哪些变化会出现在所有 Android 设备上有了更好的了解。

谷歌已经提到了一些新功能和 API 的变化，作为 Android 12L 更新的一部分，这些变化将出现在所有设备上，包括AOSP 启动器中的一个新的“快速壁纸拾取器”选项，附近调用 Pixel 和 Nest 设备，音量控制的回归用于铸造，AOSP 的动态主题支持，以及引导动画中的材质你的颜色。

其他一些变化现在已经被突出显示。从 Android 12L 开始，设备制造商将能够覆盖应用程序要求的首选方向，这将最有利于可折叠设备(例如，当打开仅支持纵向的应用程序时，Galaxy Fold 3 不会侧翻)，但也可能用于其他外形。

谷歌还在实现一个新的音频空间化 API ，旨在统一空间音频支持的所有不同实现(目前在索尼手机等设备中可以找到)，因此应用程序可以轻松确定当前设备是否支持空间音频。Android 12L 还支持全面的屏幕到头部跟踪，因此如果你有自带加速度计和陀螺仪的耳机，它们可以向 Android 报告你当前的头部位置，以获得更具沉浸感的音频。将支持三种头部跟踪模式:静态(无头部跟踪)、世界相对(无屏幕跟踪)和屏幕相对(全屏头部跟踪)。

一个更大的变化可能会用在最近推出的谷歌 Pixel 6 系列上。目前，VoiceInteractionSession API 可以获得关于设备上当前运行的应用程序的信息，它可以理解屏幕上的内容的一部分是通过截图，然后通过 OCR 分析截图。不过，Android 12L 中增加了两个新的回调- registerVisibleActivityCallback和unregisterVisibleActivityCallback。

这两个回调中的第一个允许助理应用程序注册一个回调，以便在可见活动发生变化时得到通知，而第二个只是取消注册回调。本质上，这将让助手应用程序跟踪用户通过应用程序的旅程，以提供更强大的建议和更集成的体验。斯珀认为，从 Pixel 4 开始为 Pixel 设备提供动力的“新”谷歌助手将广泛利用这一能力。

最后，Google 还在 Android 12L 中为外部存储访问提供了两种不同的 API。在 Android 10 中，谷歌引入了备受争议的范围存储框架。这是为了阻止大多数应用程序访问他们自己的文件夹和某些共享目录(如下载和图片文件夹)以外的数据。Google 随后同时弃用了两个广泛使用的外部存储 API:getExternalStorageDirectory()(用于访问主外部存储)和getExternalStoragePublicDirectory()(用于访问特定类型的存储目录)。谷歌现在取消了这两个 API。这是在 Android 11 允许对大多数外部存储进行读取访问的变化之后出现的。需要澄清的是，没有增强的权限，应用程序仍然不能在公共目录中读取或写入其他应用程序创建的内容(如 Chrome 保存在下载文件夹中的文件)。

在下一个测试版发布后，我们可能会了解更多关于 Android 12L 中所有 API 和跨设备的变化，目前计划在下个月发布。
