WSL 现已在微软 Windows 11 商店上架

原文：https://www.xda-developers.com/windows-subsystem-linux-microsoft-store-windows-11/#0001-01-01

微软在微软商店为 Windows 11 设备发布了Windows 子系统 Linux (WSL)的预览版。该公司正在测试一种新的方式来为 WSL 提供更新，这种方式将它们与核心 Windows 更新分开。到目前为止，WSL 一直是 Windows 10 和 11 的可选功能，你可以通过设置应用安装。然而，这也意味着对 WSL 平台的任何更新也必须通过对 Windows 本身的更新来交付，这意味着您必须等待一段时间才能获得新功能。

通过在微软商店提供 WSL，该公司现在让 Windows 11 用户更频繁地接收更新，从而获得新功能。这是微软近年来对收件箱 Windows 应用程序和功能采取的一种方法。最显著的例子是 Edge 浏览器，它在 2020 年变成了基于 Chromium 的浏览器，现在与谷歌 Chrome 或 Firefox 一样经常更新，这使它实际上可以与这些浏览器竞争。

就 WSL 而言，这个新的 Microsoft Store 版本已经包含了一些新特性，包括一个更新的 Linux 内核(版本 5.10.60.1)和捆绑的 WSLg。WSLg 是一个允许你在 Windows 中运行 GUI Linux 应用程序的组件，以前设置它可能有点复杂。现在，它只是应用程序的一部分，这应该会使事情变得容易得多。以下是此版本中的所有新增功能:

Microsoft Store 上 WSL 预览版的新功能

  • WSLg 现已捆绑为 WSL 应用的一部分！
  • 新wsl.exe --mount功能！
    • 添加--mount --vhd使安装 VHD 文件更容易。
    • 为wsl --mount实现文件系统检测。如果在使用wsl.exe --mount时没有指定--type，这一更改将实现文件系统类型检测。
    • 将--name特征添加到wsl --mount中。这一更改增加了在通过 WSL 挂载磁盘时对可选挂载点命名的支持。
  • 将 Linux 内核更新到 5.10.60.1
  • 添加了进度指示器帮助器函数，用于在转换过程中显示带有动画圆点的请稍候消息，以向用户显示 WSL 仍在运行。
  • 将wsl --install切换为不需要--distribution参数。这一变化将wsl --install切换为不需要--distribution参数，但保持支持以避免破坏现有脚本。
  • 增加了显示相关版本信息的wsl.exe --version命令

这个版本值得注意的一点是，它不支持原始的 WSL 1，所以任何没有更新为 WSL 2 的 Linux 发行版都需要通过 Settings 应用程序安装可选功能。幸运的是，您可以同时使用 WSL 的 Microsoft Store 版本和可选特性，所以这应该不是什么大问题。此外，微软表示，它计划解决用户对 WSL 2 的一些投诉，这些投诉使他们在某些情况下更喜欢 WSL 1，因此前者应该成为未来的最终版本。

这也可能有助于微软为 Android 建立基于 WSL 的 Windows 子系统。这项功能将允许用户在 Windows 11 上运行 Android 应用程序，如果它依赖于 WSL，那么这可能是在没有完全更新操作系统的情况下在 Windows 11 上启用 Android 应用程序的道路。Android 的 Windows 子系统过去也曾在微软商店出现过，但是我们仍然不知道什么时候能够测试它。

如果你感兴趣，只要你运行的是 Windows 11，你现在就可以从微软商店下载 WSL 预览版。
