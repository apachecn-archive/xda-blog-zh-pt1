第三方网络浏览器和应用商店正在进入微软商店

原文：https://www.xda-developers.com/third-party-web-browsers-and-app-stores-are-coming-to-the-microsoft-store/#0001-01-01

微软今天宣布微软商店政策的一些变化。虽然它们意义重大，但并不意外或令人惊讶。该公司宣布的两大变化是，它将向第三方店面开放微软商店，并开始允许第三方浏览器引擎在商店中使用。

首先是第三方店面进入微软商店的想法。这不仅仅是 Windows 11 的专利，新的微软商店也将登陆 Windows 10，但是 T2 的一些商店将会是新操作系统的专利。最明显的例子是第一个展示这个新政策的例子，亚马逊应用商店。

微软在 6 月份宣布 Windows 11 时宣布，亚马逊应用商店将包含在自己的商店中。它还没有到来，也不会在 Windows 11 发布之前到来。但它的特别之处在于它将 Android 应用程序带到了操作系统中，所以这仍然是 Windows 11 的一个功能。像 Epic Games 商店这样的东西，这是微软宣布的另一个店面，更有可能也会出现在 Windows 10 中，因为它不需要特殊服务。

然后是第三方浏览器引擎进入微软商店的想法，这更是一个令人惊讶的变化。当 Windows 10 首次推出时，该规则类似于苹果在其应用商店使用的规则。如果你想在商店里放一些带有任何浏览器窗口的东西，它必须使用平台引擎。当时，这是 EdgeHTML。没有人想开发基于微软 Edge 的浏览器，所以没有浏览器出现在微软商店。

当微软决定从谷歌 Chrome 使用的开源浏览器 Chromium 重建其 Edge 浏览器时，最初的计划是取消对 EdgeHTML 的要求，并让开发者在 EdgeHTML 和 Chromium 之间进行选择。然而，这已经转变为允许微软商店中的任何浏览器，这将从 Opera 和 Yandex 开始。

整个想法是，微软将采取微软商店中的任何应用程序。不管你是一个竞争的应用商店，一个竞争的浏览器，一个为 Android 开发的应用，还是其他什么。该公司希望这是你在 Windows 上获得应用程序的地方。

至于这些变化什么时候生效，应该是立竿见影。这是一个政策变化，所以它不像是一个新的微软商店应用程序，必须先经过内部人士的测试。你可以期待亚马逊应用商店、Epic 游戏商店、Opera 和 Yandex 浏览器等服务随时出现在商店中。

至于亚马逊 Appstore，微软仍然只是说 Android 应用程序支持将很快进入 Windows 11。
