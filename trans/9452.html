由于恶意软件的攻击，微软默认关闭 Excel 4.0 宏

原文：https://www.xda-developers.com/microsoft-excel-4-0-macros-disabled/#0001-01-01

Microsoft Excel 的第一个版本是在 35 年前发布的，从那时起，我们对计算机安全有了更多的了解。Excel 宏的第一次迭代可以调用并执行计算机上的其他程序，这在数字处理的早期很有帮助，但此时它在企业环境中主要只是恶意软件的攻击载体。经过多年的安全问题，默认情况下，Excel 不再运行旧的宏。

Excel 4.0 宏，也称为 XLM 宏，通常被用于在企业环境中分发恶意软件。在大多数情况下，发送到工作电子邮件的文档会要求收件人单击 Excel 中的“启用编辑”和“启用内容”按钮，以便恶意软件运行。微软已经允许管理员默认关闭 XLM 宏(甚至完全阻止它们)，但是 哔哔声电脑报道公司正在为每个人打开开关。

Excel 4.0 宏允许在 Excel 文档中实现复杂的自动化和工作流，但没有容器化，并且可以完全访问计算机的应用程序和 Windows 上的 Win32 API。Excel 5.0 包含了Visual Basic for Applications(VBA)作为编写宏的新选项(在 2020 年被 Stack Overflow 的调查列为最可怕的编程语言)，它有更多的限制来防止安全问题，但在最新版本的 Excel 中仍然支持原始宏，以避免破坏文档。

哔哔声电脑报告称，XLM 宏在 Excel 版本 16.0.14527.20000 及更新版本中被默认禁用，该版本于 10 月在当前频道推出，12 月在每月企业频道推出。半年度企业渠道(预览版)和半年度企业渠道将分别在 3 月和 7 月进行更改。

这并没有完全取消对旧宏的支持，只是显示了一个带有“启用内容”提示的安全警告。许多管理员已经启用了安全警告，但(非恶意软件)XLM 宏的使用似乎已经减少到了微软增加额外路障的程度。Excel 4.0 宏从未在 Excel 的 web 或移动版本中可用。
