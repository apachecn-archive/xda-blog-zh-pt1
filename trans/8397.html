WSATools 让在 Windows 11 上侧装 Android 应用变得更加容易

原文：https://www.xda-developers.com/wsatools-sideload-android-apps-windows-11/#0001-01-01

上周， Android 应用终于来到 Windows 11 供内部人员注册测试版频道。官方说法是，只有亚马逊应用商店的应用程序可用，而且数量有限。然而，我们已经向你展示了如何让 Android 的 Windows 子系统在非业内人士的 PC上运行，以及如何使用 ADB 下载应用。但是如果你不想在 Windows 11 上使用命令行安装 Android 应用，有一种更简单的方法可以用 WSATools 来实现。

WSATools 是由 Simone Franco 开发的一款工具，他在 Twitter 上分享了这款应用的外观(通过 T2、T3、T4、T5)。它可以在微软商店买到，它本质上是一个 GUI，专门用于复制 ADB 命令来安装 Android 应用程序。这很简单，但是已经运行得很好了。你可以打开应用程序，选择你想要安装的 APK 文件，或者直接从文件资源管理器中打开 APK 文件，因为 WSATools 也可以在 Windows 11 中处理 APK 文件关联。

因为它只是 ADB 命令的 GUI，所以您仍然需要设置一些重要的步骤。首先，你需要在你的 Windows 11 PC 上下载并设置 ADB ，然后你需要在 Windows 11 中将 ADB 添加到你的系统环境变量中。这使得您可以在系统范围内运行 ADB 命令，而不必在特定文件夹中打开命令行工具。你可以这样做:

  1. 打开开始菜单，输入“环境”，直到你看到一个名为的选项，改变系统环境变量。点击它。
  2. 在弹出的窗口中，点击环境变量。
  3. 在新窗口的下半部分，在系统变量下，选择路径变量，然后点击编辑。
  4. 点击浏览并导航至存储 ADB 平台工具的文件夹。
  5. 点击确定关闭所有窗口并确认您的更改。

一旦完成，你只需要在 Android 的 Windows 子系统中启用开发者模式，并通过打开一个 Android 应用程序来确保它正在运行。如果您将 WSA 设置为连续模式，它将继续在后台运行。

有了所有这些设置，你现在可以简单地下载 APK 文件并像打开其他文件一样打开它们。WSATools 将安装 Android 应用程序，它会显示在你的 Windows 11 开始菜单中。这比每次你想安装一个应用程序都要运行命令行要简单得多。开发者表示，更多功能正在开发中，例如自动安装 ADB 的能力，这在开发版本中已经可用。
