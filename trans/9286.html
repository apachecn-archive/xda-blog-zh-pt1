Canonical 正在改造 Linux 上的 Snap 应用程序包

原文：https://www.xda-developers.com/canonical-snap-rework/#0001-01-01

Snap 是一个用于 Linux 的软件打包和分发平台，由 Ubuntu Linux 的创建者 Canonical 开发。Snap 应用程序比传统的 Linux 软件具有更好的可移植性，并且大多数都是容器化的，以防止一些常见的安全问题。然而，Snap 也有很多问题，这可能是 Canonical 正在试验新架构的原因。

Canonical 在一篇新博客文章 ( via omg)中谈到了“Snapcraft 的未来”！ubuntu！ )，主要是把 Snap 框架分解成更小的模块化组件。关于最终结果会是什么样子，或者对于安装和使用 Snap 应用程序的普通人来说是否会更好，还没有任何确切的细节。然而，它应该使应用程序开发人员和 Canonical 更容易创建和维护 Snap 应用程序，这可能会为 Canonical 腾出时间来关注 Snap 框架的其他方面。

Canonical 表示，“基本概念围绕着将 Snapcraft 拆分成更小、更模块化和可重用的组件，这些组件可以在一系列不同的产品中使用。这项工作的共同基础是一组 Craft 库，正如我们在 Craft Parts 博客文章中已经讨论过的。该理论要求使用基于工艺供应商和工艺零件的通用零件生成器，并添加 Snapcraft 功能作为单独的一层。”

对于在 Linux 上分发应用程序来说，Snap 包无疑是一种更简单的方式，因为它们不必依赖于系统自己的包管理器，而不同桌面 Linux 发行版之间的包管理器并不总是相同的。例如，如果你想为 Ubuntu、Fedora 和 Arch Linux 开发一个应用程序，你必须维护三种完全不同的发布方式( PPA 、 RPM 和吃豆人)。相比之下，Snap 应用几乎可以在所有基于 Linux 的现代操作系统上运行:Ubuntu、Arch、Debian、Fedora、Majaro、Pop！_OS 等。

多年来，Snapcraft 因各种各样的问题受到了批评。不支持定制库或应用服务器，因此所有软件都必须通过 Canonical 自己的 Snap store 发布，Canonical 的没有发布 Snap store 服务器的源代码。集中式模型并不是每个人都喜欢，尤其是 Canonical 已经慢慢地用 Snap 版本(比如 Chromium )取代了 Ubuntu 中的核心应用。Linux Mint 完全阻止安装 Snap 应用程序，一些其他发行版已经认可 Flatpak 作为替代。Canonical 的博客帖子没有提到任何关于支持第三方商店和存储库的内容。
