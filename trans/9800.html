Windows 11 bug 在出厂重置后会在磁盘上留下一些用户数据

原文：https://www.xda-developers.com/windows-11-reset-bug-onedrive-files/#0001-01-01

在 Windows 11 和 Windows 10 版本 21H2 中新发现的一个 bug 是在工厂重置后将一些用户数据留在磁盘上不受保护。Rudy Ooms 在 Call4Cloud 博客上发现的这个漏洞意味着，如果你在重置后出售或赠送笔记本电脑，你的数据可能很容易被访问。这似乎与在本地同步到你的 PC 的 OneDrive 文件特别相关，因此仅限本地的数据不会受到影响。

Ooms 是通过尝试使用微软 Intune 远程擦除 Windows 11 设备而发现的，这是一种面向企业客户的管理套件。然而，经过进一步测试，该漏洞会影响使用设置应用程序在本地完成的工厂重置，即使使用云下载选项也是如此。

在重置过程中，用户通常会被询问是否希望保留他们的用户数据(并删除他们的应用程序和程序)，以便他们可以重新开始，或者他们是否希望删除驱动器上的所有内容。如果你打算卖掉笔记本电脑，推荐后一种选择，因为你不想让下一个所有者看到你的个人数据。在以前版本的 Windows 10 中，它可以按预期工作，但在版本 21H2 和 Windows 11 中，OneDrive 的用户数据保存在 Windows.old 文件夹中。这仅包括本地存储的数据，特别是同步到 PC 的 OneDrive 文件，包括使用已知文件夹移动同步的文件。

虽然打开文件不像直接进入文件夹并打开它们那么简单，但用户可以在 Windows 登录屏幕上按下 Shift + F10 来打开命令提示符并轻松访问这些文件。这不是每个人都会想到要做的事情，但这也不是一项困难的任务。这尤其令人担忧，因为即使驱动器在重置前进行了 BitLocker 加密，该加密也会在重置后被删除。

并不是每个人都会受到这个错误的影响，因为你必须使用 Windows 10 或 11 中的 OneDrive 集成才能暴露数据，而且只有离线时可用的文件才能在重置后访问，但这仍然是一个主要问题。出于某种原因，Windows 11 和 Windows 10 版本 21H2 更改了重置过程，使这些文件在重置后仍保留在磁盘上。

据 Ooms 称，微软正在努力解决这一问题，但在此之前，有一个 PowerShell 脚本将配置您的 PC 在重置后删除 Windows.old 文件夹中的所有用户数据。您需要在重置之前运行并部署这个脚本，这样数据就会在重置之后自动删除。

Windows 11 对重大 bug 并不陌生，包括一些大的安全问题。早在 11 月，一个早期的错误允许任何人获得 Windows 11、Windows 10 和 Windows Server 的管理员权限。
