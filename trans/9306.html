Linux Mint strikes 与 Mozilla 达成协议，将 Firefox 作为默认网络浏览器

原文：https://www.xda-developers.com/linux-mint-firefox-deal/#0001-01-01

Linux Mint 多年来一直是最受欢迎的桌面 Linux 发行版之一，它将 Ubuntu 基础与不同的桌面环境和内置应用程序相结合。上周刚刚发布了 Linux Mint 20.3，增加了几个新功能，现在该操作系统与 Mozilla 合作，将 Firefox 作为默认的网络浏览器。

Linut Mint 基于 Ubuntu 并使用 Ubuntu 的软件包库，但 Canonical(操作系统背后的公司)在去年的 Ubuntu 21.10 更新中将 Firefox 切换为“Snap”容器包。尽管 Snap 包通常比非容器化的 Linux 软件更安全，并且它使得跨许多不同的 Linux 发行版的分发更容易，但是并不是每个人都喜欢 Snap 包。特别是 Linux Mint 项目与 Snap 进行了斗争，理由是 Canonical 缺乏透明度和集中化的本质——除了 Canonical，没有人能运营 Snap 驱动的应用商店。

Linux Mint 在公布了一篇博文 ( via omg！ubuntu！ )宣布，它已经与 Mozilla 签署了一项新的商业和开发协议，将 Firefox 作为默认的网络浏览器。Linux Mint 将继续通过系统的软件仓库以. deb 包的形式分发 Firefox，就像它现在的工作方式一样(以及它在 21.10 之前在 Ubuntu 上的工作方式)，而不使用 Snap。然而，Linux Mint 添加到 Firefox 的所有定制都将被删除——起始页将不再是linuxmint.com/start，默认搜索引擎将恢复到谷歌，等等。

Linux Mint 在其博客文章中表示，“对于 Mozilla 来说，目标是让 Firefox 在所有平台上以相同的方式工作，以简化维护，简化开发和 bug 修复。有了这些改变，Firefox 将在 Linux Mint 中提供与在其他操作系统中相同的体验。对我们来说，这种变化意味着维护和开发方面的极大简化。我们曾经使用 Ubuntu 的打包程序自己构建 Firefox(随着 Ubuntu 转向 snap，该打包程序将停止使用)。我们现在打包了火狐的 Mozilla 版本。”

火狐也有官方的 Flatpak 包，由 Mozilla 发布，在 Mint 上运行(是 Snap 的替代产品)，但是 Linux Mint 显然选择了与 Mozilla 合作来保持标准。deb 版活着。更新后的火狐浏览器已经在 Linux Mint 20.3 上线，并将随着火狐 96 的发布陆续推出到 19.x、20.x、LMDE。
