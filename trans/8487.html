Windows 11 build 22494 在任务栏中为团队带来了静音按钮

原文：https://www.xda-developers.com/windows-11-build-22494-mute-button/#0001-01-01

正如通常在周三发生的那样，微软正在向今天注册开发频道的内部人士发布一个新版本的 Windows 11 。今天的 Windows 11 版本是 22494，它在任务栏上以静音按钮的形式带来了一个新功能。它还包括常见的修复和已知问题。

任务栏上的静音按钮是为了让用户在进行团队通话时，可以在任何屏幕上轻松地将自己的麦克风静音。是的，这意味着静音按钮仅在团队通话期间可用，但不仅如此，它仅适用于工作和学校版本的团队。需要澄清的是，这不是 Windows 11 内置的团队版本——与微软团队聊天——这是针对个人微软账户的。微软表示，它计划稍后增加这一功能，以便与微软团队聊天。

这实际上是在 6 月份与 Windows 11 一起宣布的，但直到现在才被内部人士知晓。然而，微软指出，这将通过服务更新进入稳定的 Windows 11，所以你不必等到明年的功能更新才能收到它。

一些 Windows 11 build 22494 的 Windows 内部人员也可能开始看到快照组出现在任务视图和 Alt+Tab 应用切换器中，类似于它们现在在任务栏上的显示方式。在设置应用程序中还有一些其他的改进，尤其是应用程序部分。你现在可以在默认应用部分看到搜索结果，而不必按下 Enter ，你可以用新的 URI 打开已安装的应用页面，并且该页面上的排序选项也进行了调整。

除此之外，这个新版本主要关注修复和较小的改进。您可以在下面找到完整的修复列表。

Windows 11 内部版本 22494 中的修复

【任务栏】

  • 将鼠标滑过音量、电池、网络或任务栏角落的其他图标后，工具提示将不再出现在任务栏的随机位置。
  • 解决了导致任务栏角落出现某些图标意外重复的潜在问题。

【文件浏览器】

  • 解决了当你试图滚动上下文菜单时，某些人会崩溃的问题。
  • 做了一些工作，以帮助解决一个问题，即在屏幕的某些区域，上下文菜单子菜单会绘制在上下文菜单的顶部，而不是旁边(例如，如果你悬停在新的)。
  • 现在，在具有混合 DPI 的多显示器系统上，上下文菜单图标应该不那么模糊了。
  • 解决了在某些情况下可能导致在上下文菜单中选择“打开方式”意外地仅打开文件，而不是实际打开“打开方式”对话框的问题。
  • 重命名桌面上的文件是备份和运行在这个航班。
  • 对命令栏底层逻辑进行了另一项调整，以帮助提高文件资源管理器中命令操作的性能。

【搜索】

  • 修复了最近的一个问题，该问题导致索引器数据库变得过于碎片化，从而导致索引器意外地长时间消耗大量内存和 CPU。对于拥有大型 Outlook 邮箱的人来说，这一点尤其明显。

【输入】

  • 缓解了当按住 Shift 或 Ctrl 键拖动某些应用程序时导致其挂起的问题。
  • 修复了一个问题，导致触摸键盘不出现在平板电脑上时，点击文本字段，如果你试图从登录屏幕重置您的 PIN。
  • 改进了笔菜单的可靠性。

【开窗】

  • 修复了一些与使用窗口功能(捕捉、ALT + Tab 和桌面)相关的 explorer.exe 崩溃。
  • 如果在有多台显示器的系统上打开任务视图，那么两台显示器上的背景都应该是丙烯酸。
  • 解决了任务视图和 ALT + Tab 中窗口缩略图的几个 UI 问题，特别是如果应用程序窗口太薄，关闭按钮可能会被切断。

【设置】

  • 解决了面部识别(Windows Hello)在某些情况下可能会在登录设置中意外变灰，直到关闭和打开设置的问题。
  • 修正了一个存储感知不能清理 C:\Windows\SystemTemp 的问题。
  • 标准用户(又名非管理员)现在应该可以在设置中更改时区，如果位置访问权限没有被授予，而不是下拉菜单变成空白。

【其他】

  • 修正了导致开发者的 Windows 更新、恢复和链接显示在主 Windows 更新设置页面下的问题。
  • 修复了在 HDR 模式下，Adobe Photoshop，Adobe Lightroom 和 Adobe Lightroom Classic 中的图像具有黄色色调的错误。
  • 缓解了一个与 DHCP 相关的问题，该问题在最近的一些内部版本中，当屏幕关闭时会导致意外的用电。
  • 做了一些工作，以帮助解决一个问题，其中服务主机:WinHTTP Web 代理自动发现服务将意外地利用大量的 CPU。
  • 修复了一个问题，可能会导致一些设备在睡眠中出现黑屏(锁屏不显示)。
  • 解决了一个潜在的问题，该问题导致一些使用 ARM64 PCs 的用户在过去几个开发频道版本中遇到更多的微软团队崩溃。
  • 我们已经增加了所选项目的填充，如通过单击文件资源管理器上下文菜单中的“显示更多选项”或任务管理器中的菜单选项所看到的。
  • WSL:修复了通过`\\wsl.localhost `或`\\wsl$ `访问 Linux 发行版时的错误 0x8007010b】问题#6995 )。

这个版本也有一些已知的问题，但是这是软件的任何开发版本都应该有的。以下是您在此版本中需要注意的内容:

Windows 11 内部版本 22494 中的已知问题

【常规】

  • 从版本 22000.xxx 或更早版本更新到使用最新开发频道 ISO 的较新开发频道版本的用户可能会收到以下警告消息:您尝试安装的版本是 Flight Signed。要继续安装，请启用航班签名。如果您收到此消息，请按下启用按钮，重启电脑，然后重试更新。
  • 一些用户可能会经历他们的屏幕和睡眠超时减少。我们正在调查更短的屏幕和睡眠超时对能耗的潜在影响。

【开始】

  • 在某些情况下，使用“从开始”或任务栏搜索时，您可能无法输入文本。如果遇到此问题，请按键盘上的 WIN + R 启动“运行”对话框，然后关闭它。

【任务栏】

  • 切换输入法时，任务栏有时会闪烁。
  • 我们正在调查此版本中的一个问题，即任务栏中的时钟可能会停滞不前且无法更新，尤其是在通过远程桌面访问电脑时。

【输入】

  • 剪贴板历史记录显示它是空的，即使它已启用并且应该包含内容。这是我们正在调查的一个 UI 问题——当一个航班修复后，任何固定的项目都应该可以再次使用。

【搜索】

  • 单击任务栏上的搜索图标后，搜索面板可能不会打开。如果出现这种情况，请重新启动“Windows 资源管理器”进程，并再次打开搜索面板。

【快速设置】

  • 我们正在调查来自内部人士的报告，即音量和亮度滑块在快速设置中显示不正确。

在第一个公共版本发布后，Windows 11 添加新功能的速度肯定会放缓，这只是预览版的性质。随着时间的推移，这些变化和改进将会越来越多，等到下一个功能更新面向公众时，普通用户将会有很多期待。我们有一个页面，如果你有兴趣了解即将到来的事情而不注册不稳定的版本，你可以在那里跟踪预览版中可用的所有 Windows 11 功能。
