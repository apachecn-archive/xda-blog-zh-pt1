这是微软的 Andromeda 操作系统在 Windows phone 上的样子

原文：https://www.xda-developers.com/windows-phone-if-microsoft-tried/#0001-01-01

Windows Phone 的故事相对较短，但对于微软智能手机的粉丝来说是一个悲伤的故事。微软在 2010 年推出了 Windows Phone 7，并在 2012 年和 2014 年分别推出了 Windows Phone 8 和 8.1，获得了更多的主流关注，微软有很多创新计划。

例如，操作系统专注于中枢，而不是应用程序，这是一个超越时代的想法。你会有一个人人中心，从各种社交网络引入内容，照片会从各种服务中引入你的照片。Windows Phone 在 2016 年成为 Windows 10 Mobile，虽然微软最初对它有很大的计划，但它只是悄悄地扼杀了它。甚至很少有手机被允许升级到 Windows 10，最后一次功能更新是在 2017 年。

但这并不意味着手机 Windows 的终结。微软有一个名为 Andromeda 的计划，这是一个软件(称为 Andromeda OS)和双屏硬件的结合，后来成为 Android 驱动的 Surface Duo 。

视窗中枢的扎克·鲍登实际上获得了仙女座菌株软件，但运行在微软的 Lumia 950 上，一个单屏设备。原因是从事仙女座操作系统工作的工程师实际上没有仙女座设备，所以他们与 Lumia 950 设备合作。

这里有很多东西需要解开，因为事实证明，仙女座菌株是一个相当激进的想法。它不同于我们见过的任何其他智能手机操作系统。

这方面最大的例子是它以日志为中心。当你解锁手机时，它会把你带到一个笔记屏幕，这可能不是用户会采用的东西。它还有一个单手开始菜单，这使它成为智能手机操作系统中罕见的不全屏选择应用程序的例子。

但是连锁屏都有点不一样。这两个快捷方式是相机和 Cortana。是的，这个版本来自 2018 年，当时 Cortana 仍然是一个像 Alexa 和谷歌助理一样的实际数字助理。当通知到来时，你可以按下 Cortana 快捷键与之交互。你也可以在锁屏上写字，算是给自己留便条的一种方式。它们会一直留在那里，直到你删除它们。

毫不奇怪，这个操作系统是基于手势的。这实际上是我们早在谣传的时候就知道的事情。你可以从左边滑动进入开始菜单，从右边滑动进入 Cortana，或者从顶部向下滑动进入控制中心。

许多这种东西都进入了实际的 Surface Duo，或者更具体地说，进入了微软使用的 Android 皮肤。一些手势是相同的，显然有相似的设计语言。

和往常一样，如果微软真的尝试了，看看手机上的 Windows 会是什么样子是很有趣的。事实上，这个平台失败的原因有很多，但一个关键的原因是，雷德蒙公司从未真正全力支持它。在三家资源似乎无限的公司之间的两匹马的比赛中，毫不奇怪，获胜的两家是愿意优先考虑移动平台而不是桌面平台的两家公司。

然而，Windows 10 Mobile 是一个不同的故事，因为它似乎在发货前就被内部扼杀了。微软承诺为任何运行 Windows Phone 8.1 的人免费升级，然后实际上为一小部分设备提供了升级。最重要的是，我们知道的许多内部计划的功能，如笔支持，都被废弃了。

当 Windows 10 for PCs 在 2017 年获得了代号为 Redstone 3 的秋季 Creators 更新时，Windows 10 Mobile 没有。它得到了一个功能更新——没有任何功能——来自一个完全独立的分支，代号为 feature2。当它在测试时，微软甚至没有告诉我们 feature2 是什么。一些人认为这些分支将合并成一个 Redstone 3，为 Windows Core OS 让路，另一些人认为他们只会在 Redstone 4 中合并回同一个开发分支。这两种情况都没有发生，因为 feature2 最终成为了 Windows 10 Mobile 的最后一次功能更新。

这就是为什么从 2018 年开始在手机上看到 Windows 的内部版本如此有趣。从我们看到的外部情况来看，微软刚刚认输了。如果你是这个平台的粉丝，看看可能会发生什么很有趣，但也有点令人沮丧。
