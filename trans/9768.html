谷歌货架计划在 Android 13 中增加对另一个私有 DNS 标准的支持

谷歌已经搁置了在 Android 13 中添加对另一个私有 DNS 标准的原生支持的计划。请继续阅读，了解更多信息。

Android 目前提供对一个私有 DNS 标准的支持——TLS 上的 DNS(DoT)。然而，谷歌一直致力于增加对另一个私有 DNS 标准的本地支持。去年 9 月，我们在 AOSP 发现了一个代码变化，表明谷歌计划在 Android 13 中添加对 HTTPS 域名系统(DoH)标准的原生支持。但最近的一项合并交易表明，该公司可能已经改变了主意。

根据最近合并的代码变更，谷歌默认不会在 Android 13 中启用 DoH。提交的描述声明:“DoH:默认情况下不要在 T 中启用它。”虽然这一声明并不意味着谷歌完全放弃为 Android 添加原生 DoH 支持的计划，但它确实澄清了这不会在 Android 13 提拉米苏中发生。目前，我们没有关于此事的进一步细节。但是一旦我们了解到更多情况，我们会尽快通知你。

对于不知情的人，DoT 和 DoH 是专用 DNS 标准，用于加密您的设备和域名服务器(DNS)之间的通信。虽然这两种标准执行相同的功能，但 DoT 使用 TLS(也称为 SSL)来加密 DNS 流量，而 DoH 使用 HTTP 或 HTTP/2 协议来发送查询和响应，而不是直接通过 UDP(用户数据协议)。

这两种标准也使用不同的端口，DoT 使用专用于 DNS 流量的端口，DoH 使用端口 443——所有其他 HTTP 流量都使用这个端口。这意味着在使用 DoH 时，您的所有 DNS 流量都与其他 HTTPS 流量混合在一起，这使得监控和阻止 DoH 查询变得更加复杂。从隐私的角度来看，这些差异给了 DoH 一点优势。出于这个原因，我们期待在 Android 13 中获得原生 DoH 支持。不幸的是，我们可能要再等一年，谷歌才能为 Android 添加原生 DoH 支持。
