Swift Backup 现在支持 Android 12 和通过滴的无根批量备份

原文：https://www.xda-developers.com/swift-backup-android-12-shizuku/#0001-01-01

备份应用程序以便安装在新设备上通常是设置新智能手机最简单快捷的方式。过去，诸如 Titanium Backup 之类的应用程序一直被用作全面的备份和恢复选项，尽管它需要 root 用户。然而，像 Swift Backup 这样的应用程序可以免费备份你的 apk、你的文本、你的通话记录和应用的壁纸，而无需 root。要恢复特殊的应用程序数据，如权限、电池优化等，您仍然需要 root 访问权限。然而，现在 Swift Backup 3.7 已经发布，它引入了对 Android 12 和通过 T2 滴 T3 的无根批量备份的支持。

对于外行人来说，滴有点类似于魔术师。然而，滴管理外壳访问，而不是根访问。滴使用 shell 级别的权限运行自己的进程。用户如何激活这个过程取决于他们的设备、Android 版本和选择。滴可以通过 ADB、通过设备上的无线 ADB(Android 11 及更高版本的)或通过 root 访问来激活。实现滴的应用程序可以请求使用该进程来执行提升的操作。对于 Swift 备份，滴并不像 root 用户那样授予相同级别的访问权限，但它允许进一步的访问。

从上面的 changelog 可以看出，Swift Backup 现在可以通过使用滴，批量备份支持的应用程序部分，批量恢复，批量启用/禁用应用程序，批量卸载应用程序，备份和恢复权限，以及切换每个应用程序的电池优化。以前，这些功能需要 root 访问权限，尽管仍有其他功能(如完整应用程序数据备份和恢复)需要 root 访问权限。配置滴很容易做到，它允许从你的智能手机授予特殊的 Android 权限。Swift 备份开发人员还表示，他们打算在未来的更新中整合你们支持的材料。
