Windows 11 KB5007215 更新为操作系统带来了更多修复

原文：https://www.xda-developers.com/windows-11-kb5007215-build-22000-318/#0001-01-01

今天又是这个月的第二个星期二，这意味着微软的补丁星期二到了。今天，每个受支持的 Windows 版本都在不断更新，包括各种修复和安全增强。对于 Windows 11 来说，这是一个特别大的更新，它将获得一个标签为 KB5007215 的更新，这将使内部版本号达到 22000.318。

这是一件大事，因为 Windows 11 上个月首次发布，尽管一周后就收到了一些修复，但几天后又出现了一些大问题。这包括 AMD 处理器上的 L3 高速缓存可能导致性能下降的问题。最近发现的另一个问题是，由于证书过期，导致一些内置应用无法运行。这些问题都得到了修复，但这些更新需要你手动下载，因为它们在技术上是预览版。

周二补丁更新是强制性和自动的，所以每个人都将获得 Windows 11 的 KB5007215 更新，尽管你仍然可以在这里手动下载。这意味着这些问题将为大多数用户修复。当然，更新包括的修复比我们目前在预览版中看到的更多，但似乎并不多。微软的亮点只提到了 Windows 11 的一般性安全更新。即使是完整的修复列表也只包含一个要点:

  • 解决了某些应用程序在渲染某些用户界面元素或在应用程序中绘图时可能会出现意外结果的问题。对于使用 GDI+并在每英寸点数(DPI)或分辨率较高的显示器上设置零(0)宽度钢笔对象的应用程序，或者如果应用程序使用缩放，您可能会遇到此问题。

除此之外，还有一个服务栈更新，旨在改善更新体验，并确保像这样的服务更新可以像往常一样继续提供。如果你没有手动下载任何更新，那么值得检查一下 Windows 11 build 22000.282 的变化，因为这些修复也包括在本次更新中。

微软还不知道 KB5007215 更新的任何已知问题，尽管关注 Windows 11 中的已知问题列表总是值得的。目前，列出的大多数问题都标记为已解决或已缓解。

通过这些更新，Windows 11 正在成为一个更强大的操作系统，这应该有助于加快采用。在我们的 Windows 11 评测中，我们发现它是 Windows 10 的一大进步，这些更新应该会减轻用户对新操作系统可靠性的担忧。
