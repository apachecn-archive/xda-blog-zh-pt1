Interop 2022 项目旨在修复跨 web 浏览器的不一致特性

原文：https://www.xda-developers.com/interop-2022/#0001-01-01

现在大多数网络浏览器共享一个通用的渲染引擎，谷歌的 Chromium，但 Safari 和 Firefox 也有自己的引擎，由其他项目共享(如 Tor Browser)。这些不同的架构意味着一些 web 功能在所有浏览器上的工作方式并不完全相同，这可能会成为 web 开发人员和浏览 web 的人头疼的问题，但一项新的努力正在进行中，以解决这一问题。

苹果、谷歌、Bocoup、微软、Mozilla 以及软件咨询公司 bo coup T1 和 T2 Igalia T3 正在启动一项名为 Interop 2022 的新计划。该项目旨在提高 web 浏览器之间的互操作性，确保所有重要的 web 功能和 API 在您使用哪种 web 浏览器的情况下都能正常工作。

苹果在 WebKit 开发博客上写道，“Interop 2022 是一个不断发展的指标，由一组自动化测试产生，旨在评估对 web 开发人员最重要的某些 web 标准的支持。Interop 2022 仪表板将在全年不断更新，随着浏览器工程师修复错误、实现新功能和改进测试，显示进度。”

新的 Interop 2022 仪表板根据“浏览器引擎在 15 个重点领域和 3 项联合调查工作中的表现”为不同的网络浏览器提供分数当前关注的领域是 CSS 层叠层、颜色空间和功能、CSS 包容、对话框元素、表单、滚动、CSS 子网格、版式和编码、视口单元和一般 web 兼容性。

希望这个新项目能减少 web 开发人员的头痛，并为每个人改进 web。现在谷歌 Chrome 是最受欢迎的网络浏览器，任何经常使用 Firefox 或 Safari 的人都可能会遇到一些因测试不佳或不兼容问题而崩溃的网站(或只是告诉你使用 Chrome)，但 Interop 2022 可能会减少这些情况。该集团还计划随着时间的推移增加更多的重点领域。
