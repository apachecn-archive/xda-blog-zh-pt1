谷歌正在为第三方 Android 应用开发超宽带(UWB) API

根据 AOSP 最新发现的提交，谷歌似乎正在为超宽带(UWB)准备一个公共 API。

去年，谷歌在 AOSP 增加了一个新的 API 来更好地支持超宽带设备。当时，我们了解到 API 仅限于系统应用。换句话说，第三方应用程序无法访问它。但这应该很快就会改变，因为谷歌现在正在为 UWB 开发一个单独的公共 API，第三方 Android 开发者将能够将其集成到他们的应用程序中。

根据 AOSP 最新发现的承诺，谷歌似乎正在为超宽带(UWB)准备一个公共 API。公共 UWB API 将允许第三方应用程序开发人员在支持的设备上利用超宽带技术，实现新的功能和用例。

该 API 还应该有助于推广基于 UWB 的跟踪设备的普及，这种设备提供的位置跟踪远远优于基于蓝牙的跟踪器。Tile 已经证实其即将推出的 Tile Ultra tracker 将是第一个在 Android 和 iOS 上提供 UWB 支持的产品。

首先，超宽带是一种短程无线标准，可以提供物联网设备的精确跟踪和定位以及高频无线电频谱上的高速数据传输。UWB 提供了优于蓝牙、Wi-Fi 和 NFC 的众多优势，被广泛认为是无线技术领域的下一个大事件。

苹果的 iPhone 11 是第一款配备超宽带支持的现代智能手机。在 Android 方面，三星是第一个将这项技术推向市场的公司，推出了 Galaxy Note 20 和 Note 20 Ultra 智能手机。小米也有一个未宣布的计划，即采用 UWB 技术，展示其如何利用该技术控制其智能家居生态系统。
