Dropbox 最终测试了 macOS 上的 M1 本地支持

Dropbox 发布了一个测试版，可以在苹果 M1 Mac 电脑上运行，不需要安装 Rosetta。

Dropbox 目前可能不是最受欢迎的云存储服务，特别是谷歌驱动和微软 OneDrive 通常提供更好的价格和功能，但许多人仍然使用它来保持不同设备之间的文件同步。Dropbox 一直在缓慢更新其 macOS 应用程序，以支持装有苹果 M1 芯片的 Mac 电脑，但现在终于实现了。

Dropbox 周三发布了桌面应用程序(viaArs Technica)的新测试版，标签为“Beta Build 140.3.1903”，这似乎是第一个原生运行在苹果 M1 Mac 电脑上的版本(而不是运行在罗塞塔 2 翻译层内)。目前还没有确定的时间表，M1 支持何时会退出测试版，并推出所有使用 Mac 的人，但该公司去年曾表示其目标是“2022 年上半年”新的 M1 版本应该比模拟的 x86 应用程序更快，消耗的内存更少。

该公司还发布了一篇关于 M1 支持的新支持文章,称:“Dropbox 对采用苹果硅(M1)处理器的 Mac 电脑的原生支持现已对所有选择加入 Dropbox 早期发布计划的用户提供测试版。你可以在你的苹果设备上期待同样的 Dropbox 体验，无论它们运行的是苹果芯片(M1)还是英特尔处理器。”

不幸的是，Dropbox 仍然没有适用于 ARM 或 ARM Linux 上的 Windows 的原生版本。普通的 Windows 桌面应用程序甚至不能在 ARM Windows 上模拟，大概是因为它挂钩到 Windows 资源管理器，而微软不支持模拟的 x86/x86_64 软件。骁龙 Windows 笔记本电脑的唯一选择是 S 模式的Dropbox，文件只能从 Dropbox 应用程序内部访问。

Google Drive 在 2021 年 10 月增加了原生苹果 M1 支持，而微软在 12 月开始测试 M1 客户端(稳定版本尚未推出)。苹果自己的 iCloud 文件同步内置在 Finder 应用程序中，因此它一直是 M1 本地的。
