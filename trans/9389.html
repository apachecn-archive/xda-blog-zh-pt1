谷歌为 Chrome 操作系统上的附近分享准备新的自我分享功能

谷歌为 Chrome 操作系统上的附近分享准备新的自我分享功能

谷歌正在为 Chrome OS 上的 Nearby Share 准备一项新的自我分享功能，这将让用户在自己的设备之间无缝分享文件。

在今年的 CES 上，谷歌宣布了 Chrome OS 的几个新功能，这些功能将帮助 Chromebooks 更好地与 Android 手机协同工作。该公司透露，它将在 Chrome OS 中添加一个新的快速设置功能，让你可以使用 Android 手机设置新的 Chromebook，并让它立即访问你所有保存的信息。此外，谷歌计划在 Chromebook 上引入新的手机中枢功能，这将允许你在 chrome book 上访问安装在手机上的聊天应用程序，而不必先安装它们。但这些并不是 Chrome OS 旨在提高 Chromebooks 和 Android 手机之间互操作性的唯一功能。

根据 Chromium Gerrit 上最近合并的提交文件(viaChrome Story)，谷歌还在努力改进 Chrome OS 设备上的附近共享功能，以帮助用户无缝地将 Chromebooks 上的文件共享到登录同一谷歌 ID 的手机或平板电脑上。提交的描述指出:

[附近][自我分享]为自我分享添加功能标志

这为附近共享添加了自己共享的特征标志。这将用于在发布之前对所有新的变更进行把关，包括初始原型。

尽管谷歌尚未在 nearly Share 中分享任何关于这一新的 Self Share 功能的官方信息，但我们相信，它将使用户能够在 nearly Share 屏幕上轻松访问连接到同一谷歌帐户的其他设备。实际上，这将允许用户在他们的设备之间快速共享文件，而不需要搜索和连接到接收设备。

Nearby Share 中的 Self Share 功能应该与 Chrome OS 设备上的 Phone Hub 中即将推出的 Camera Roll 功能配合使用，该功能将允许您在 Chromebook 上共享手机中的照片/视频，反之亦然。自我分享功能标志应该会出现在即将发布的 Chrome OS Canary 版本中。我们希望当它在金丝雀频道推出时能了解更多关于这个功能的信息。
