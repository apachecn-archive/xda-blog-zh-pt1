Windows 11 build 22000.346 向内部人士推出新的流畅表情符号

原文：https://www.xda-developers.com/windows-11-build-22000-346-fluent-emoji/#0001-01-01

微软在测试版和发布预览版渠道向 Windows 内部人士发布了 Windows 11 的新累积更新。这次更新使 Windows 11 升级到了 22000.346 版本，包括了微软几周前在 Dev 频道推出的新的流畅表情符号。它还包括大量的错误修复。

今天推出的新表情符号设计也支持表情符号 13.1 Unicode 标准，但大新闻是全新的设计，这是微软多年来首次更新。这些新的表情符号在网上引起了不小的争议，因为微软最初在夏天宣布了它们的 3D 动画设计。然而，Windows 11 中使用的表情符号设计是 2D 对原始版本的改编，meany 认为这些设计远没有那么吸引人。

无论如何，公平地说，新的表情符号比 Windows 以前使用的设计有了很大的改进，所以这不全是坏消息。

除了新的表情符号，这个版本还包括对 Windows 11 中广泛问题的大量修复，以及一些小的变化。例如，微软已经恢复了当设备停止工作或 Windows 崩溃时的蓝屏死机(BSoD)。Windows 11 最初将这块屏幕改为黑色背景，但现在又回到了经典的蓝色。以下是完整的变化列表:

Windows 11 内部版本 22000.346 的新增功能

  • 我们修复了影响 PowerShell 7.1 和更高版本上的 Appx PowerShell cmdlet 功能的问题。
  • 我们修复了一个问题，导致一些用户在启动时看到一个意外的“坏图像”错误消息对话框。
  • 我们修复了导致搜索索引器的问题。 exe 在远程桌面环境下的卸载操作中停止响应。
  • 我们修复了一个影响SearchFilterHost.exe进程开启的问题。
  • 我们增加了对斐济共和国在 2021 年取消夏令时的支持。
  • 我们修复了一个问题，该问题导致具有特定处理器的设备在从休眠中唤醒时停止响应。
  • 我们修复了 dll 中的一个 COM 初始化问题，该问题可能会导致调用进程停止工作。
  • 我们修复了 Hyper-V 虚拟机总线(VMBus)中的一个问题，该问题会导致 Windows 子系统 for Linux (WSL)虚拟机在连接磁盘时偶尔超时。此问题还会阻止实用程序启动。
  • 我们修复了休眠后影响系统内存管理单元(SMMU)故障处理的问题。
  • 我们修复了启用 Hyper-V 后导致系统停止工作的问题。
  • 我们修复了无法在启动时或在后台将计算机组策略对象自动应用到具有特定处理器的域中的设备的问题。
  • 我们修复了导致服务器管理器 cmdlet 返回失败的问题。因此，在安装可选功能的过程中，许多软件定义的数据中心(SDDC)验证会失败。
  • 我们添加了在接口上配置小于 576 字节的互联网协议版本 4 (IPv4)最大传输单元(MTU)的选项。
  • 我们修复了导致 get-winevent 失败的问题，错误是 InvalidOperationException。
  • 我们修复了一个错误地渲染一些可变字体的问题。
  • 我们修复了当您使用 Meiryo UI 字体和其他垂直字体时以错误的角度显示字形的问题。这些字体经常在日本、中国或亚洲其他国家使用。
  • 我们增加了一个功能，以方便某些跨浏览器的数据传输。
  • 我们修复了在 Internet Explorer 中打开对话框时出现的问题。
  • 我们修复了 CLSID_InternetExplorer 中的一个问题。
  • 我们修复了在使用输入法编辑器(IME)复制和粘贴文本时导致 Internet Explorer 停止工作的问题。
  • 我们修复了导致某些应用程序停止响应输入的问题。此问题出现在带有触摸板的设备上。
  • 我们修复了影响 Windows UI Library 3.0 (WinUI 3)应用程序中 WebView2 控件的触摸键盘部署问题。
  • 我们修复了在ctfmon.exe中当你在不同的编辑客户端之间切换时发生的内存泄漏。
  • 我们为电话号码错误的地区更新了 Windows 激活的电话号码。
  • 我们修复了一个已知问题，当连接到 Windows 打印服务器上共享的远程打印机时，该问题会导致错误代码 0x000006e4、0x0000007c 或 0x00000709。
  • 我们修复了一个影响 USB 打印设备的问题，该设备支持通过 USB 的互联网打印协议(IPP)。此问题会阻止这些 USB 打印设备完成安装。
  • 我们修复了一个问题，该问题会导致某些 USB 打印安装程序报告在您插入打印机后无法检测到打印机。
  • 我们修复了当调用 microsoft-edge: links 时操作系统功能可能被不正确地重定向的问题。
  • 我们修复了 Windows 音频系统中的一个问题，该问题可能会导致audiodg.exe进程停止工作，从而导致音频暂时丢失。
  • 我们修复了当您配置通用路由封装(GRE) VPN 带宽限制时，软件定义的网络(SDN)虚拟机无法工作的问题。
  • 我们修复了一个问题，该问题可能会导致 GetCommandLineA() 的返回值在某些开发人员场景中是小写的。
  • 我们修复了 VPN 用户在 VPN 连接离线时使用 Windows Hello for Business 登录时出现的主要刷新令牌(PRT)更新问题。对于在Azure Active Directory-条件访问中为用户登录频率(SIF)配置的在线资源，用户会收到意外的身份验证提示。
  • 我们添加了一条消息，表明组织的策略管理用户的位置隐私设置。当隐私设置由管理从 Windows 10 和 Windows 11 操作系统组件到微软服务的连接中记录的组策略控制时，会出现此消息。
  • 我们修复了影响 Fast Identity Online 2.0 (FIDO2)凭据提供程序并阻止显示 PIN 输入框的问题。
  • 我们修复了导致 Windows Defender 应用程序控制错误地比较两个文件版本号的问题。
  • 我们增强了 Microsoft Defender for Endpoint 识别和拦截勒索软件和高级攻击的能力。
  • 我们修复了当你戴上耳机时可能导致 Windows 混合现实启动的问题。即使您关闭了“当我的耳机的存在传感器检测到我戴着它时启动混合现实门户”选项，此问题也会发生。
  • 我们修复了一个音频失真问题，该问题会影响 Xbox One 和 Xbox 系列音频外围设备，并在您使用空间音频时发生。
  • 我们修复了一个问题，如果远程桌面客户端正在运行或 RemoteApp 断开连接，该问题会导致 AltGr 密钥停止工作。
  • 我们修复了快速设置中的编辑按钮和电池图标间歇性消失的问题。
  • 我们修复了影响通知区域中焦点辅助按钮的问题，并为屏幕阅读器提供了一个可访问的名称。
  • 我们更新了 Windows 表情符号的几个方面。作为迭代和持续工作的一部分，我们对此版本进行了以下改进:
    • 将 Segoe UI 表情符号字体中的所有表情符号更新为流畅的 2D 表情符号风格
    • 包括对表情符号 13.1 的支持，它:
      • 更新了表情词典
      • 增加了在所有支持的语言中搜索表情符号 13.1 的功能
      • 更新了表情符号和更多面板，以便您可以在应用程序中输入表情符号
  • 我们修复了一个影响未读通知数量显示的问题；有些数字不会出现在通知区域的圆心。
  • 当您安装大量应用程序并更改屏幕分辨率时，我们修复了一个影响“开始”菜单的问题。应用程序名称出现在“开始”菜单上，但缺少应用程序图标。当您在混合分辨率场景中使用辅助显示器时，此更新也可能提高“开始”菜单的可靠性。
  • 我们修复了一个问题，当你悬停在任务栏上的图标时会导致闪烁；如果您应用了高对比度主题，就会出现此问题。
  • 我们将开始菜单的易用文件夹名称更新为“Accessibility ”,以符合我们在 Windows 11(原始版本)中的命名标准。
  • 我们修复了影响 Microsoft 讲述人用户在设置中选择盲文选项的问题。
  • 当设备停止工作或出现停止错误时(如以前版本的 Windows ),我们将屏幕颜色改为蓝色。
  • 我们修复了一个问题，导致开始的所有应用程序列表中的一些应用程序图标在您更改屏幕分辨率后在底部被切断。
  • 我们修复了一个问题，在某些情况下，当您使用任务视图、Alt-Tab 或捕捉助手时，键盘焦点矩形不可见。
  • 我们修复了一个问题，该问题会影响在文件资源管理器和桌面上下文菜单中提供上下文(快捷)菜单项的应用程序。如果这些应用程序使用目录或目录\后台注册，就会出现此问题。
  • 我们修复了一个问题，该问题会自动从设备中删除塞尔维亚语(拉丁语)Windows 显示语言。
  • 我们修复了在通知区域显示 iFLY 简体中文 IME 图标背景不正确的问题。
  • 我们修复了一个问题，即当您关闭键盘而建议 UI 已展开时，触摸键盘底部会显示一个空白。
  • 我们修复了阻止显示文件资源管理器和桌面快捷菜单的可靠性问题。当您选择使用单击打开项目时，通常会出现此问题。
  • 我们添加了一个选项，供您选择是否在 Windows 功能更新后的第一个小时自动打开对焦助手。
  • 我们改进了任务栏上图标的动画性能。
  • 我们修复了锁定屏幕上影响网络状态文本呈现的可靠性问题。
  • 我们修复了影响蓝牙音频设备的音量控制问题。
  • 我们修复了导致文件资源管理器在您关闭文件资源管理器窗口后停止工作的问题。
  • 我们修复了一些视频显示不正确的闭路字幕阴影的问题。
  • 我们修复了一个问题，该问题导致设置应用程序中的 Windows Update 历史页面在列出更新时显示每个类别的零(0)个更新的摘要计数。
  • 我们修复了在 64 位版本的 Windows 11 上运行 32 位应用程序时出现的问题。如果调用netserver enum()，可能会返回错误 87 或者错误 1231。
  • 我们修复了一个阻止您的设备启动的问题，它会因为许可 API 调用而变得无响应。
  • 我们修复了 Windows 网络文件系统(NFS)客户端中的一个问题，该问题可能会阻止您在装载 NFS 共享后重命名文件。如果使用文件资源管理器重命名文件，会出现此问题；如果使用命令行重命名文件，则不会出现此问题。
  • 我们修复了一个问题，防止闪存驱动器，如 SD 卡和某些 USB 驱动器，出现在整理和优化驱动器用户界面。
  • 我们修复了一个问题，当您删除卷时，可能会导致 volmgr.sys 中的停止错误。
  • 我们修复了启用更新序列号(USN)日志时影响 NTFS 的问题。NTFS 每次执行写入操作时都会执行不必要的操作，这会影响 I/O 性能。
  • 我们启用了 onunload 事件，在 Microsoft Edge Internet Explorer 模式下创建弹出窗口。

自从 Windows 11 于 10 月 5 日首次亮相以来，我们已经看到了几个重大的漏洞修复更新，这是意料之中的。Windows 11 是一个大软件，推出后必然会出现漏洞。这些更新应该有助于使操作系统对那些在早期阶段犹豫升级的人更有吸引力。

Dev 频道的内部人士今天也有一些消息，微软推出了一个新的累积更新，使版本号达到 22499.1010。然而，这次更新没有包含任何新内容，微软表示这只是为了测试服务管道。
