《我的世界》的洞穴和悬崖第二部分更新现在可以在所有平台上使用

原文：https://www.xda-developers.com/minecrafts-caves-cliffs-part-2-update/#0001-01-01

《我的世界》是有史以来最受欢迎的视频游戏之一，Mojang 继续发布基岩版(手机版、Windows 10 版、游戏机版等)的新更新。)和 Java 原版。大规模的“洞穴和悬崖”更新于去年首次公布，但后来分成两次更新。第 1 部分于 6 月 8 日作为《我的世界》1.17 版发布，现在第 2 部分正在推出。

《洞穴与悬崖 Caves & Cliffs Part 2)，也被称为《我的世界》1.18 版本，现已在《我的世界》基岩版和 Java 版(viaAndroid Police)上推出。基岩更新增加了世界的高度和深度，“增加了 50%的垂直空间来建造和探索，”全新的洞穴世代，新的洞穴和山脉生物群落(如“郁郁葱葱的洞穴”和“锯齿状的山峰”)，改进的矿石和生物群落世代，新的音乐曲目，以及许多错误修复。有很多令人兴奋的事情，尽管你需要创造一个新世界来看到这些变化。

Java 版本更新有大部分相同的变化，包括更新的生物群落和矿石世代，新的音乐，更高的世界，和扩大的建造限制。《我的世界》Java 版现在需要 Java 17 或更高版本，但只要你用它自己的 Java 版本从官方的《我的世界》启动器打开游戏，一切都会为你更新。

基岩版变更日志

增加世界高度和深度

  • 新世界和现存世界的高度和深度已经延伸到 y320 和 y-64，为探索和建造打开了更多的空间

世界交融

  • 玩家现在可以更新现有的世界，包括新的超高尺寸和世界生成功能。更多详情请见我们的世界升级常见问题解答
  • y0 处的基岩层在保存的岩块下方，将被深板岩取代，并在下方形成新的洞穴
  • 跨越新的和现有的区块边界生成的生物群落和地形将无缝融合，创造出看起来自然的环境过渡

新地形和山脉生成

  • 地形形状和海拔不再总是由生物群落决定，导致了新的生物群落布局，如山顶上的沙漠
  • 山比以往任何时候都高，现在可以产生高达 256 块的最大高度

新的生物群落、3D 生物群落生成和生物群落分布

  • 增加了 3D 生物群落-洞穴生物群落现在可以直接在地表下生成生物群落
  • 增加了新的洞穴和山脉生物群落，包括锯齿状的山峰、草地、茂盛的洞穴和滴水石洞穴

新洞穴一代

  • 附加噪音洞穴——噪音洞穴的三种主要类型是大的、开放的奶酪洞穴、宽的隧道式面条洞穴和狭窄的、弯曲的面条洞穴
  • 附加的蓄水层——这些创造了当地水平的水体，如被水淹没的洞穴或地下湖泊

矿石分布和大型矿脉

  • 增加了大矿脉——这些长长的构造延伸穿过洞穴系统，比通常在地下发现的矿脉包含更多的矿石
  • 矿石分布已经被调整以补偿新的世界高度

暴民产卵

  • 怪物现在只能在完全黑暗的环境中繁殖。玩家可以使用光源在黑暗的地方产卵，比如洞穴

新音乐

  • 增加了莉娜·瑞恩和谷冈久美的新音乐
  • 增加了一张莉娜·瑞恩的名为“彼岸”的新音乐碟。这在要塞走廊箱子里很少能找到，在地牢箱子里更少
  • 移动播放器:要听新音乐，你需要下载更新的《我的世界》原创音乐包，可以从市场上免费获得

新的成就/奖杯

  • 洞穴和悬崖-从世界之巅(建筑极限)自由降落到世界之巅并幸存
  • 感觉像家一样——乘坐漫游者在 Overworld 的熔岩湖上漫游
  • 音乐之声——自动点唱机的音乐让草地变得生机勃勃
  • 明星交易者——在建筑高度限制时与村民交易

  • 本版本中的新世界世代目前与“自定义生物群落的创建”实验切换不兼容。预计自定义生物群落生成的世界可能是不稳定的，自定义生物群落只存在于世界上目前保存的区域。

更新的附加模板包

  • 为 1.18.0 更新的附加模板包含新的资源、行为和文档，可从aka.ms/MCAddOnPacks下载

修复

  • 如果服务器和客户端有不同的运行时块 id，播放器会断开连接
  • 修正了一个渲染错误，当试图在一个基础游戏版本中渲染一个普通的 mob 时，这个版本早于 mob 通过 JSON 文件引入的版本
  • 大多数内容错误和警告现在每个世界只显示一次( MCPE-135153 )
  • 改进了不动的怪物在“rangedattackgalgo”中的表现
  • 修正了当 JSON ( MCM-1390 )中没有指定时怪物的默认召唤能力
  • 修正了动画控制器的一个版本错误，该错误导致在动画控制器版本 1.8.0 的市场世界中使用了错误的模式
  • 函数中“/execute”触发的函数调用顺序现在是一致的( MCPE-111849 )

命令

  • “/clone”命令已更新，可正确检查重叠的源和目标区域，不再允许每个轴上有 1 个块深度的重叠。这可以通过强制克隆选项继续覆盖
  • 增加了“/loot”命令和繁殖战利品过载

配子框架

  • 新项目 API 更新:项目堆栈
    • 用于访问 ItemStack 上启用脚本的 ItemComponents 的公开接口。请注意，这些项目组件仅与通过 Holiday Creator 特性实验定义的自定义项目一起工作
    • has component(componentId:string)-如果 ItemStack 附加了组件[componentId],则返回 true
    • get component(componentId:string)-返回附加到此 ItemStack 的组件的句柄。如果组件不存在，或者组件尚未向脚本公开，则返回未定义的句柄
    • getComponents() -返回 ItemStack 上所有附加的启用脚本的组件的数组
  • 启用脚本的项目组件
    • minecraft:食物
      • 只读属性“营养”——描述该食物给玩家带来多少营养的数字
      • 只读属性 saturationModifier - number，它是用于在被吃掉时应用饱和度缓冲的饱和度修饰符
      • 只读属性 canAlwaysEat -如果为真，玩家可以一直吃这个物品(即使不饿)
      • 只读属性 usingConvertsTo -该项的字符串名称，它将在被吃掉时被转换为。如果为空，该项目将不会转换为任何其他内容
    • minecraft:耐用性
      • 只读属性 maxDurability 该物品在破碎前可以承受的伤害数量
      • 只读属性损坏范围-描述物品失去耐久性的几率的数字范围
      • 属性损害-获取或设置 ItemStack 上的当前损害
    • getDamageChance(unbreaking:number = 0)-如果给定一个牢不可破的等级，使用 damageRange 属性获取该物品被损坏的最大几率。传入的不间断参数必须大于 0
  • 向 TickEvent 添加了 deltaTime 只读属性，该属性表示最后一个级别 tick 之间的时间(秒)

Molang

  • 修复了当 Molang 表达式没有仅在“min_engine_version”为 1.17.40 或更高版本时触发的标记时的内容错误
