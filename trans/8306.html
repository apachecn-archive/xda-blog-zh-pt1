在 Windows 11 上使用 Android 应用

原文：https://www.xda-developers.com/hands-on-android-apps-windows-11/#0001-01-01

今天是我期待已久的一天。Android 应用终于可以提供给 Windows 11 测试版测试人员了。这些应用程序来自亚马逊应用商店；然而，事实证明，它实际上是一个精心策划的应用程序和游戏列表。只有 50 种可供选择。

奇怪的是，有一个栏目叫做编辑推荐。因此，这是一个精选的应用程序集合，但也有编辑的选择，但我跑题了。

为什么要在 Windows 11 上安装 Android 应用？

当微软宣布这一点时，我竞相尝试。该公司承诺在 Windows 11 中实现这一点，但它被推迟到最初发布之后。我的理解是，延迟是亚马逊应用商店，而不是 Android 的 Windows 子系统。

所以让我们来看看房间里的大象。为什么 Windows 11 上有安卓应用？每当我写关于这个话题的文章时，这可能是人们阅读文章前我最常被问到的问题。毕竟，你能用 Android 应用做些什么你已经不能在 Windows 上做的事情呢？如果没有原生应用，几乎所有东西都有 web 应用。

对我来说，它是亚马逊 Kindle，这也是我欣赏亚马逊合作伙伴关系的原因。亚马逊有一个名为 Kindle Cloud Reader 的网络应用程序，总体来说还不错，但对于触摸或离线使用来说并不太好。漫画是我想要的另一个东西，但我现在可以通过 Kindle 应用程序访问我的漫画书籍。

这就是我个人想要的。我希望能够坐在我的 Surface Pro X 旁边，用它做我用 iPad 做的事情。我很想停止在火车上携带 iPads 和 Kindles 来阅读。

但是对于其他人来说，游戏是重要的。几周前，当我与 BlueStacks 的首席执行官交谈时，我被告知，这项服务的几乎所有用户都用它在个人电脑上玩手机游戏。这是你将会看到的发展方向，也是为什么现在提供的许多应用只是游戏。

实际体验

为了开始使用，您需要成为测试版渠道的 Windows 内部人员。你可以从微软商店下载亚马逊应用商店，或者你可以尝试下载一个安卓应用，它会把你重定向到这个应用。

一旦你点击亚马逊应用商店上的 install，你将被引导完成整个过程，不仅应用程序会安装，Android 的 Windows 子系统也会安装。一旦完成，你将被引导重新启动。

一旦你的系统重启，你必须用一个亚马逊账户登录亚马逊应用商店。要从微软商店获得一些东西，必须登录第三方服务，这有点奇怪，但这是你的新商店。

登录后，您将看到所有下载应用的选项。需要明确的是，你也可以在微软商店找到所有这些应用程序，除了在微软商店，你只会看到一个按钮，带你回到亚马逊 Appstore 应用程序。

你可能会注意到，这不是一个超级流畅的体验。滚动感觉有点滞后。它似乎是为触摸而生的。用鼠标滚轮滚动感觉不如用触摸滚动，甚至用鼠标点击拖动。

我安装的第一个东西是我等待已久的应用程序，Android 版 Kindle。我还安装了 Comixology，尽管我不再经常使用它，我还安装了地铁冲浪者。

使用 Kindle 和 Comixology 之类的应用程序似乎和我预期的完全一样。我不知道我为什么感到惊讶，可能只是因为我不习惯在 Windows 上有像样的平板电脑应用程序。

玩地铁冲浪是一个不同的故事。我在 Surface Pro X 的 SQ1 机型上玩过，不太好。这让我想起了以前在 Surface RT 上玩的【T2 神庙逃亡:勇敢的。我只记得当时在想，为什么它在我见过的任何手机或平板电脑上都运行良好，而不是更好呢？图形是不稳定的，帧速率足够低，根本不足以获得适当的反应时间。

我提到它是 SQ1 处理器，因为我将在各种机器上尝试它。对于 x64 电脑，Android 的 Windows 子系统使用英特尔桥技术，而这些应用程序应该在 ARM 上运行。同时，Surface Pro 8 中的酷睿 i7-1185G7 比微软 SQ1 芯片组强大得多。

我想指出的另一件事是，获得应用程序的体验并不完全无缝。你在上面看到的图片是一个已经安装了 Comixology 应用程序的电脑的截图。显示“安装”或“打开”不是按钮的正常行为。你只需点击“从亚马逊应用商店获取”按钮，因为微软商店不知道它已经安装了。

当然，我还有很多想尝试的东西。我想在各种硬件上测试这些应用程序。我也想看看 app 更新进展如何。这些应用程序是会自动无缝更新，还是需要我通过亚马逊应用商店手动更新？

我也对侧装的过程感兴趣，这是完全可能的。微软承诺会很快回来，它做到了。我已经准备好对 Windows 上的侧载这个术语大加指责，因为我认为 Android 应用程序会像其他任何应用程序一样工作，但事实并非如此。你不能像安装 EXE 文件一样，双击一个 APK 文件来安装它。

因为它已经在测试版中，我们可能会很快看到它投入生产。
