一加诺德 2 的最新更新修复了根漏洞，但几个月来内核源代码仍然被破坏

原文：https://www.xda-developers.com/oneplus-nord-2-update-root-vulnerability-fix-broken-kernel-sources/#0001-01-01

公平地说，一加诺德 2 带有许多令人惊讶的元素。虽然其中一个惊喜是包含了联发科 SoC，但另一个惊喜最近以安全漏洞的形式出现了。该漏洞允许攻击者只需物理访问您的设备和手边的计算机，就可以访问 root shell，甚至无需打开 USB 调试。不过值得庆幸的是，OEM 已经通过发布快速更新相对快速地解决了这个问题。

针对一加诺德 2 的 OxygenOS 新的稳定渠道更新目前正在全球铺开。根据官方变更日志，最新的 A.14 build 优化了系统的稳定性。它还为欧洲和全球版本带来了【2021 年 11 月的 Android 安全补丁，因为他们在之前的更新中错过了这些补丁。事实上，这个版本也修复了根外壳漏洞，尽管一加在变更日志中没有明确提到它。

验证部分其实挺简单的。安装后，所有你需要做的是重新启动你的一加诺德 2 股票恢复模式。接下来，将设备连接到 PC/Mac 上，你应该根本找不到任何 USB 调试接口。

每当 OEM 厂商发布新的智能手机，我们通常会看到它的第一批 OTA 更新集中在优化和漏洞修复上。好吧，一加诺德 2 已经发生了这种情况，它的软件在发布近五个月后仍在继续工作。尽管如此，一加在发布内核源代码方面还是跟不上。

回想一下，该公司确实在 8 月份发布了一加诺德 2 的内核源代码。最初的版本基于 OxygenOS A.07。不幸的是，在一加的官方 GitHub repo 上看不到一个与后续更新相对应的公开提交。用户和开发人员理应获得更新的内核源代码，缺乏 Nord 2 内核源代码中如此重要的一部分意味着在这款设备上进行调整和修补可能会非常麻烦。

一加诺德 2 XDA 论坛

这种越轨行为不仅仅给开发者社区带来了不便，还应该受到谴责，因为一加没有及时提供 GPLv2 授权的源代码，实际上违反了通用公共许可证。例如，如果没有最新的资源，我们将无法获得完全工作的定制内核，定制 ROM 多样性最终将停滞不前，并对一加诺德 2 造成损害。

最终，如果这些问题持续存在，或者这种不愿意遵守 GPLv2 的情况在未来表现出来，一加在他们的设备背后建立的信任和热情将受到不可挽回的伤害。
