Android Studio 的模拟器在 Windows 11 上有问题

原文：https://www.xda-developers.com/android-studio-emulator-bug-windows-11/#0001-01-01

Android Studio 开发 SDK 最重要的组件之一是 Android 模拟器，它允许开发人员在广泛的虚拟化硬件上测试他们的应用程序。然而，Android 模拟器似乎还没有为 Windows 11 的发布做好准备，因为在使用 Hyper-V 时，模拟器会出现错误报告。谢天谢地，开发中也有一个修复。

更新 1(11/11/21@12:35ET):Android Studio Emulator 31 . 1 . 2 现已发布并对 Windows 11 上的 Hyper-V 进行了修复。它应该很快就会作为更新出现在 Android Studio 中。发表于【T11 月112021的文章保留如下。

至少有几十份报告在网上称，当使用微软在 Windows 中的原生虚拟机管理程序 Hyper-V 时，Android 模拟器无法在 Windows 11 上启动(T2 通过米沙·拉赫曼在 Twitter 上发布)。该问题会影响采用英特尔或 AMD 处理器的电脑，但使用 AMD 处理器的虚拟机管理程序驱动程序而不是 Hyper-V 的模拟器不会受到影响。运行 Windows 10、Linux 或 macOS 的电脑也不会受到该漏洞的影响。

谢天谢地，修复程序已经在开发中了。微软的开发人员证实，该问题是 Windows 11 上 Hyper-V API 中与 XSAVE 相关的一个 bug，谷歌和微软正在努力解决该问题。如果你现在需要一个解决方案，你可以删除 Android Studio 模拟器源代码中的一行，然后自己编译，或者用别人做的预编译版本替换一个文件。

即使 Android Studio 模拟器现在可能在 Windows 11 上(部分)崩溃，你仍然可以通过 Android 内置的 Windows 子系统在 Windows 11 上运行 Android 应用程序。事实上，有了侧载应用的能力，你甚至可以在 Windows 11 中测试你的 Android Studio 项目，而不用担心模拟器。谷歌的模拟器仍然是测试特定硬件功能的最佳选择，例如加速器运动、自定义 GPS 位置、可折叠界面等等。当然，如果你有一个实体的 Android 设备，你可以把它连接到你的电脑上来测试应用和游戏。
