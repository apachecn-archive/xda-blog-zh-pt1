用户对新的 Windows 11 表情符号不满意，这是有原因的

原文：https://www.xda-developers.com/windows-11-emojis-not-3d/#0001-01-01

昨天，微软在 Dev 频道向 Windows 内部人士发布了 Windows 11 build 22478 。该版本的主要特点是它包含了全新的表情符号。具体来说，这些新的表情符号早在 7 月份就已经发布，据说它们将出现在微软的所有产品中，包括 Windows 11 。事实证明，新的表情符号只有种来到 Windows 11，用户并不高兴。

事情是这样的:当微软在 7 月份宣布新表情符号时，它写了一篇长的博客文章解释制作它们的一切。该公司首先强调的一点是，新的表情符号是 3D 的，而不是该公司一直使用的平面 2D 设计(这些设计随着时间的推移有所调整，但一直是 2D)。如果你和我一样，这些新的美丽表情符号的前景令人兴奋，它与 Windows 11 的新设计语言完美契合。

然而，我们在昨天的建设中得到的不是我们在 7 月份看到的。事实上，Windows 11 build 22478 中的所有表情符号都是平面 2D 图像。本质上，它们类似于我们看到的表情符号，但任何阴影的迹象都被移除，并替换为平坦的颜色。以下是我们在 7 月份看到的表情符号与 Windows 内部人员可用的表情符号的一些比较。

当然，设计是主观的，2D 表情符号不一定比 3D 表情符号差。这取决于个人喜好，粉丝们的问题与此无关。问题是，微软明明承诺了一件事，兑现了另一件事。事实上，即使在最初的博客帖子发布一个月后，Windows 的官方 Twitter 帐户也提到了新的 3D 表情符号是 Windows 11 的一部分:

公平地说，不包括表情符号的“3D”阴影版本可能有合理的理由。阴影可能会影响设计的可见性，特别是在较小的尺寸下，并且对于有特定视觉障碍的用户来说，它们可能不容易接近。但如果是这样的话，微软一开始就宣布表情符号的阴影变体，然后强调这些表情符号将会出现在 Windows 11 中，这是没有意义的，而事实上，它们并没有出现。

归根结底，表情符号有点多余，你可以对视觉设计的任何方面说同样的话。然而，对许多人来说，良好的视觉设计有助于体验的整体享受，正如我们在 Windows 11 评论中强调的那样。微软还没有解释为什么选择 2D 的设计，但它确实看起来像这就是我们将坚持的。
