Boxedwine 可以在 web 浏览器中模拟 Windows 应用程序

原文：https://www.xda-developers.com/boxedwine-emulate-windows-applications-web-browsers/#0001-01-01

Wine 项目已经存在多年，允许人们在 Linux、macOS 和 BSD 操作系统上运行(一些)Windows 应用程序。这是开源软件开发的最好例子之一，但是 Wine 不能完全在 everything 上运行。Boxedwine 是一个基于 wine 的新项目，旨在解决这个问题——它可以在更多的操作系统上运行，甚至可以在网络浏览器上运行。

Wine 不是一个仿真器——相反，它将 Windows API 调用转换成 POSIX 兼容的调用。然而，Boxedwine 是一个仿真器。它使用未经修改的 32 位版本的 Wine，运行在模拟的 Linux 环境中。Boxedwine 是用 C++编写的，有一个 SDL 接口，所以它甚至比 wine 更具有跨平台性。支持 Mac 和 Linux，与普通 Wine 一样，但也可以在 Windows 上运行。如果您想要使用无法在现代 Windows 中工作的较旧的 16 位 Windows 应用程序，或者如果您只是想要一个不涉及设置 Windows 虚拟机的隔离环境，这可能会很有帮助。Boxedwine 也正在被移植到其他平台，包括俳句操作系统。

多亏了 WebAssembly 和 Emscripten ，Boxedwine 也可以在现代网络浏览器中运行。你可以在 boxedwine.org/demo 的尝试一些 Windows 应用程序和游戏的演示，包括帝国时代(1997) 、AbiWord 和3d 超级弹球:爬行之夜。Boxedwine 的速度已经达到了主机系统速度的 25%左右(据开发者称，在浏览器中运行的速度可能还会进一步下降，但在我装有锐龙 5 1500 处理器的电脑上，AbiWord 等基本应用程序可以与 Chromium 配合使用。

尽管浏览器中的 Boxedwine 在大多数情况下还不够实用，但它仍然是一个令人印象深刻的技术演示。该浏览器运行的是一个完整的 Linux 内核，一个未经修改的 Wine 副本，以及 Windows 应用程序，所有这些都在上面——没有来自服务器的视频流。

如果性能能够得到提高，Boxedwine 将来可能成为运行 Windows 应用程序的一个极好的工具。由于 x86 仿真是用可移植的 C++代码编写的，它可能是在 ARM 设备(如 Android 平板电脑)上运行 x86 Windows 应用程序的一种解决方案。Wine 已经可以在 ARM Android 上使用，但它只能运行为 ARM 编译的 Windows 软件——没有仿真层。

希望这个项目继续改进。如果你想尝试一下桌面版本，可以从box wine 网站获得预编译的 Windows 版本。
