如何使用 WSA PacMan 在 Android 的 Windows 子系统上安装应用程序

原文：https://www.xda-developers.com/wsa-pacman-gui-package-manager-windows-subsystem-for-android/#0001-01-01

WSA PacMan 是一个 GUI 包管理器和包安装程序，用于 Android 的 Windows 子系统

想找一个安卓版 Windows 子系统的包管理器？看看 WSA PacMan 吧，这是一个用于 Windows 11 的 GUI 应用安装程序和管理器。

Windows 11 的 Android 兼容层，通常被称为 Android 的 Windows 子系统(WSA)，没有谷歌 Play 商店或任何种类的谷歌应用程序。取而代之的是，亚马逊应用商店作为下载第三方应用的内置解决方案。虽然可以在 WSA 上手动下载应用程序，但没有简单的方法来管理 Android 子系统上所有已安装的应用程序——除非你安装一个启动器，或者更进一步启用 Play Store 。这正是 WSA PacMan 的用武之地。

顾名思义，WSA PacMan 是一个用于 Android 的 Windows 子系统的包管理器实用程序。这个开源工具由 XDA 资深会员阿莱西穆拉创建，旨在通过为 APK 文件提供一个易于使用的 GUI 安装程序来简化应用程序的侧装工作。它还会显示 APK 的完整包名、版本号、权限和图标，就像标准的包管理器应用程序在实际的 Android 设备上所做的一样。

在应用管理方面，WSA PacMan 提供了一个按钮，可以从 Android 子系统打开设置应用，这样你就可以根据需要更改底层参数。还有一个按钮可以直接打开“管理应用程序”页面，您可以在其中卸载或禁用已安装的应用程序，以及授予或撤销权限。此外，该工具的安装向导使您能够创建要安装在 Windows 桌面上的应用程序的快捷方式。

如何使用 WSA PacMan 在 Android 的 Windows 子系统上安装应用程序

  1. 从其 GitHub repo 的发布区下载 WSA PacMan 的最新版本并安装。
    • 如果你不想安装这个工具，也有一个可移植的版本。
  2. 确保 Android 环境的 Windows 子系统正在运行，并且调试选项已打开，然后打开 WSA PacMan。默认情况下，它监听端口 58526(这是本地调试 WSA 的默认端口)，因此它应该自动连接 Android 实例。
  3. 现在找到你想安装的 APK 文件，双击它。如果一切顺利，您应该会看到一个由 WSA PacMan 支持的安装向导。
  4. 新安装的应用程序应该会列在 Windows 11 开始菜单下。您也可以通过桌面快捷方式启动它，前提是您在安装过程结束时勾选了相应的选项。

我们过去见过像 WSATools 这样的 ADB 包装器，但是 WSA PacMan 的主要优势是可以方便地访问它的代码库。无论你是开发人员还是普通用户，你都可以去 GitHub repo 请求或者访问问题跟踪器。
