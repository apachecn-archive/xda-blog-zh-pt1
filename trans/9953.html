AMD 承认 Windows 11 和 10 上的 TPM 口吃问题

原文：https://www.xda-developers.com/amd-stuttering-windows-11-10-tpm/#0001-01-01

如果你在基于 AMD 的 Windows PC 上遇到了口吃问题，你可能会很高兴知道 AMD 最终承认了这个问题。几周前，关于该问题的报告开始在 Reddit 上流传，表明固件 TPM (fTPM)导致 AMD 系统在 Windows 11 和 10 上随机口吃。现在，AMD 已经发布了一个支持页面，详细说明了该问题，一个可能的解决方法，以及何时可以找到合适的修复方法。

AMD 证实了 Reddit 上最初的报告，称该问题是由 fTPM 引起的，该功能有时可以在一些内部没有独立 TPM 的台式电脑上启用。由于 TPM 是 Windows 11 的一个要求，它可能会在新的操作系统上更频繁地被注意到；但这在 Windows 10 上也是一个问题。根据 AMD 的描述，口吃是由于主板上 SPI 闪存中的扩展内存事务导致的，这会导致系统暂时停止响应。AMD 是这样描述这个问题的:

AMD 已经确定，部分 AMD 锐龙系统配置可能会在主板上的 SPI 闪存(“SPIROM”)中间歇性地执行与 fTPM 相关的扩展内存事务，这可能会导致系统交互性或响应性暂时暂停，直到事务结束。

如果你希望这个问题立即得到解决，你必须自己解决它。AMD 承诺将会以 BIOS 更新的形式为受影响的主板推出适当的修复程序。然而，这要到 5 月初才会到来，这个日期可能会推迟。您需要基于 AMD AGESA 1207 或更新版本的主板来获取更新。

另一个可能的选择是为您的 PC 获取一个分立的 TPM 模块，并禁用固件 TPM，从而规避该问题。然而，如果你正在使用任何 TPM 支持的加密解决方案，AMD 警告说，在切换 TPM 设备之前，你需要在 Windows 11(或 10)中禁用它，因为否则你可能会失去对所有数据的访问。这需要管理员权限。

AMD 一直处于 Windows 11 的一些问题的中心，性能受到 L3 缓存和 AMD 处理器中的任务调度程序的初始错误的影响。不过，AMD 和微软很快就解决了这些问题，所以情况正在好转。
