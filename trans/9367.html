Windows 11 build 22000.466 向内部人士推出了大量修复程序

测试版和发布预览版渠道中的 Windows 内部人员可以获得 Windows 11 build 22000.466，该版本将于周末前推出。

本月的补丁星期二似乎对 Windows 用户的修复很少，但测试版和发布预览版渠道的 Windows 内部人员今天获得了一些额外的改进。微软正在推出 Windows 11 build 22000.466，像往常一样，测试版频道的版本都是关于修复和改善体验的各种问题。

虽然它主要是关于修复，但在这个版本的变更日志中隐藏了一个附加内容。你现在会在设置应用的账户部分看到一个你的微软账户页面。这是在 Windows 11 家庭版和专业版中添加的。

更改列表包括广泛的修复，包括一个可能导致 Surface Pro X 等基于 ARM64 的设备在休眠或从休眠中唤醒时无响应的问题。现在在 ARM PCs 上安装 app 应该也更靠谱了。您可以在下面找到完整的列表:

Windows 11 内部版本 22000.466 的新增功能

  • 我们提高了 ARM64 设备上应用安装的可靠性。
  • 我们将约旦的夏令时从 2022 年 2 月开始，而不是 2022 年 3 月。
  • 我们修复了导致 ARM64 设备在休眠或从休眠状态恢复时停止响应的问题。
  • 我们修复了一个问题，该问题可能会阻止某些图像编辑程序在某些高动态范围(HDR)显示器上正确渲染颜色。这通常会影响可能以亮黄色或其他颜色显示的白色。
  • 我们修复了影响 Microsoft Edge Internet Explorer 模式下预测预渲染的问题。
  • 我们修复了一个问题，有时会阻止您在输入法编辑器(ime)中输入字符串。
  • 我们修复了导致音频服务在一些支持硬件加速蓝牙音频的设备上停止响应的问题。
  • 我们修复了一个问题，即通知客户 Windows 更新进度的文本在日语中不正确。
  • 我们修复了当应用程序不运行时影响图标的问题。在任务栏上，这些图标可能会显示为活动状态，就像应用程序正在运行一样。
  • 我们修复了一个可能导致 VPN 配置文件消失的问题。当您使用 Microsoft Intune 或第三方移动设备管理(MDM)工具在 Windows 11(原始版本)上部署 VPN 配置文件时，会出现此问题。
  • 我们修复了一个影响只与 Azure Active Directory (AAD)集成的应用程序的问题。这些应用程序将无法在加入 Active Directory 联合身份验证服务(ADFS)的计算机上运行。
  • 我们修复了一个问题，该问题可能会导致 Get-TPM PowerShell 命令在尝试报告可信平台模块(TPM)信息时失败。该命令失败，并显示错误“0x80090011 Microsoft。Tpm.Commands.TpmWmiException，Microsoft。Tpm.Commands.GetTpmCommand "。
  • 我们修复了导致远程桌面协议(RDP)会话断开连接或服务器核心屏幕空白的问题。安装 AppCompat 功能时会出现此问题。
  • 我们修复了一个影响 windows.system.profile.retailinfo.dll 的问题。
  • 我们修复了一些在您浏览文件和选择文件时影响文件资源管理器性能的问题。
  • 我们在家庭版和专业版的 Windows 设置中的帐户类别中添加了一个新的您的 Microsoft 帐户页面。
  • 我们修复了一个错误的显示任务栏中的音量图标为静音的问题。
  • 我们修复了导致文件资源管理器和桌面上下文菜单停止工作的可靠性问题。
  • 我们修复了当您使用韩文 IME 时，无法将 Shift KeyUp 事件传递到应用程序的问题。
  • 我们添加了 HelpWith 功能，它使用 Microsoft Bing 技术来建议与每个设置页面相关的帮助主题。
  • 我们修复了一个问题，当设备拥有 Microsoft 帐户(MSA)时，触摸键盘不会出现在锁定屏幕上。
  • 我们修复了一个影响任务栏上徽章信息加载的问题，这有时会导致设备停止工作。
  • 我们修复了一个阻止一些选项出现在 Win + X 菜单上的问题。
  • 我们修复了当设备连接到多个显示器时会导致设备停止工作的问题。
  • 我们修复了一个影响任务栏自动隐藏功能的问题。当您将鼠标悬停在主显示器或辅助显示器上时，任务栏可能不会可靠地出现。
  • 我们修复了一个问题，有时会阻止您使用简体中文 IME。
  • 我们修复了一个问题，可能会阻止图标出现在第二个显示器的任务栏上。
  • 我们修复了在安装打印机设备驱动程序时无法安装某些打印机配套应用程序的问题。
  • 我们修复了在设置中的蓝牙和其他设备页面上显示已连接蓝牙设备的过期电池百分比的问题。
  • 我们修复了阻止 IP 摄像机连接和流式传输到某些 DirectShow (DShow)应用程序的问题。
  • 我们改进了自动亮度算法，以便在所有支持的系统上，在弱光条件下提供更好的响应。
  • 我们修复了导致lsass.exe停止工作和设备重启的问题。当您在 NTDS 服务停止后查询 Windows NT 目录服务(NTDS)计数器时，会出现此问题。
  • 我们修复了导致 WebDav 重定向器死锁的问题。当您尝试从本地 TfsStore 读取文件时，会发生此问题，这会导致系统停止响应。
  • 我们修复了启用更新序列号(USN)日志时发生的性能退化问题。
  • 我们修复了在某些情况下无法应用组策略对象(GPO)“不允许在所有 NTFS 卷上压缩”的问题。
  • 我们修复了一个阻止 Robocopy 重试文件复制过程的问题。
  • 我们修复了导致 Windows 停止工作并产生错误“IRQL _ 非 _ 小于 _ 或 _ 等于”的问题。
  • 我们修复了调用WinVerifyTrust()时发生的内存泄漏。如果对具有多个签名的文件的第一个签名验证失败，就会出现此问题。

像往常一样，这些修复应该很快就会推广到普通用户。首先，在一月的下半月将会有一系列可选的累积更新，届时这个确切的更新将会对公众公开。然后，微软将把这些修复捆绑到二月份的补丁星期二更新中，这是强制性的，应该包括更多的改进。

说到周二补丁，Windows 11 和 10 的最新更新似乎给 VPN 用户带来了一些问题。如果您依赖 VPN 连接，您可能希望在安装本月的更新之前等待。
