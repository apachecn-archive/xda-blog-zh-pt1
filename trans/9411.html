欧盟委员会希望由欧盟建立并提供 DNS

原文：https://www.xda-developers.com/european-union-dns4eu/#0001-01-01

从表面上看，浏览网页似乎是一个非常简单的过程。在后台，当您在设备的地址栏中键入 URL 后，您的设备会向域名服务器(DNS)发送查询，将 URL 转换为机器可读的 IP 地址。一旦您的设备收到相应的 IP 地址，它就会打开网站。用户希望在智能手机上设置自定义 DNS 有很多原因；无论是为了避免内容过滤，为了隐私，还是出于任何其他原因，谷歌和 Cloudflare 等服务都提供自己的 DNS。然而现在，欧洲健康和数字执行机构(HaDEA)提出了一个由欧盟建立和提供的 DNS 基础设施，称为 DNS4EU。

在 DNS4EU 基础设施项目(由发现记录 )中，它说“部署 DNS4EU 旨在解决 DNS 解析在少数公司手中的这种整合，这使得解析流程本身在影响一个主要提供商的重大事件的情况下易受攻击。”虽然许多 DNS 提供商确实位于欧盟以外，但该项目页面也概述了欧盟出于网络安全和数据隐私的原因希望推出 DNS4EU。欧盟委员会在 12 月份概述了对主权 DNS 的需求。

DNS4EU 还可以过滤危险域上的非法内容，如恶意软件、钓鱼网站和其他网络安全威胁。被法院命令禁止的网站也可以被添加到过滤器中。DNS4EU 还需要完全遵守 GDPR，确保数据在欧洲处理，个人数据不能出售或货币化。

至于技术细节，该文件称，“服务基础设施应符合最新的安全和隐私增强标准(如 HTTPS、DNSSEC)，包括 DNS 加密(如 DNS over TLS (DoT)和 DoH)并完全符合 IPv6。”DNS-over-TLS (DoT)或 DNS-over-HTTPS (DoH)是私有 DNS 标准，可确保您的 DNS 请求得到加密。许多流行的 DNS 服务器，如 Google Public DNS、NextDNS 和 Cloudflare，都支持 DoT 和 DoH 标准。 Android 12 目前只原生支持 DoT，但对 DoH 的支持也在 Android 13 中加入。

欧盟似乎并不打算强制要求在欧盟使用这种 DNS 基础设施，因为它已经表示，将为用户提供指南，以便他们“通过一个专门的网站，在一个明确的品牌 URL 下”在自己的设备上配置它。它似乎也不是一个完全的公共服务，因为将有“针对特定部门需求定制的增强安全性的优质服务(例如临时过滤、监控、24x7 支持)”。
