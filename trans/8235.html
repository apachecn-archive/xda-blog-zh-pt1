Windows 11 泄露的截图展示了对 Android 应用程序的多实例支持

原文：https://www.xda-developers.com/android-apps-windows-11-multi-instance/#0001-01-01

自从微软在今年 6 月宣布 Windows 11 以来，我们一直在期待对 Android 应用程序的支持。虽然该功能最终没有和 Windows 11 一起发布，但 Windows 内部人士应该能够在未来几个月测试它。与此同时，最近在网上发现了 Windows 11 上运行的 Android 应用程序的截图，它们似乎证实了多实例支持。

这些图片是由中国社交网络 Billibilli 的一名用户分享的(通过 T3，Windows 最新版 T5)，尽管它们似乎已经被撤下。在图片中，我们看到两个社交网络应用程序在 Windows 11 上运行，但最值得注意的是我们在 Windows 11 任务栏上看到的微信应用程序。你可以看到微信在两个实例中运行，看起来是在应用程序的两个完全不同的部分。

Windows 11 对 Android 应用的多实例支持将是一件大事，尤其是考虑到许多 Android 手机不支持这种功能。这将极大地有助于多任务处理，因为你不必在给定的时间内停留在一个应用程序屏幕上。然而，我们不知道这种能力的完整程度，也不知道应用程序是否需要专门编程来支持多个应用程序实例。

除此之外，我们知道 Android 应用程序 windows 的大多数行为与任何其他 Windows 应用程序一样。你可以调整窗口的大小，或者将窗口贴在屏幕两侧，还可以将应用程序固定在你的开始菜单或任务栏上。应用程序也将能够支持鼠标、键盘、触摸等等。然而，还没有具体证实 Android 应用程序是否能够使用你的网络摄像头进行视频。

自最初发布以来，微软一直对 Android 应用保持沉默，但与此同时，一些内容已经泄露。最近在微软商店上发现了用于 Android 的 Windows 子系统，我们已经看到弹出的基准让我们一瞥在 Windows 11 上 Android 应用的性能。希望这些泄露的截图表明 Windows 内部人员将能够很快测试该功能。
