WSAPackagingTool 使得修改 Android 的 Windows 子系统变得更加容易

试图为 Android mod Windows 子系统？WSAPackagingTool 是一个易于使用的实用程序，可以根据您选择的 mods 重新打包 WSA 安装程序。

多年来，如果 Windows 用户想在他们的电脑上运行他们最喜欢的 Android 应用程序，他们必须通过第三方模拟器或其他非官方的方法来实现。现在，有了 Windows 11 和它的AndroidWindows 子系统(WSA)功能，你可以在你的 PC 上轻松运行智能手机应用。微软的 Android 环境附带了亚马逊应用商店，这是你可以为你的 Windows 实例获得 Android 应用的唯一官方来源。虽然可以在 WSA 上侧载应用程序或使用额外的包 管理器应用程序，但是安装谷歌 Play 商店需要手动修改底层系统映像。

Android 层的 Windows 子系统的内部架构与常规的 Android 设备有点不同。例如，没有 bootloader/fastboot 接口，通过它可以刷新目标分区。用户可访问的恢复模式也不存在，因此你不能用像 TWRP 这样的自定义恢复来代替它，并获得安装 mods 的能力。由于微软通过其商店发布和更新安卓的 Windows 子系统，在安装之前捕获 WSA 的安装包并修改它会更容易。这就是wsapakagintool项目的用武之地。

WSAPackagingTool 是由 XDA 知名开发人员 mlgmxyysd 和 XiaoMengXinX 创建的，它是一组脚本，旨在简化在本地修改 Android 安装程序的 Windows 子系统的过程。这个开源工具有两个组件——一个解包脚本，它提取 WSA 的有效 MSIX 捆绑包的内容进行修改；另一个重新打包脚本，它在您完成所需的更改后，从修改后的文件集创建一个新的安装程序。

任何使用 Windows 11(甚至 Windows 10)的人都可以运行 WSAPackagingTool，因为大多数依赖项都存在于工具集中。但是，重新打包模块需要 PowerShell 6.0 或更高版本，可以从这里的安装。

  1. 下载 Android 版 Windows 子系统的最新官方安装包。必要的步骤可以参考本教程。
  2. 直接从您的浏览器下载最新版本的wsapakagetool。
    • 或者，如果安装了 git，可以使用 git 克隆 WSAPackageTool GitHub repo。
  3. 解压缩 WSAPackageTool ZIP 文件，并将内容提取到一个文件夹中(例如 D:\ wsapakagintool-master)。
  4. 复制 WSA MSIX 包(带有扩展名。msixbundle)到前面提到的文件夹中。
  5. 将 msixbundle 文件拖到unpack.cmd开始解包。您也可以使用下面的命令解压文件:
     .\unpack.cmd <msixbundle>
  6. 成功解包后，可以在新创建的临时文件夹中找到解包的内容。
  7. 根据需要修改解包的 WSA 实例。例如，您现在可以在 Android 安装程序的 Windows 子系统上应用 WSA GApps 脚本。
  8. 完成后，运行repack.cmd文件开始重新打包。
  9. 重新打包的安装程序将放在“out”文件夹中。导航到那里，以管理员身份运行install.cmd文件，安装修改后的 msixbundle 包。
  10. 多亏了 WSAPackagingTool，您已经为 Android 实例准备并安装了一个定制的 Windows 子系统。快乐修补！
