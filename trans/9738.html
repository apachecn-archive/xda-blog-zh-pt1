Chrome 和 Firefox v100 可能会破坏一些网站

原文：https://www.xda-developers.com/chrome-firefox-100/#0001-01-01

谷歌 Chrome 是第一个采用快速发布周期的网络浏览器，Mozilla Firefox 最终也改用了同样的模式。今年晚些时候，这两种浏览器都将达到版本 100，这可能会在一些网站上引起一些意想不到的问题。谢天谢地，Mozilla 和 Google 正在努力解决兼容性问题。

Web 浏览器有一个用户代理字符串，它向站点标识自己，通常包含 web 浏览器、操作系统和 CPU 架构。用户代理应该如何呈现并没有标准，而且随着时间的推移，它们变得越来越长。例如，Linux 上 Chrome 的以下字符串包含单词“Chrome”、“Safari”(因为 Chrome 的引擎基于 Safari)、“KHTML”(Safari 基于该引擎)和“Gecko”(以避免破坏需要 Firefox 的旧网站)。

 Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/98.0.4758.80 Safari/537.36

许多库是为 web 开发人员创建的，以便轻松识别用户代理字符串的特定部分，例如实际的浏览器名称和版本号，但其中一些很快就会崩溃。一些库和其他 web 代码在构建时没有考虑三位数的主要版本号，这可能会给浏览网站的人带来意想不到的结果——只查看前两位数的网站可能会将 Chrome 100 视为 Chrome 10。

好消息是谷歌和 Mozilla 都在研究缓解策略(通过呼叫计算机)。Chrome 和 Firefox 已经进行了几个月的实验，改变了一些用户代理来报告版本 100，这已经导致了许多错误报告和一些修复。

但是，如果仍然有许多重要网站与 100 版本决裂，Google 和 Mozilla 有备份计划。Mozilla 目前的想法是识别被破坏的网站(它已经为其他功能这样做了)，并将这些网站的版本号冻结在 v99。谷歌的计划是将版本的第一位数字保持在 99，并将实际的主要版本移动到第二位数字(例如 Chrome 101 的 99.101.4988.0)。
