Android 12 允许 IT 管理员禁用 USB 数据以提高安全性

Android 12 带来了一些有用的企业级变化，包括允许 IT 管理员禁用 USB 数据以提高安全性的选项。

除了我们之前强调的所有新的 Android 12 功能，谷歌还在软件版本中加入了一些以企业为中心的变化。通过这些变化，该公司旨在让企业和 IT 管理员获得更多隐私控制和额外的安全措施，从而使 Android 成为一个更可靠、更安全的操作系统。其中一项功能为 IT 管理员提供了禁用企业设备上 USB 数据的选项，以提高安全性。

根据谷歌 The Keyword 博客上最近的一篇文章，Android 12 将让 IT 管理员可以选择决定员工可以使用哪些输入法编辑器(ime)来降低第三方键盘捕获设备数据的风险。他们还可以选择禁用除充电以外的任何 USB 信号，以降低基于 USB 的攻击的风险。但是，此功能仅在安装了 USB HAL 1.3 的设备上可用。谷歌提到设备管理应用必须调用 canUsbDataSignalingBeDisabled()来检查设备是否支持启用或禁用 USB 数据信号。

此外，IT 管理员将可以选择“将 5G 网络和专用连接切片到完全受管设备上的所有应用，或者专门连接到工作配置文件中的应用。”这将允许企业将优先手机服务限制在工作档案应用上，并给予他们更多“对服务质量和工作数据安全性的控制”

Android 12 还将包括其他一些以企业为中心的功能，包括批准或拒绝个人设备上传感器相关权限的选项，以及通过一种新的网络 API 建立 Wi-Fi 网络，这种 API 不需要用户位置权限。此外，Android 12 还更改了个人设备的硬件设备 id。

在本月晚些时候推出 Pixel 6 系列之后，这些变化应该会开始在 Android 12 稳定版本的设备上铺开。想了解更多关于 Android 12 中新的以企业为中心的功能的信息，请访问谷歌的博客文章。
