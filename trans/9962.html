Android 更新的 Windows 子系统增加了 H.264 硬件解码

原文：https://www.xda-developers.com/windows-subsystem-android-hardware-decoding/#0001-01-01

微软已经向该计划所有渠道的 Windows 内部人员发布了 Android Windows 子系统的更新。此次更新带来了一些重大改进，应该会使一些应用程序更加有用。其中之一是在 Android 的 Windows 子系统中支持硬件加速的 H.264 视频解码，这意味着 Android 应用程序在播放这种格式的视频时可以流畅得多。

一些应用程序中的渲染也有所改进。以前，多样本抗锯齿(MSAA)在应用程序上处于启用状态，无法禁用。虽然 MSAA 可以让游戏看起来更好，但它也会影响性能，所以这一变化应该有助于一些应用程序运行得更好。

另一个显著的改进与电子邮件应用程序有关。如果你安装了一个要求你打开电子邮件应用程序(例如，分享反馈)的 Android 应用程序，它现在会无缝地打开相应的 Windows 应用程序。

最后，微软还对 Windows 上的 Android 应用程序中的“一般输入”进行了一些修复。例如，这包括在亚马逊应用商店和 Kindle 应用中滚动。

不过，这个 Android 版的 Windows 子系统确实存在一些已知的问题。微软指出，在一些 Windows PCs 上，Android 应用程序中的视频播放可能会断断续续，这可能是启用硬件解码的结果。此外，当 PC 脱离连接待机状态时，Android 应用程序可能会重新启动。

如果您注册了 Windows Insider 计划，您可以立即更新 Android 的 Windows 子系统，方法是前往微软商店，然后前往库 > 获取更新。如果你还没有安装它，你可以在这里获得亚马逊应用商店，它会自动在你的 PC 上安装 Android 的 Windows 子系统。如果你不在美国，我们也有关于如何在 Windows 11 上下载 Android 应用程序的指南，这是唯一官方支持的地区。这些更新应该很快就会让非业内人士看到，因为 Android 应用程序自上个月起就可以使用了。
