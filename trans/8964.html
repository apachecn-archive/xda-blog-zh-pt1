危险的“Log4j”安全漏洞影响从苹果到《我的世界》的一切

原文：https://www.xda-developers.com/log4j-security-vulnerability-minecraft-apple/#0001-01-01

零日漏洞几乎是最糟糕的，尤其是当它们在像 Apache 的 Log4j 日志库这样无处不在的软件中被发现时。网上分享了一个概念验证漏洞，使每个人都面临潜在的远程代码执行(RCE)攻击，它影响了网络上一些最大的服务。该漏洞已被确定为“正在被积极利用”，是近年来公开的最危险的漏洞之一。

Log4j 是一个流行的基于 Java 的日志包，由 Apache 软件基金会开发， CVE-2021-44228 影响 2.0-beta-9 版和 2.14.1 版之间的所有 Log4j 版本。在几天前发布的本库最新版本2 . 15 . 0中已经打了补丁。许多服务和应用程序依赖于 Log4j，包括像《我的世界》这样的游戏，该漏洞是在那里首次发现的。Steam 和 Apple iCloud 等云服务也被发现存在漏洞，使用 Apache Struts 的任何人也可能存在漏洞。甚至更改 iPhone 的名字也被证明会触发苹果服务器上的漏洞。

这个漏洞是由阿里云安全团队的陈兆军发现的。任何记录用户控制字符串的服务都容易受到攻击。记录用户控制的字符串是系统管理员的一种常见做法，以便发现潜在的平台滥用，尽管这些字符串随后应该被“净化”——清除用户输入的过程，以确保没有对提交的软件有害的东西。

Log4Shell 竞争对手的严重程度令人心痛

该漏洞被称为“Log4Shell”，因为它是一个未经验证的 RCE 漏洞，允许整个系统被接管。网上已经有了一个的概念验证漏洞利用，通过使用 DNS 日志软件来证明它的工作是非常容易的。如果你还记得几年前的 Heartbleed 漏洞，当涉及到严重性时，Log4Shell 绝对让它物有所值。

“与 Heartbleed 和 Shellshock 等其他备受关注的漏洞类似，我们相信在未来几周内将会发现越来越多的易受攻击产品，”Randori 攻击团队今天在他们的博客中表示。“由于易于利用和广泛的适用性，我们怀疑勒索软件行为者会立即开始利用这一漏洞，”他们补充道。恶意行为者已经开始大规模扫描网络，试图找到可以利用的服务器(通过 呼叫计算机 ) 。

“很多很多服务都容易受到这种攻击。像 Steam、苹果 iCloud 这样的云服务，以及像《我的世界》这样的应用程序已经被发现存在漏洞，”LunaSec 写道。“任何使用 Apache Struts 的人都可能容易受到攻击。我们以前也见过类似的漏洞被利用，比如 2017 年的 Equifax 数据泄露事件。”LunaSec 还表示，理论上，高于 6u211、7u201、8u191 和 11.0.1 的 Java 版本受到的影响较小，尽管黑客仍然可以绕过这些限制。

这个漏洞可以由像 iPhone 的名字这样平凡的事情触发，这表明 Log4j 真的无处不在。如果一个 Java 类被附加到 URL 的末尾，那么这个类将被注入到服务器进程中。使用 Log4j 最新版本的系统管理员可以使用下面的参数来执行他们的 JVM，以防止漏洞被利用，只要他们至少使用 Log4j 2.10 。

 -Dlog4j2.formatMsgNoLookups=true

新西兰国家计算机应急小组(CERT NZ)已经发布了一个安全咨询警告，称有人在野外积极利用，这也得到了联盟工程安全总监蒂亚戈·亨利克斯和安全专家凯文·博蒙特的证实。Cloudflare 也认为该漏洞非常危险，因此默认情况下，所有客户都会获得“一些”保护。

这是一个极其危险的漏洞，可能会在网上造成严重破坏。我们将密切关注接下来发生的事情。
