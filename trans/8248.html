Windows 内部人员获得 Windows 11 上 AMD 性能问题的修复

原文：https://www.xda-developers.com/windows-11-fix-amd-performance-beta-channel-insiders/#0001-01-01

你可能已经知道， Windows 11 推出时带有一个 bug，导致 AMD 处理器在新操作系统上表现更差。然后，当本周的补丁周二更新推出时，似乎问题变得更加糟糕。现在，测试版和发布预览版渠道的 Windows 内部人员可以获得 Windows 11 build 22000.282 ，其中包含对最初 Windows 11 版本中影响 AMD 处理器的 L3 缓存问题的修复。正如之前报道的那样，L3 缓存的问题导致内存访问延迟增加，因此依赖内存访问速度的应用程序在最初版本的 Windows 11 上可能会慢 15%。

这次更新应该会将性能恢复到正常水平，但是，Windows 11 上 AMD 处理器的另一个问题似乎尚未解决。这就是处理器可能无法将任务分配给最快的可用内核的问题，这可能会影响仅使用单个或几个内核的应用程序的性能。AMD 似乎表示，这个问题的修复将作为“软件更新”推出，而不是 Windows 更新，所以我们可能需要等待 AMD 的新驱动程序。

除了对 AMD 处理器上的 L3 缓存问题的修复，今天对 Windows 内部人员的累积更新包括大量修复。这包括一些用户可能仍然看到 Windows 10 任务栏而不是新的任务栏等问题。您可以在下面找到完整的修复列表:

Windows 11 build 2200.282 修复了什么

  • 我们修复了一个 L3 缓存问题，该问题可能会在升级到 Windows 11(原始版本)后影响配备 AMD 锐龙处理器的设备上的某些应用程序的性能。
  • 我们为少数用户修复了一个问题，该问题导致开始菜单无法工作，并且在升级到 Windows 11(原始版本)后无法看到更新的任务栏设计。
  • 我们修复了在启动早期发生的可能导致停止错误的竞争情况。
  • 我们修复了在注册表中使用非 ASCII 文本的一些机器配置上可能导致 stop 错误 0x38 的回归。
  • 我们修复了某些处理器的中断处理可能导致设备停止响应的问题。
  • 我们修复了导致 PowerShell 创建无限多个子目录的问题。当您使用 PowerShell Move-Item 命令将目录移动到其子目录之一时，会出现此问题。结果，卷被填满，系统停止响应。
  • 我们修复了导致服务器管理器应用程序在您使用它删除 Hyper-V 功能后消失的问题。当您使用远程服务器管理工具(RSAT)在 Windows 11(原始版本)客户端上安装服务器管理器时，会出现此问题。
  • 我们修复了一个线程问题，该问题可能会导致 Windows 远程管理(WinRM)服务在高负载下停止工作。
  • 我们修复了导致 Windows Management Instrumentation(WMI)提供程序主机进程停止工作的问题。发生这种情况是因为在使用所需的状态配置(DSC)时发生了未处理的访问冲突。
  • 我们修复了一个问题，当您在更新组策略时强制关闭设备时，该问题会导致设备停止响应。
  • 我们修复了导致存储在不同卷上的分布式文件系统(DFS)路径之间的文件迁移失败的问题。当您使用使用 Move-Item 命令的 PowerShell 脚本实现迁移时，会出现此问题。
  • 我们修复了一个问题，该问题阻止您在内存不足的情况下写入 WMI 存储库。
  • 我们修复了 PowerShell 快捷方式文件中硬编码字体扭曲日语、中文和朝鲜语字体的问题。此更新为机器上所有新创建的用户修复了此问题。现有用户可以使用C:\ Users \ Default \ AppData \ Roaming \ Microsoft \ Windows \ Start Menu \ Programs \ Windows PowerShell \ Windows PowerShell . lnk文件打开 PowerShell 来解决该问题。或者，现有用户可以在桌面上创建该文件的快捷方式，并使用该快捷方式打开 PowerShell。
  • 我们修复了当毫秒被排除在外时解析事件中的时间格式的问题。
  • 我们修复了一个错误地渲染一些增强型图元文件格式(EMF)文件的问题。如果您使用带有 ExtCreatePen() 和 ExtCreateFontIndirect() 的第三方应用程序构建 EMF 文件，就会出现这个问题。
  • 我们为管理员提供了在 Microsoft Edge Internet Explorer 模式下将 HTML 对话框的缩放重置为默认值的选项。
  • 我们修复了企业模式网站列表从 Internet Explorer 11 重定向到 Microsoft Edge 的问题。在某些情况下，重定向会在 Microsoft Edge 的多个选项卡中打开一个站点。
  • 我们修复了当您在输入法编辑器(IME)中输入某些字符时导致 Internet Explorer 停止工作的问题。
  • 我们修复了 JScript9.dll的 PropertyGet 的一个问题。
  • 我们修复了在 VBScript 中使用嵌套类时发生的内存泄漏。
  • 我们修复了快速启动重启后无法保持 NumLock 状态的问题。
  • 我们修复了移动某些应用程序窗口的问题。如果屏幕上显示文件资源管理器窗口，移动这些应用程序窗口可能会异常缓慢。
  • 我们修复了一个间歇性阻止邮件应用程序在地址和主题框中接受键盘输入的问题。
  • 我们修复了在文件浏览器中使用文件夹视图重命名文件的 UI 问题。用户界面无法使用新的日语 IME 正确处理内联合成。
  • 我们修复了一个问题，即在凭据页面登录时，使用 App-V 会间歇性地导致黑屏。
  • 我们修复了一个互联网打印服务器的问题，该服务器无法在将包发送到客户端之前正确打包修改的打印机属性。

开发者频道的 Windows 内部人员昨天收到了一个包含新表情符号的新版本，这在粉丝中引发了一些争议。今天，这些内部人士也获得了累积更新，但它不包含任何新的内容，微软 it 的意思是测试开发频道的服务管道。
