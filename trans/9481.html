Magisk v24.0 版本引入了 Zygisk，带来了对 Android 12 的支持，等等

原文：https://www.xda-developers.com/magisk-v24-release-zygisk-android-12-support/#0001-01-01

XDA 资深公认开发人员 topjohnwu 的 Magisk 最初是一个无系统的 root 方法，经过多年的发展，已经成为一个比普通 root 更加多样化和强大的工具。但是即使在今天，如果你需要 root，很有可能为你的设备推荐的 root 方法提到了为超级用户访问安装 Magisk 。Magisk 的最新主要更新对应于第二十四个版本(v24)，现在已经到达公共测试分支，正式为所有 Android 12 设备带来全面支持。

Magisk XDA 论坛

开发者今天在 Twitter 上宣布了新的更新。除了对 Android 12 的支持，这个版本的下一个重要事情是引入 Zygisk (例如 Zygote 中的 Magisk)。这一功能开启了过多的修补可能性，因为模块开发人员现在可以直接在每个 Android 应用程序的进程中运行代码。为了简化改编，topjohnwu 还发布了 Zygisk API v2 的最终版本。文档和相关模块模板可在这里找到。

新建筑标志着 MagiskHide 的最终退役。不过，这并不出人意料，因为我们已经在金丝雀频道看到了 MagiskHide 的消失。Magisk v24.0 没有任何开箱即用的根隐藏功能，但您仍然可以使用第三方模块来规避根检测和通过安全网的。

此外，集中式 Magisk 模块存储库也永久消失了。因此，在 v24.0 中无法从 repo 下载模块。从现在开始，Magisk 应用程序将单独在模块中查找updateJson属性，并使用相应的 URL 来检查、下载和安装模块更新。

鼓励模块和根开发者阅读这里的发布说明。Magisk v24.0 的完整变更日志如下:

  • [常规] MagiskHide 从 Magisk 中移除
  • 【常规】支持 Android 12
  • [常规]支持不支持 32 位且仅运行 64 位代码的设备
  • [常规]将 BusyBox 更新到 1.34.1
  • [Zygisk]引入新特性:Zygisk
  • [Zygisk]引入 DenyList 功能，在用户选择的进程中恢复 Magisk 功能
  • [MagiskBoot]支持修补 32 位内核 zImages
  • [MagiskBoot]支持启动映像头 v4
  • [MagiskBoot]支持从 dtb bootargs 修补 skip_initramfs
  • [MagiskBoot]添加新的环境变量 PATCHVBMETAFLAG 以配置是否应修补 vbmeta 标志
  • [MagiskInit]支持从/system/etc 加载 fstab(Pixel 6 需要)
  • [MagiskInit]Support/proc/boot config 用于加载引导配置
  • [MagiskInit]更好地支持一些魅族设备
  • [MagiskInit]更好地支持一些一加/Oppo/Realme 设备
  • [MagiskInit]在某些 Sony 设备上支持 init.real
  • [MagiskInit]检测 DSU 时跳过加载 Magisk
  • [MagiskPolicy]从 system_ext 加载*_compat_cil_file
  • [MagiskSU]如果内核支持，请使用隔离的 devpts
  • [MagiskSU]如果设置了独立挂载命名空间，请修复根 shell
  • [resetprop]删除的属性现在会从内存中清除，而不只是取消链接
  • [App]为所有 ABI 构建单一 APK
  • [应用程序]切换到使用标准底部导航栏
  • [App]从集中式 Magisk 下载模块-模块-Repo 已删除
  • [App]支持启动映像 vbmeta 修补的用户配置
  • [App]恢复在某些 A/B 设备的另一个插槽上安装 Magisk 的功能
  • [App]允许模块为应用内更新+安装指定更新 URL
