Windows 11 内部版本 22468 已发布，VPN 设置有所改进

原文：https://www.xda-developers.com/windows-11-build-22468-is-out-with-vpn-setting-improvements/#0001-01-01

今天，微软向开发者频道发布了 Windows 11 内部预览版 build 22468 。再说一次，这个版本中没有什么有趣的东西。事实上，微软已经非常清楚，在预发布版本中出现新功能之前，我们必须等待一段时间。

有两个新特性。首先，如果你在你的 VPN 设置中点击一个连接，你会看到它的统计数据。此外，当你将鼠标悬停在任务栏中的搜索图标上时，可以选择删除最近的搜索。就是这样。

雷德蒙公司在其博客文章中还提到了其他一些内容。旧的开发频道版本将于 10 月 31 日到期，所以如果你在开发频道，你会想要升级。新的有效期是 2022 年 9 月 15 日。此外，新的 Paint 应用程序正在开发频道推出，但这并不仅限于 Windows 11 build 22468，而且已经宣布了。

和往常一样，有一堆修复和已知问题:

Windows 11 内部预览版内部版本 22468 修复

【任务栏】

  • 缓解了一个罕见的与加载网络图标相关的 explorer.exe 初始化挂起。

【搜索】

  • 改进了当鼠标悬停在任务栏中的搜索图标上时显示弹出按钮的可靠性。
  • 如果您使用键盘导航到任务栏中的搜索图标，导航到别处将会关闭最近搜索弹出按钮。
  • 在搜索某些应用程序(如 Word)时，右键单击最近显示的文件时的选项现在应该可以在您选择它们时工作了。

【文件浏览器】

  • 如果您在文件资源管理器中右键单击 OneDrive 位置中的文件，当您将鼠标悬停在打开子菜单的条目上时，上下文菜单将不再意外关闭，如“打开方式”
  • 双击一个网络文件夹来打开它，将不再意外地试图锁定到快速访问，而不是打开它。

【输入】

  • 解决了一个潜在的字体问题，该问题导致 shruggie kaomoji \_(ツ)_/的右手不能显示在正确的位置，以及在某些情况下的撇号。

【设置】

  • 您的首选麦克风输入格式设置(如声音设置中所配置的)现在应该持续升级。
  • 解决了一个问题，其中某些驱动器是出乎意料地不显示在整理和优化驱动器。

【其他】

  • 我们已经解决了阻止 MDM 注册的 PC 成功更新到先前版本的问题。这些设备现在可以升级到最新版本。
  • 修复了当使用具有不同刷新率的多台显示器时，可能导致某些应用程序(如 Microsoft Edge)意外闪烁的问题。
  • 缓解了一个显示相关的问题，该问题导致一些内部人员在最近的飞行中经历了错误检查的增加。
  • 做了一些工作来解决一个问题，即任务栏中的 Windows Update 图标可能会显示，但当您悬停在它上面时会突然消失。
  • 解决了睡眠后某些设备的一个罕见问题，即 Wi-Fi 将卡在关闭状态，试图将其切换回不工作。
  • 缓解了某些设备在某些情况下可能导致系统冻结的问题。

注意:在 10 月 5 日日全面上市后，来自积极开发分支的 Insider Preview builds 中提到的一些修复可能会成为 Windows 11 发布版本的服务更新。

Windows 11 内部预览版内部版本 22468 已知问题

【常规】

  • 从版本 22000.xxx 或更早版本更新到使用最新开发频道 ISO 的较新开发频道版本的用户可能会收到以下警告消息:

您尝试安装的版本是 Flight 签名的。要继续安装，请启用航班签名。

如果您收到此消息，请按下启用按钮，重新启动电脑，然后重试更新。

  • 一些用户可能会经历他们的屏幕和睡眠超时减少。我们正在调查更短的屏幕和睡眠超时对能耗的潜在影响。

【开始】

  • 在某些情况下，使用“从开始”或任务栏搜索时，您可能无法输入文本。如果遇到此问题，请按键盘上的 WIN + R 启动“运行”对话框，然后关闭它。

【任务栏】

  • 切换输入法时任务栏有时会闪烁。
  • 我们正在调查有关通知中心在最近的版本中将处于无法启动的状态的报告。如果您受此影响，重新启动 explorer.exe 可能会解决您的问题。

【搜索】

  • 单击任务栏上的搜索图标后，搜索面板可能不会打开。如果出现这种情况，请重新启动“Windows 资源管理器”进程，并再次打开搜索面板。
  • 搜索面板可能显示为黑色，并且不显示搜索框下方的任何内容。

【微件】

  • 小部件面板可能显示为空。要解决此问题，您可以注销，然后重新登录。
  • 外接显示器上的小工具可能会以错误的尺寸显示。如果您遇到这种情况，您可以首先在实际的 PC 显示器上通过触摸或 WIN + W 快捷方式启动小工具，然后在辅助显示器上启动。

Windows Insider 程序上的 Dev 通道是为来自 rs_prerelease 分支的构建而设计的。它是为测试新功能而设计的；然而，这些特性并不依赖于特定的版本。为了测试 Windows 11 的特定功能更新，这就是测试版频道的目的。

现在，Dev 频道带来了额外的不稳定性，但是没有新的特性。你现在可能想远离它。毕竟，你可以随时注册开发频道，但从开发切换回测试版并不容易，因为它需要工厂重置到一个较低的版本号。
