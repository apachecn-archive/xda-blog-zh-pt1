Windows 11 获得了一个新的打开文件的应用程序选择器

原文：https://www.xda-developers.com/windows-11-app-picker-build-22563/#0001-01-01

当你试图在 Windows 11 中打开文件时，微软正在为应用程序选择器 UI 进行新的设计。在 Dev 频道向内部人士发布的最新版本中， build 22563 ，你可以启用新的 UI，它看起来与 Windows 11 的其余部分更加一致。

新的设计用一个新的窗口取代了尖角和不透明的白色背景，该窗口使用半透明的背景和圆角，并遵循系统的亮/暗主题设置。这是继 Windows 11 build 22557 中添加的新任务管理器之后，使操作系统在视觉上更加一致的又一步。

为了启用该功能，您需要使用一个名为 ViVeTool 的程序，该程序允许您在 Windows 中配置功能 id。特性 id 用于启用或禁用某些特性，通常在特性完全可用之前添加。微软还使用功能 id 来 A/B 测试某些功能，其中一些用户将看到一种行为，而其他用户将看到另一种行为。最新版本中特性 ID 变化的完整列表可以在 GitHub 上找到，由开发者 Rafael Rivera 分享。

要启用新的 Windows 11 应用程序选择器，请下载并提取 ViVeTool，然后在该文件夹中运行提升的命令提示符。然后，使用命令vive tool add config 36302090 2启用该功能。完成后，您可以尝试打开一个文件并查看新的应用程序选择器。请记住，应用程序选择器仅显示尚未设置默认文件关联的文件扩展名，或者如果您最近安装了能够处理特定文件或协议的新应用程序。

值得注意的是，篡改特性 id 可能会导致不稳定，甚至比通常 Windows Insider 版本更不稳定。虽然这是一个相对较小的变化，但它可能会导致一些问题。很可能这个新的应用程序选择器将很快对所有内部人员开放。
