新的 Windows 11 版本改进了讲述人的声音和媒体控制

原文：https://www.xda-developers.com/windows-11-build-22543-insiiders-narrator-voices/#0001-01-01

尽管比往常晚了一天，微软还是向 Dev 频道的内部人士发布了 Windows 11 的每周版本。这一次，我们得到了 Windows 11 build 22543，它打包了一些用户可能会喜欢的变化。

最大的改进是可访问性，为讲述人添加了新的自然声音。你可能已经猜到了，在 Windows 11 中使用讲述人时，这些新声音应该会提供更自然、更舒适的聆听体验。一旦你安装了新版本，就可以下载新的声音，如果你选择安装它们，它们使用设备上的文本到语音转换技术，所以它们可以在没有互联网接入的情况下工作。这些新的声音被称为 Jenny 和 Aria，它们只适用于 Windows 的英语和美国版本。

除此之外，还有几个新的键盘快捷键，让你可以在讲述者声音和阅读模式之间快速切换，这样你就可以随时选择最适合你的方式，而不用回到设置应用程序。

另一个改进是媒体控制，继最近推出的新的音量和亮度指示器之后。Windows 11 锁屏中有一个新的媒体控件设计，它与 Windows 11 的其他设计元素更好地保持一致。

在桌面上，您会注意到当您调整快照组中的窗口大小时，活跃窗口将保持可见，而其他窗口将淡出，仅在半透明背景上显示应用程序图标。微软还做了这样的事情，如果你用键盘快捷键 Win + Alt + K 将团队中的麦克风静音，你会在任务栏上方看到一个你已经将自己静音的指示。

此外，微软已经向开发频道的所有内部人士推出了输入切换器的新丙烯酸设计，安装新版本时有一个新的进度动画，设置中的磁盘和卷以及存储空间页面的设计已经更新，以更好地适应 Windows 11。

除此之外，Windows 11 build 22543 附带了一系列常见的修复程序，如下所示。

Windows 11 内部版本 22543 中的修复

【常规】

  • 切换到性能选项卡时，任务管理器应该不会崩溃。
  • 音量的硬件指示器的更新弹出按钮现在将显示音量。
  • 修正了在 OOBE 给你的电脑命名时不允许使用某些字符(比如带元音字母的字符)的问题，尽管在完成 OOBE 后在设置中允许使用。
  • 解决了导致 OneNote 在最近版本中尝试打开插入了 pdf 的页面时崩溃的问题。
  • 缓解了启用 HDR 后导致用户界面无响应的问题。

【文件浏览器】

  • 修复了在文件资源管理器中从压缩文件夹中拖放文件时可能导致 explorer.exe 崩溃的问题。
  • 文件资源管理器中的上下文菜单现在在“复制为路径”旁边显示 CTRL + Shift + C 键盘快捷键。

【输入】

  • 修正了导致光标在某些应用程序中意外消失的问题。
  • 修正了从那里打开输入切换器时快速设置的崩溃。
  • 解决了有时中文简体 IME 用户的个性化>文本输入中候选窗口定制意外丢失的问题。

【任务栏】

  • 我们修复了导致隐藏图标弹出和快速设置图标出现两个箭头，以及日期和时间错位的问题。

【设置】

  • 缓解了一个问题，导致设置陷入暂停状态，无法启动。
  • 如果你选择关闭启动声音，这个设置应该会持续升级。
  • 修复了通知中显示您是否选择推迟完成设备设置的图标。
  • 快速设置中的 tab 键不再将焦点放在一个不可见的元素上。
  • 修正了一个问题，有时会导致快速设置上面的媒体控制不在屏幕上。

【开窗】

  • 任务视图中指示活动桌面的下划线现在将跟随您选择的强调颜色，而不是蓝色。
  • 缓解了导致最近飞行中 DWM 崩溃的 WER 问题。

和往常一样，还有一些已知的问题。不过，这是本周一份相当短的清单:

Windows 11 内部版本 22543 中的已知问题

【任务栏】

  • 切换输入法时任务栏有时会闪烁。

【搜索】

  • 单击任务栏上的搜索图标后，搜索面板可能不会打开。如果出现这种情况，请重新启动“Windows 资源管理器”进程，并再次打开搜索面板。

【输入】

  • 在 UAC 提示符下，IME 候选窗口不会出现。

【微件】

  • 任务栏左对齐时，温度等信息不会显示。这将在未来的更新中得到解决。

【旁白】

  • 从语音选择组合框中下载 Microsoft Aria(自然)后，如果语音没有自动切换，请重新启动讲述人。

如果你不是 Windows 内部人士，本周仍有一些令人兴奋的 Windows 11 消息。微软刚刚宣布，Android 应用将于下个月面向公众开放，同时内部人士在过去几周一直在测试一些变化。
