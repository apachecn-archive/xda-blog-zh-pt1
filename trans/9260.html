Renegade 项目旨在简化 Android 设备上 ARM64 Windows 11/10 的引导

原文：https://www.xda-developers.com/renegade-project-boot-arm64-windows-10-11-on-android-devices/#0001-01-01

Windows 作为常规 x86 PCs 的操作系统而闻名，但微软的操作系统也可以在几种具有非传统外形的设备上找到。得益于 Windows 11 的 64 位仿真能力，ARM 上的 Windows 也在稳步扩张。然而，当谈到智能手机时，这家位于雷德蒙德的科技巨头早就放弃了用自己的操作系统与 Android 和 iOS 竞争的想法。

尽管微软对移动生态系统三心二意，但修改者多年来一直在努力将 Windows 系统移植到 T2 现有的 T4 智能手机上。由于 Windows 从来不应该在这样的设备上运行，他们不得不破解底层固件来启动它。当然，售后开发社区需要对核心安装程序进行许多更改，并编写特定于设备的驱动程序。如果没有这个领域的强大背景，很难执行所需的步骤，但是如果做得好，我们可以像 Renegade Project 这样进行合作。

为什么没有直接的方法在标准的 Android 设备上安装 Windows，原因有很多。简而言之，让您的智能手机保持最新的定制 rom 是一回事，但用操作系统和平台固件之间的标准化软件接口取代主板支持包(BSP)是一种更复杂的情况。尽管智能手机芯片组上的现代引导加载器实现支持统一可扩展固件接口(UEFI)，但这些平台上的引导序列通常由各自的 OEM 定制。因此，启动任何兼容 ARM64 UEFI 的操作系统都是不可能的。这就是 Renegade 项目派上用场的地方。

第一步是从头构建一个固件接口，为此，Renegade 项目开发人员建议在目标智能手机的引导加载程序上编译一个 Tianocore EDK II 映像。您还可以在相应的 GitHub repo 的 版本部分找到针对特定设备的预编译二进制文件。接下来，从 UUP 转储下载 Windows 10 或 11 的 ARM64 版本，手动准备安装程序。因为擦除整个内部存储可以硬砖化设备，所以分区作业只修改/userdata分区。在应用了 Windows 预安装环境中的 WIM 软件包并整合了驱动程序后，您应该会看到 Windows 在您的手机上启动。

依赖于更多可以维护或移植到特定设备的贡献者的加入，Renegade 项目开发人员希望看到受支持设备的列表增加。说到这里，这里是官方列出的设备名单，这些设备已经确认能够引导 Windows 10/11 的 ARM64 版本，并取得了不同程度的成功。

  • AYN
  • 谷歌
  • LG
  • 魅族
  • 摩托罗拉
  • 诺基亚
  • 一加

我们真的很高兴看到这项倡议将如何结束，因为它看起来像是一个有前途的替代方案，为那些喜欢在手机上获得相同的 Windows 体验的人。要了解更多，请访问他们的网站，在那里你会找到关于这个项目的一切。如果你对 Renegade 项目感兴趣，可以在 GitHub 查看源代码。
