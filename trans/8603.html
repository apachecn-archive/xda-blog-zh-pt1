应用和游戏应该阻止用户定制智能手机吗？

许多 Android 应用和游戏会阻止用户启用开发者选项、ROM 或安装自定义 ROM。应该允许他们这样做吗？

Android 最大的优点之一是它给高级用户提供了大量定制设备的选项。用户可以轻松地将他们的手机植入安装定制的 rom和定制的内核，使用像 Tasker 这样强大的自动化工具，以及做更多的事情。然而，根植或修改 Android 手机确实有一些缺点。例如，一些应用和游戏拒绝在改装设备上运行。这就把我们带到了今天的讨论主题。

PUBG 手机开发商 Krafton Inc. 最近发布了一款名为 PUBG: New State 的现代版《皇家战役》。新游戏无法在启用了开发者选项的设备上运行，并将用户重定向到以下网页:

Krafton 的游戏发行说明将此列为已知问题，并声明:

"某些设备可能会启用 USB 调试，即使开发者选项已被禁用。请确保禁用 USB 调试和开发人员选项。

所以，如果你想玩这个游戏，你必须禁用 USB 调试和开发者选项。PUBG: New State 并不是唯一一个对用户施加此类限制的安卓应用。Pokémon GO 是另一个流行的游戏，以阻止根设备和启用了开发者选项的设备而闻名。许多银行和金融应用程序也拒绝在根设备或运行定制 ROM 的设备上运行。虽然有一些变通办法可以让你在有限制的情况下访问这些应用，但不能保证它们能在所有手机上运行。这使得许多高级用户的生活变得艰难。

尽管这些应用背后的开发者辩称，他们已经实施了这样的限制来减轻安全风险，但你认为应该允许应用和游戏阻止用户做像启用开发者选项这样简单的事情吗？由于 XDA 社区充斥着大量修改设备的高级用户，我们很想听听你对这个问题的看法。
