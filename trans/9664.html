Windows 11 build 22000.526 为更多圈内人士带来任务栏天气

原文：https://www.xda-developers.com/windows-11-22000-526-beta-release-preview/#0001-01-01

微软已经在 Windows Insider 计划的测试版和发布预览版渠道中发布了 Windows 11 build 22000.526。此次更新增加了微软上个月宣布的用于 Windows 11 的的功能，比如任务栏上新的 Widgets 按钮和团队集成。

对于小工具，新按钮意味着你现在可以直接在任务栏上看到你所在位置的当前天气，类似于去年添加到 WIndows 10 的新闻和兴趣功能。如果你把任务栏放在中间，Widgets 按钮现在会在左上角，通过图标和文本显示天气信息。此外，如果您将鼠标悬停在按钮上，Widgets 面板将自动打开，当您停止悬停时，它将关闭。如果你将任务栏向左对齐，按钮将保持在原来的位置，并以图标形式显示你当前的天气，但没有文本。

至于团队集成，Windows 11 build 22000.526 意味着你可以直接从任务栏快速共享一个团队会议窗口。当您将鼠标悬停在任务栏上相应应用程序的图标上时，将会看到该选项。此外，该版本在任务栏上包括一个新的静音图标，允许您在团队会议中静音麦克风，甚至无需打开“团队”窗口。这两个功能都只适用于 Teams 的商业/学校版本，而不适用于 Windows 11 内置的版本。

除此之外，Windows 11 build 22000.629 中还有一系列常见的修复和改进，旨在增强您的整体体验。微软 Edge、远程桌面等中的 Internet Explorer 模式的修复都包含在这里。这个列表相当长，所以你可以在下面找到它。

Windows 11 内部版本 22000.526 的新增功能

  • 我们修复了 Windows Server 2016 作为使用特定云计算虚拟桌面基础架构(VDI)的终端服务器运行时出现的问题。因此，服务器在运行一段时间后会随机停止响应。这也解决了一个回归问题，该回归主动检查以确保rpcss.exe中的 CSharedLock 被正确设置以避免死锁。
  • 我们修复了一个问题，该问题可能会导致设置中的时区列表对非管理员用户显示为空白。
  • 我们修复了一个问题，该问题会影响 Windows 搜索服务，并在您使用邻近运算符进行查询时发生。
  • 我们修复了在任务管理器中无法显示启动影响值的问题。
  • 我们修复了一个问题，当iexplore.exe在微软 Edge Internet Explorer 模式上下文中运行时，该问题会阻止 ShellWindows() 返回 InternetExplorer 对象。
  • 我们提供了在 Microsoft Edge Internet Explorer 模式和 Microsoft Edge 之间共享 cookies 的能力。
  • 我们修复了影响 Microsoft Edge Internet Explorer 模式下对话框的问题。
  • 我们修复了当您按 F1 键时导致 Microsoft Edge Internet Explorer 模式停止工作的问题。
  • 我们修复了导致动态数据交换(DDE)对象清理不当的问题。这可以防止会话断开并导致会话停止响应。
  • 我们修复了阻止某些低完整性流程应用程序正常打印的问题。
  • 我们为业务云信任引入了对 Windows Hello 的支持。这是 Windows Hello for Business 混合部署的新部署模式。它使用与支持快速身份在线(FIDO)安全密钥的本地单点登录(SSO)相同的技术和部署步骤。云信任消除了部署 Windows 的公钥基础设施(PKI)要求，并简化了 Windows Hello for Business 部署体验。
  • 我们修复了一个问题，当驱动程序受虚拟机管理程序保护的代码完整性(HVCI)保护时，您无法卸载和重新加载驱动程序。
  • 我们修复了一个影响静默 BitLocker 启用策略的问题，可能会无意中添加受信任的平台模块(TPM)保护器。
  • 我们修复了一个可靠性问题，该问题会影响远程桌面应用程序将客户端的本地驱动器装载到终端服务器会话的使用。
  • 我们修复了在文件资源管理器命令菜单和上下文菜单中将从右向左(RTL)语言文本显示为左对齐的问题。
  • 我们修复了阻止您使用 Windows Management Instrumentation(WMI)桥访问 LanguagePackManagement 配置服务提供程序(CSP)的问题。
  • 我们打开了浏览器中“开始”菜单的“推荐”部分中的 Microsoft Office 文件。如果设备没有适当的 Microsoft Office 许可证，并且文件存储在 Microsoft OneDrive 或 Microsoft SharePoint 中，就会出现这种情况。如果有许可证，文件将在桌面应用程序中打开。
  • 我们修复了导致登录时远程桌面会话的键盘和远程桌面协议(RDP)客户端不匹配的问题。
  • 当您将其他显示器连接到您的设备时，我们会将时钟和日期添加到其他显示器的任务栏中。
  • 如果任务栏居中对齐，我们将天气内容添加到任务栏的左侧。当您将鼠标悬停在天气上方时，微件面板将出现在屏幕左侧，当您停止悬停在该区域上方时，微件面板将消失。
  • 我们增加了直接从任务栏快速共享打开的应用程序窗口到微软团队呼叫的能力。
  • 我们修复了一个问题，当您将鼠标悬停在电池、音量或 Wi-Fi 等其他图标上时，会在任务栏的空白区域出现不正确的工具提示。
  • 我们修复了当您尝试写入服务主体名称(SPN)别名(如 www/FOO)而 HOST/FOO 已经存在于另一个对象上时发生的问题。如果 RIGHT_DS_WRITE_PROPERTY 在碰撞对象的 SPN 属性上，您会收到一个“拒绝访问”错误。
  • 我们修复了一个问题，即在您重新启动操作系统并登录后，网络驱动器上的离线文件会断开连接。如果分布式文件系统(DFS)路径被映射到网络驱动器，就会出现此问题。
  • 我们修复了一个问题，当你安装一个网络驱动器时，会显示两次验证对话框。
  • 我们添加了对热添加和移除非易失性内存(NVMe)名称空间的支持。
  • 我们增加了在任务栏上即时静音和取消静音微软团队呼叫的功能。在通话过程中，活动的麦克风图标将出现在任务栏上，以便您可以轻松地将音频静音，而不必返回到 Microsoft Teams call 窗口。

Windows 11 build 22000.526 中没有直接包含的是对 Android 应用的支持，但那是因为该功能已经可以用于发布预览版和测试版渠道。Android 的 Windows 子系统本质上是一个可以从微软商店下载的应用程序，所以它不需要像这样通过系统更新来交付。

所有这些新功能，包括 Android 应用程序，应该可以在本月晚些时候 Windows 11 的稳定版本上为用户提供。考虑到它们不包括在周二的更新中，它们可能只是可选的，所以大多数用户要到三月份才能得到它们。
