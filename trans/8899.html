Windows 11 显著降低了一些 NVMe 固态硬盘的速度

原文：https://www.xda-developers.com/windows-11-ssd-performance-downgrade/#0001-01-01

Windows 11 再次给用户带来了一些问题，这一次有大量报告称，使用该操作系统的 NVMe 固态硬盘性能下降。这些报告可以在多个 Reddit 线程上找到，也可以在微软的支持论坛(通过 Neowin )上找到，它们实际上并不新。关于这个问题的报道可以追溯到几个月前，当时 Windows 11 还处于预览阶段，但在发布后的两个月内，投诉仍然存在。

根据这些报告，在 CrystalDiskMark 和 AS-SSD(流行的 PC 驱动器基准测试应用程序)的支持下，Windows 11 的性能在某些情况下可能会比 NVMe 驱动器慢几倍。该问题特别影响随机写入性能，而大多数其他指标不受影响，但差异仍然非常显著。下面的基准测试，由用户 mahtidrudi在 Reddit 上分享，比较了运行 Windows 10 和 Windows 11 的相同 SSD。

问题的根源尚未明确，但已经出现了一些可能的原因。用户最初指出的一点是，基于虚拟化的安全(VBS)导致 SSD 性能下降，但这并没有弥补 Windows 10 和 Windows 11 之间的全部差异。

最近的发现表明，问题与哪个驱动器包含 Windows 11 安装有关。如果在辅助驱动器上运行基准测试，结果将与 SSD 的宣传值一致。下图由用户 PleasedPen25317 在微软支持论坛上分享，展示了在同一台机器上比较两个 2TB 三星 980 Pro NVMe 固态硬盘的差异有多大。

三星 980 Pro 是 PCIe 第四代固态硬盘，但旧的 PCIe 第三代驱动器也受到影响。这种差异在理论速度如此之高的驱动器上尤为明显，非 Windows 驱动器几乎快了五倍。

另一份最近的报告称，最新的 Windows 11 更新， build 22000.348 ，略微提高了 SSD 的性能，尽管它与 Windows 10 上看到的仍然相去甚远。Reddit 上的一名微软工程师在三个月前 Windows 11 仍处于预览版时承认了这个问题，但似乎一个修复程序没有及时推出，两个月后，问题仍然存在。随着用户对潜在原因的更多关注，希望微软能更快地解决问题。

Windows 11 已经受到了一些问题的影响，包括一些具体的降低了 AMD 处理器的性能，加上一些应用程序根本无法工作。与此同时，这些问题已经得到了修复，所以我们只能希望同样的事情会在这里发生。
