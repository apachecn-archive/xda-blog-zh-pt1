开发频道中新的 Windows 11 版本有更多的修复

原文：https://www.xda-developers.com/windows-11-build-22471-more-fixes/#0001-01-01

我们很少在周一看到 Windows 的新版本，但今天，这正是正在发生的事情。微软正在开发频道向 Windows 内部人士推出 build 22471 ，其原因似乎是 Windows 11 将于本周上市。

当然，这个新的开发频道版本与大多数用户本周将开始获得的版本不同。现在，开发频道的 Windows 内部人员直接从微软的活动开发分支获得构建，为未来的 Windows 11 更新做准备。尽管如此，距离 Windows 11 builds 在预览版中添加任何大的新功能已经过去几周了，build 22471 也不例外。微软强调的唯一变化是 TebletInputService 已被重命名为 TextInputManagementService。这是在支持笔的设备上处理笔输入的服务。

除此之外，这个版本都是关于修复的，而且包里有很多。这些修复应该有助于改善整体体验，但可以理解的是，它们并不那么令人兴奋。值得注意的是，微软表示，这些修复可能会在 Windows 11 的普通版本中推出服务更新，这意味着微软可能仍在使用这些积极的开发版本来准备 Windows 11 初始版本的发布后修复。以下是所有的新内容:

Windows 11 buiild 22471 中的修复程序

【任务栏】

  • 我们修复了一个潜在的问题，导致任务栏的隐藏图标弹出有时有尖角而不是圆角。
  • 使用向下箭头与桌面弹出的上下文菜单交互时，现在应该将焦点向下移动到菜单上，而不是取消它。

【文件浏览器】

  • 修正了关闭文件浏览器窗口时 explorer.exe 崩溃的问题。

【输入】

  • 解决了一个问题，即语音输入可能会失败，并显示一条错误消息，指出网络连接问题，而这实际上并不是失败的原因。
  • 如果您在日语 IME 的“假名”按键模式下使用触摸键盘，按键标签现在将正确显示按住 Shift 键时输入的字符。
  • 减轻了有时会导致 textinputhost.exe 在启动时崩溃的竞赛状态。

【开窗】

  • 在运行对话框中输入 wt 将会在前台而不是后台打开 Windows 终端。
  • 解决了在断开外部显示器并从睡眠状态恢复系统后，可能导致应用程序陷入最小化状态的问题。

【其他】

  • 解决了运行某些流量优化软件的内部人员的网络问题。这将提高使用 HTTP/3 的网站的性能。
  • 修复了一些设备无法更新到新版本的问题，错误代码为 0xc1900101。如果您仍然遇到此错误代码，请在反馈中心提交一份新的反馈。
  • 更新了 Windows Update 的“需要重启”对话框，现在显示为 Windows 11。请注意，只有在下一次飞行提示重启时，您才能看到这一更改的结果，因为您需要使用更改后的版本。
  • 修复了启用对比主题时可能发生的 DWM 崩溃。
  • 修正了 ARM64 电脑的一个问题，如果在升级到 Windows 11 之前安装，可能会导致某些应用程序不响应显示语言更改。
  • 解决了当 explorer.exe 从提升的进程启动时使用较低的内存优先级，影响在它之后启动的所有进程的性能的问题(问题#55 )。

尽管有很长的修复列表，这个版本中仍然有一些问题。以下是您应该做好的准备:

Windows 11 内部版本 22471 中的已知问题

【常规】

  • 从版本 22000.xxx 或更早版本更新到使用最新开发频道 ISO 的较新开发频道版本的用户可能会收到以下警告消息:您尝试安装的版本是 Flight Signed。要继续安装，请启用航班签名。如果您收到此消息，请按下启用按钮，重启电脑，然后重试更新。
  • 一些用户可能会经历他们的屏幕和睡眠超时减少。我们正在调查更短的屏幕和睡眠超时对能耗的潜在影响。

【开始】

  • 在某些情况下，使用“从开始”或任务栏搜索时，您可能无法输入文本。如果遇到此问题，请按键盘上的 WIN + R 启动“运行”对话框，然后关闭它。

【任务栏】

  • 切换输入法时任务栏有时会闪烁。
  • 我们正在调查有关通知中心在最近的版本中将处于无法启动的状态的报告。如果您受此影响，重新启动 explorer.exe 可能会解决您的问题。

【搜索】

  • 单击任务栏上的搜索图标后，搜索面板可能不会打开。如果出现这种情况，请重新启动“Windows 资源管理器”进程，并再次打开搜索面板。
  • 搜索面板可能显示为黑色，并且不显示搜索框下方的任何内容。

【小部件】

  • 小部件面板可能显示为空。要解决此问题，您可以注销，然后重新登录。
  • 外接显示器上的小工具可能会以错误的尺寸显示。如果您遇到这种情况，您可以首先在实际的 PC 显示器上通过触摸或 WIN + W 快捷方式启动小工具，然后在辅助显示器上启动。

一旦 Windows 11 本周发布，我们可能会开始看到更多的功能被添加到开发频道的版本中，但这不会在本周发生。微软表示，这是我们得到的唯一版本，因为它将忙于面向公众推出 Windows 11。然而，我们应该在下周回到正轨，这也是星期二补丁发生的时候。那将是 Windows 11 获得第一个服务更新的时候，这可能包括我们在过去几周的 Dev channel builds 中看到的一些修复。
