谷歌有一个用“主题 API”取代浏览器 cookies 的新计划

原文：https://www.xda-developers.com/google-topics-api/#0001-01-01

浏览器 cookies，尤其是跨域 cookies，可以很容易地在互联网上跟踪人们，并为有针对性的广告收集数据。这就是为什么现在许多浏览器都屏蔽第三方 cookies，从 Firefox 到 Vivaldi，谷歌一直在试图建立一个替代方案来解决一些隐私问题。该公司的第一次尝试并不成功，所以谷歌再次尝试主题 API。

Mozilla、Vivaldi、Brave 和其他浏览器都没有实现 FloC 的计划。

谷歌第一次尝试取代 cookies 是在 2019 年 8 月宣布的作为“隐私沙箱”。第一个实现是群组联合学习，简称 FLoC，它将跟踪的责任从 cookies 和第三方广告网络转移到浏览器。该技术将具有相似浏览历史的人聚集在一起，然后允许广告商在没有 cookies 通常提供的大多数个人身份信息的情况下锁定这些人群(“群组”)。谷歌去年开始在 Chrome 中测试 FloC。

FloC 比第三方 cookies 稍微好一点，但是它并不受许多隐私保护团体和公司的欢迎。电子前沿基金会(EFF)批评谷歌在可能的隐私风险得到充分解决之前向数百万 Chrome 用户推出 FloC， DuckDuckGo 建议人们关闭 FloC (并更新了其浏览器扩展以阻止所有 FloC 跟踪)。Mozilla、Vivaldi、Brave 和其他浏览器没有计划实现 FloC ，而苹果和微软采取了“等等看”的方式。

谷歌周二宣布，它正在用一种叫做 Topics API 的新技术取代 FLoC 提案。与早期的 FLoC 设计类似，它使用你的浏览器在本地生成广告商可以瞄准的群体，但现在它基于特定的主题，而不是将对多个主题感兴趣的人分组在一起。谷歌是这样解释的:

有了主题，你的浏览器会根据你的浏览历史，确定一些代表你那一周最感兴趣的主题，比如“健身”或“旅游”。话题只保留三周，老话题删除。这个过程完全发生在你的设备上，不涉及任何外部服务器，包括谷歌服务器。当你访问一个参与网站时，Topics 只选择三个主题，过去三周的每一周一个主题，与网站及其广告合作伙伴分享。主题使浏览器能够给你有意义的透明度和对这些数据的控制，在 Chrome 中，我们正在建立用户控制，让你看到主题，删除任何你不喜欢的，或者完全禁用该功能。

尽管这项新技术可能是对 FloC 的改进，但它可能不会更受欢迎。Topics API 依赖于你的网络浏览器，根据你的浏览历史生成你感兴趣的主题，本质上是把 Chrome 本身变成了一个广告定位平台。这比第三方 cookies 向广告商发送大量用户数据要好一点，但仍然不尊重隐私。

谷歌拒绝证实 Topics API 的最终实现是让 Chrome 用户选择加入还是退出，该公司也没有透露是否已经与其他浏览器供应商讨论了该标准。谷歌 Chrome 在浏览器市场占有大约 50-60%的市场份额(在移动市场约占 70%)，因此谷歌 T2 可以在没有微软或苹果的情况下继续前进，但这可能会给 T4 反竞争诉讼和罚款提供更多的弹药。
