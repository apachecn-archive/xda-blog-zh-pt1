以下是如何在 Android 的 Windows 子系统上安装启动器应用程序

原文：https://www.xda-developers.com/windows-subsystem-for-android-launcher-app/#0001-01-01

Android 的 Windows 子系统无疑是 Windows 11 的一个优秀特性，可以尽可能原生地在你的 PC 上运行 Android 应用。无论你是一名正在寻找无缝调试环境的应用程序开发人员，还是一名试图在运行常规 Windows 程序的同时运行他们最喜欢的Android 应用程序的普通用户，Windows 的集成 Android 层都非常方便。

在 Windows 11 上获取安卓应用不需要成为 Windows Insider

当谈到界面时，微软从自己的 Linux 子系统中借用了设计线索，这意味着 Android 应用程序的呈现方式与现代 Windows 应用程序相同。虽然 Windows 11 在某种程度上简化了 Android 应用程序管理过程，只要你使用 ADB 从亚马逊 Appstore 或 sideload 下载它们，就会在开始菜单中自动创建相关的快捷方式，但 Android 的 Windows 子系统并不提供专用的主屏幕来访问已安装的应用程序。

在 Windows 11 上用安卓应用动手

这种设计背后的理念非常明显:微软希望你对待 Android 应用程序与传统的 Windows 程序没有什么不同。问题是，不是每个人都同意这个提议。如果 Windows 无法创建应用程序快捷方式，您必须修改 WsaClient 可执行文件来从头创建它，或者直接从 ADB shell 调用它。你甚至可能希望有一个集中的空间来存放所有的 Android 应用程序，就像 Linux 中的桌面环境一样，在那里你可以自由地访问核心设置模块，并自定义它的每个角落。嗯，对于 Android 的 Windows 子系统来说，这并不是完全不可能的。你需要做的就是侧装一个启动器应用(H/T@米沙拉赫曼 )。

如何在 Android 的 Windows 子系统中安装启动器应用程序

  1. 抓取你想要的启动器程序的 APK 文件。既然有数百个，请务必提前查看我们的综合列表最佳 Android 发射器。
  2. 一旦你手里有了 APK 的文件，在亚行的帮助下把它放到。命令应该是:
     adb install <full_path_to_the_APK_file>
  3. 从“开始”菜单运行刚刚安装的启动程序的快捷方式。
  4. 瞧啊。

安装 Android 启动程序后，你注意到的第一件事是主屏幕的可用性。从那里，您可以访问应用抽屉以及 Android 实例的底层设置应用。想要整理已安装的应用程序吗？创建一些文件夹，并将相似的文件夹堆叠在一起。通过从设置中清除数据来修复故障应用程序？继续吧。见鬼，你甚至可以在 Android 主屏幕上添加一些很酷的小部件，并与 Windows 11 自己的小部件窗格结合使用。

请记住，微软用 Windows 的等效链接替换了许多 Android 设置应用程序的意图链接。这意味着在点击 Android layer 的设置应用程序的一些部分后，你可能会被重定向到 Windows 11 的设置应用程序。这完全正常，没什么好担心的。
