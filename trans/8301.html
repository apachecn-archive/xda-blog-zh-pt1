Android 应用程序现在可以在 Windows 11 的测试版中使用

原文：https://www.xda-developers.com/android-apps-available-windows-11-beta-channel/#0001-01-01

今天早些时候，在微软网站上发现了用于安卓系统的 Windows 子系统的文档后，T2 公司正式宣布安卓应用程序可以在 T4 的 Windows 11 上使用。奇怪的是，你需要注册测试频道，而不是开发频道，才能在 Windows 11 上访问 Android 应用的初始预览。微软表示，它希望首先在已经向公众提供的 Windows 11 版本上测试这项功能，这表明当它向公众推出时，这项功能将与典型的功能更新分开提供。

如果你想在 Windows 11 上试用 Android 应用，除了 Windows 11 的基本要求之外，你还需要一些系统要求。根据微软发布的 Windows 11 功能要求，你需要一台至少 8GB 内存的电脑，一个固态硬盘，第八代英特尔酷睿 i3，AMD 锐龙 3000 或高通骁龙 8c 处理器或更高。这些不是运行 Windows 11 的基本要求，但你需要满足 Android 应用的这些要求。

另一个要求，至少现在，是你的电脑的地区设置为美国，要访问亚马逊应用商店，你还需要美国的亚马逊帐户。

除此之外，您只需在电脑的 BIOS/UEFI 中启用虚拟化，并成为 Windows Insider 测试版渠道的一员。微软的博客帖子应该包括一个开始设置 Android 应用支持的链接，但它实际上还没有上线。然而，你需要做的就是从微软商店下载安卓的 Windows 子系统，你很快就会看到亚马逊应用商店出现在你的开始菜单中。

一旦你从微软商店安装了亚马逊应用商店，你将自动获得 Android 的 Windows 子系统，这是允许这些应用程序运行的。最初，亚马逊应用商店将只提供 50 款应用，包括 Lords Mobile 等游戏、Kindle 应用以及 Khan Academy Kids 等应用的儿童内容。不过，未来几个月还会有更多应用推出。微软并没有说你是否可以在商店外安装安卓应用。

这一推出确实带来了一些已知的问题，例如微软商店没有显示亚马逊应用商店应用的评级，以及同时运行多个应用时的性能变化。以下是已知问题的完整列表:

Windows 11 上 Android 应用的已知问题

【微软商店】

  • 微软商店不显示亚马逊 Appstore 应用的评论评级。

【亚马逊应用商店】

  • 如果您遇到亚马逊应用商店登录问题，请关闭广告拦截器。
  • 使用 Amazon Appstore 时，辅助功能选项尚未启用。

【安卓版 Windows 子系统】

  • 基于 ARM64 的蜂窝 LTE 网络接入问题。
  • 在 ARM64 上的某些应用程序中可能会看到显示视觉伪像。
  • 运行多个并发应用程序时，性能可能会有所不同。
  • 某些应用程序存在已知的摄像头问题。
  • 脱离现代待机模式的应用会遇到问题。

如果你是 Windows Insider 计划的开发者频道，你将不得不等待更长的时间才能访问 Android 应用程序。微软表示，它计划在“以后”将这一功能带给那些用户，但不是现在。

Android 应用最初预计将与 Windows 11 一起发布，但全面发布已经无限期推迟。我们目前预计在 2022 年的某个时候，但我们必须等待，看看测试阶段如何进行。
