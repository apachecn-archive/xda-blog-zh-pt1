微软概述了 2022 年 Windows Insider 计划的下一步

原文：https://www.xda-developers.com/whats-next-windows-insider-program-2022/#0001-01-01

微软一直在不断改进它为 Windows 提供服务的方式，这意味着 Windows Insider 计划也在不断变化。今天，该公司的 Windows Insider 项目负责人 Amanda Langowski 写了一篇博客文章概述了今年该项目的预期。

看起来最大的变化是 Windows Insider 程序的徽标，这是目前旧的忍者猫设计。现在它将会变得更加成熟，正如你从上面的图片中看到的。这三个化身代表程序中的三个频道。

Windows Insider 计划渠道保持不变

就这些频道而言，几乎没什么不同。开发频道仍然没有绑定到任何特定的功能更新。它是为测试新功能而保留的，博客文章重申，这些功能可能会也可能不会在下一次功能更新中发布。他们会在准备好的时候发货，或者干脆不发货。

然而，即使用户将明确地提供他们的系统稳定性，以支持尝试 Windows 开发的前沿，他们仍然可能不明白。微软很清楚它将继续它的 A/B 测试计划，它只向一些内部人士提供新功能。

这也变得有点奇怪。微软表示，虽然它将继续在其常规博客帖子中记录功能，但它不会记录功能的变化。此外，默认情况下禁用的功能将不会被记录。

测试版和发布预览版频道也没有改变。测试频道是你第一次看到实际功能更新的预览，或者如微软所说，它“更接近我们将向我们的普通客户提供的内容”。该公司还指出，由于 Dev 和 Beta 是分开开发的，可能会有 Beta 频道先获得新功能的情况。已经发生的一个例子是 Windows 11 上的 Android 应用支持。最后，发布预览频道仍然是为了测试在不久的将来非业内人士将会看到什么。

每个内部人员都应该知道的关键一点是，Dev 是实验性的。不保证会推出新功能，如果你觉得它对你来说太不稳定，你可能需要重新安装 Windows 来取消注册。

更多的体验包即将到来

微软谈论 Windows 功能体验包已经有一段时间了，早在 10 月份，它就推出了在线服务体验包。这里还有一个正在讨论的，那就是网络体验包。

然而，更广泛的一点是，这些是向 Windows 提供功能的方式，而不必推出功能更新。Windows 功能体验包可以将更新推送到 Windows 中的不同区域，而在线服务体验包的范围较窄。这些都是通过 Windows Update 提供的，但 Web 体验包实际上将通过微软商店提供。微软给出的例子是，一个 Web 体验包可能被用来更新 Widgets 体验。

当然，构建还是会一如既往地到来，尤其是在 Dev 频道。Dev 永远处于预发布状态，所以大多数构建来自 rs_prerelease 分支。当一个新的更新进入测试版时，最初可能是一个主要的构建，但随后可以通过累积更新来提供服务，直到它发布，通过各种各样的体验包添加功能。这些构建来自一个 _release 分支，前面有一个代码名，所以对于 Nickel 更新应该是 ni_release。

一旦事情最终确定，无论是功能更新，累积更新，还是体验包，这些东西都会在面向非内部人士之前进入发布预览频道。

最终，如果你是一个 Windows 内部人员，事情不会因为你而改变。Dev、Beta 和 Release Preview 频道和以前一样。你会在“设置”中看到 Windows Insider 程序的新图标，仅此而已。
