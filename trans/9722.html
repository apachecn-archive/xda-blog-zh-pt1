谷歌的安卓隐私沙箱可能有助于保护用户隐私

原文：https://www.xda-developers.com/google-android-privacy-sandbox-announced/#0001-01-01

到处都是广告。它们在街上，在电视节目和电影中，在我们的智能手机口袋里。谷歌知道有些人在隐私问题上有多谨慎，该公司一直在努力帮助保护用户隐私。不过现在，谷歌已经宣布了 Android Privacy Sandbox，这是一项为终端用户引入更多私人广告解决方案的多年计划。

Android 隐私沙盒建立在一个类似的概念上，该公司为 web 推出了一个类似的概念，以“主题的形式作为第三方 cookie 跟踪的替代。正如谷歌所说，其目标是在默认情况下保护用户隐私，同时仍然支持依赖有效广告来维持免费和广告资助应用程序的移动生态系统。这是一种独特的 Android 方法，新的 SDK 独立于应用程序的其他代码，但不会取代 Ad ID。

定义 Android 隐私沙箱的主要提议是承诺只显示相关内容和广告，测量数字广告，以及限制秘密跟踪。该公司打算利用 Topics 和 FLEDGE(第一个本地执行的群体决策实验)来获取相关内容。fragge 根据应用程序开发人员定义的“自定义受众”和他们应用程序内的先前交互来显示广告，所有信息都存储在本地。

谷歌表示，Ad ID 将至少存在两年，在完全淘汰之前，开发者将会得到很多通知。谷歌还告诉我们，将会发布一个开发者预览版，重点关注 Android 隐私沙盒。对于这些预览，开发人员将可以访问带有隐私沙盒 API 的 SDK 和包含所需模块的系统映像。一旦准备好发布，包括 SDK 运行时和隐私保护 API 在内的关键组件将作为模块化系统组件发布，您可能记得这是项目主线。

谷歌的时间表是，2022 年的 Q1 涉及最初的设计提案和设计反馈和迭代。开发者预览版将在今年晚些时候推出，测试版将在年底推出。最后，2023 年将开始大规模测试。这些预览版和测试版将独立于 Android 13 的发布节奏。一旦推出，设置应用程序中也会有面向用户的控件。

目前还不清楚使用这个 SDK 在短期内会给开发者带来什么好处，至少在 Ad ID 被淘汰之前是如此。如果你是一名想要阅读设计提案的开发人员，你可以查看 Android Privacy Sandbox 网站了解更多信息。
