高通与微软就 ARM 上的 Windows 系统达成了独家协议

原文：https://www.xda-developers.com/qualcomm-exclusivity-deal-microsoft-windows-on-arm/#0001-01-01

上周，我们报道了联发科正计划为 ARM 上的 Windows 开发一款芯片组。事实证明，ARM 芯片组空间上的 Windows 甚至可能比这更热，因为我们迄今为止只在 ARM PCs 中看到高通 SOC 是有原因的。事实上，高通与微软就 ARM 上的 Windows 达成了独家协议，通过与知情人士交谈，我们了解到该协议即将到期。

除了微软已经公开表示任何人都可以在 ARM 芯片上构建 Windows 之外，这真的不应该令人惊讶。高通不仅开始制造个人电脑芯片，还希望微软能编译 Windows 来支持它。不，这两家公司共同努力实现了这一目标。正因为如此，高通开始享受一点独家经营权。

我不知道的一件事是这项协议什么时候到期，只知道它阻碍了其他芯片厂商在这个领域的竞争。三星可能也想用它的 Exynos 处理器参加比赛，特别是考虑到它最近与 AMD 在图形能力方面的合作。这大概也是苹果硅 MAC 电脑不被官方支持运行 Windows 11 的原因，所以希望这也能改变。

高通与微软的关系一直很密切(当然，与微软和英特尔的关系相比，这算不了什么)。如果你回头看看 Windows Phone 时代，高通芯片是唯一受支持的芯片。在现代的 Windows Phones 时代，除了骁龙芯片组，你找不到任何一款手机。

在联发科的高管峰会和高通的投资者日之间，有一个非常明确的信息，即 ARM SoC 供应商绝对相信“Wintel”合作伙伴关系将会消失，向 ARM 的过渡不只是发生，而是不可避免的。自然，这意味着当它开放时，所有这些公司都想成为它的一部分。尽管高通已经领先一步，鉴于它已经这样做了几年，而且由于收购了 Nuvia，它将开始建立自己的定制硅。

早在 2016 年，微软和高通首次宣布了 ARM 上的 Windows。这是一件大事，因为它与失败的 Windows RT 的最大区别在于它能够模拟 x86 应用程序。设备在一年后发布，并在那之后不久开始发货。自那以后，微软在 Windows 11 中添加了 x64 仿真，以及 ARM64EC，这使得使用仿真插件在本地运行应用程序变得更加容易。当然，该公司还有更多工作要做，以使 Windows 在 ARM 上获得更好的体验。
