Adreno Tools 让你不用 root 就能在智能手机上轻松使用更新的显卡驱动

原文：https://www.xda-developers.com/adreno-tools-update-android-graphics-drivers/#0001-01-01

面向智能手机的网站往往只关注最新的设备，但很大一部分用户出于这样或那样的原因选择不升级他们的手机。除非硬件出现故障，否则这些设备中的许多可能还会使用数年，因为对它们的所有者来说，它们仍然工作得很好。虽然定制 rom 可以让传统智能手机存活很长时间，但如果没有官方制造商的支持，让相机和图形驱动程序保持最新对改装者来说可能是一场噩梦。

在 Android 上更新 GPU 驱动程序

与智能手机生态系统不同，更新你电脑上的显卡驱动程序是你可以经常考虑的事情。这是可能的，因为 PC 硬件驱动程序的模块化架构允许高级用户修补闭源驱动程序包，甚至选择开源驱动程序，而不是 OEM 提供的二进制文件。然而，在 Android 的情况下，驱动程序更新通常只和更大的操作系统更新一起出现在你的手机上。

可更新的 GPU 驱动程序可以方便地修复错误，提高图形性能，或者从 OpenGL 或 Vulkan APIs 添加新功能。如今，智能手机比以往任何时候都更大更快，这使它们成为完美的游戏工具，因此将类似 PC 的驱动程序设计移植到它们身上确实有意义。

事实上，在通过 Project Treble 模块化 Android 的努力中，谷歌实际上使得将特定平台的图形驱动程序集包装为应用程序并通过 Play Store 更新成为可能。从那时起，几个主流的 Android OEMs 厂商与谷歌和主要的 GPU 供应商合作，包括 ARM 和高通，为他们的手机发布独立于更大软件版本的 GPU 驱动程序更新。

可更新的图形驱动程序设计是否减少了碎片？

是的，在某种程度上。尽管如此，一些令人担忧的问题仍然没有答案。首先，Google 的机制没有提供一种简单的方法来用第三方开源产品替代封闭源代码的供应商驱动程序。当然，您可以手动替换目标设备供应商分区中的驱动程序包，前提是您拥有 root 访问权限并且以某种方式获得了必要的文件，但是这对于许多人来说并不是一个理想的解决方案。

接下来是稳定性与增强性的矛盾。Android 在加载图形驱动程序时保持了层次化的安排。虽然人们可以在每个应用程序的基础上使用开发者选项 = > 图形驱动程序首选项在工厂安装的版本和较新的更新版本之间切换，但最终归结为 OEM 决定何时推送图形驱动程序更新。因此，最终用户无法直接利用来自 SoC 制造商的最新驱动程序构建，因为原始设备制造商仍然负责推出更新。

如前所述，使用根访问可以更新 GPU 驱动程序，但是没有什么比非根方法更好的了。感谢 XDA 资深会员章程，我们现在有了一个名为 Adreno Tools 的天才解决方案，它允许在 Android 上即时修改或替换 GPU 驱动程序——也无需 root！开发人员将 Android 移植到任天堂 Switch 的背景无疑在这次冒险中发挥了至关重要的作用，因为整个项目起源于 Skyline ，这是一个开源的、正在进行中的任天堂 Switch Android 仿真器。

同为 Skyline 开发者的 Mark "Pixelylon" 向地方法规提出了运行时驱动器替换的想法，最终实现为 Adreno 工具。作为一个无根库，Adreno Tools 可以帮助任何常规应用程序加载自定义 GPU 驱动程序，处理 BCn 纹理，并重定向文件操作，以便在高通的 Adreno 系列 GPU 上进行进一步的复杂修改。它通过连接到系统库并无缝地换入新的驱动程序来做到这一点。值得注意的是，目前还不支持 Mali GPUs。

由于对驱动的来源没有限制，发烧友也可以通过 Adreno 工具加载第三方驱动。这对于那些想要尝试芜菁的人来说尤其有用——芜菁是高通 Adreno 图形硬件的开源 Vulkan 驱动程序，属于 Freedreno 旗下。例如， AetherSX2 ，Android 的 PlayStation 2 模拟器已经利用 Adreno 工具给终端用户选择芜菁而不是 OEM 驱动程序以获得更好的性能。

GitHub 上的 Adreno 工具

我如何从中受益？

请记住，Adreno 工具项目是作为软件库提供的。这意味着你找不到一个预编译的 APK 文件，它神奇地支持在整个系统中使用外部图形驱动程序堆栈的能力。相反，开发人员需要将该库合并到他们的项目中，以便他们的应用程序可以提供加载更新的图形驱动程序文件或第三方替代文件的功能，而无需 root 访问权限。这正是 Skyline 和 AetherSX2 开发人员现在使用 Adreno 工具的方式。

从最终用户的角度来看，单靠 Adreno 工具并不能解决 Android 的 GPU 驱动实现的所有限制。它只是开发者社区的一个工具，用来绕过 Android 设备上图形驱动程序更新的一些最大障碍——仅此而已。您只能通过应用程序获得它的好处，但全系统手动驱动程序更新仍然需要 root 访问权限。

如果我有根，我能做什么？

如果您是拥有根设备的超级用户，那么您可以在您的设备上执行更新的图形驱动程序版本的系统范围强制安装。为此，您需要从更新的固件包中提取驱动程序 BLOB(二进制大对象)文件，其中包含预打包的软件二进制文件，用于与 GPU 进行交互。您可以使用不同设备的固件，但驱动程序必须对应于目标设备的相同 GPU 系列。

提取所需文件后，您可以将它们打包为 Magisk 模块，并将其安装在目标 smartphone 上，以无系统的方式替换现有的供应商驱动程序。当然，这个过程比听起来要复杂一些。您可能需要注意遗留驱动程序和更新驱动程序之间的接口不兼容问题，这种情况下有必要进行调整。

“填补”就是“楔入(某物)或填满一个空间。”这实际上是开发人员在编写 shim 时所做的事情——他们添加额外的代码，以允许 BLOB 与他们正在使用的平台进行交互。对我们的读者来说幸运的是，开发者章程已经写了一个填补不同 Adreno 驱动程序版本之间的空白，你可以在我们的论坛上找到针对 Adreno 500 和 Adreno 600 系列 GPU 的现成可用的 flash OpenGL&Vulkan 驱动程序。

很迷人，不是吗？但是不要忘记客观性和相关性之间的权衡。虽然用 root 权限安装更新的 GPU 驱动程序有它的好处，但它肯定也有它的缺陷。它不适合初学者，而且乱动设备可能会导致保修失效。您需要一个解锁的引导加载程序，这可能不是在所有情况下都可行。

结论

鉴于过去 Android GPU 驱动程序建模场景的替代方案，Adreno 工具库是一个了不起的进步，尤其是其无根性。我们期待看到这个项目在未来如何成长和成熟。一定要试一试，在评论里告诉我们你的想法！
