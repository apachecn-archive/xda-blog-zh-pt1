Windows 11 上的安卓应用:三个月过去了，没有太大变化

原文：https://www.xda-developers.com/android-apps-windows-11-how-its-going/#0001-01-01

去年 6 月，当微软宣布 Windows 11 时，它展示的一个关键新功能是 Android 应用支持。然而，它并没有在发布时出现，到目前为止，你仍然必须是 Windows 的内部人员才能在 Windows 上运行 Android 应用程序。2021 年 10 月 20 日推出的预告，三个月前的今天，我们来看看进度。

Windows 上 Android 应用的简史

大约七年来，微软一直试图让这一技术为支票手表服务。在 2015 年 1 月的第二次 Windows 10 活动中，该公司宣布了四个“桥梁”，以帮助应用程序填充商店。其中一个是 Astoria 项目，该项目本质上是将 Android 应用程序放在当时被称为 Windows Store 的地方，它们只能在 Windows 10 上运行。

七年来，微软一直试图将 Android 应用程序放在 Windows 上。

Project Astoria 从未在 Windows phones 预览版之外发布，真正的原因还不清楚。一些报道称 Astoria 项目运行得非常好，以至于微软认为没有人会再费心制作 Windows 应用程序了。我曾经听说它不够好，而且也不会好。无论如何，微软表示，对于开发者来说，选择移植 Android 应用还是移植 iOS 应用(Project Islandwood)都太令人困惑了，所以它放弃了前者。

一年后，微软推出了 Linux 的 Windows 子系统，让用户在 Windows 上运行 Linux 命令外壳。几年后，在 WSL 2.0 中，它实际上开始在 Windows 10 中发布 Linux 内核。随着所有 Linux 工作的完成，它又一次开始为 Windows 上的 Android 应用铺平道路。

如上所述，随着 Windows 11 的发布，微软推出了内部称为 Latte 项目的产品，这是其在 Windows 上运行 Android 应用程序的最新举措。这些应用程序将来自亚马逊应用商店，但这也是更广泛的微软商店计划的一部分，让用户从他们想去的任何地方获得应用程序。虽然亚马逊可能会有一个初步的独家，因为它与微软在平台上合作，微软商店对其他供应商开放，甚至谷歌如果它想加入。

它是如何开始的，以及它是如何进行的

最初的预览版出现在测试频道，奇怪的是，它后来出现在开发频道。由于 Dev 频道通常首先获得新特性，我认为很多人认为这可能意味着该特性比实际上进展得更快。但事实并非如此。

事实是，在三个月的时间里，并没有太大的变化。

正在设置

你仍然需要在测试版或开发版上获得 Android 的 Windows 子系统，你可以随时注册并开始使用。请记住两个 Windows Insider 渠道之间的差异。Dev 频道会把你放在 rs_prerelease 分支中，所以你会在某个未来版本的 Windows 上，如果不进行工厂重置，你就不能轻松地回滚。测试频道只会给你累积更新，直到开始测试 Windows 11 版本 22H2。

一旦您重新启动电脑以注册 Windows Insider 计划，您应该可以立即在 Microsoft Store 中找到 Amazon Appstore。你可以选择安装它，或者如果你搜索来自亚马逊应用商店的任何东西，它也会自动安装。

一旦你开始这个过程，它将从安装 Android 的 Windows 子系统开始，因此，与你通常从应用程序商店获得的典型的一次点击安装过程不同，你必须点击几次下一步。完成后，你必须打开亚马逊应用商店，用你的亚马逊账户登录。

现在，你可以通过亚马逊 Appstore 应用程序搜索来找到新的应用程序，也可以在微软商店中找到它们，它会将你重定向到亚马逊的店面来安装它们。

需要明确的是，这个过程没有任何改变。我也不指望它会。不是一个特别痛苦的过程，或者至少不是一种无意的痛苦。强迫人们离开微软商店进入其他店面是故意的。微软已经在游戏上这么做了，让用户进入另一个自己的应用程序 Xbox。

应用程序选择

当预览版推出时，亚马逊应用商店提供了 50 款游戏和应用的精选。这有点出乎意料。我想我们都期望整个亚马逊应用商店都适用于 Windows 11。毕竟，这是亚马逊向数亿新用户(或者数十亿，如果 Windows 11 有朝一日实现的话)展示其 Android 应用市场的机会。

微软承诺，这只是一次测试，随着时间的推移，它将增加更多的应用程序和游戏。虽然它没有明确这样说，但言下之意肯定是我们最终会看到整个亚马逊应用商店。

截至今天，Windows 11 上的亚马逊应用商店中有 46 款应用和游戏可用。没错；不知何故少了一些。

Windows 11 上的亚马逊应用商店今天的应用比预览版推出时的 50 个要少。

大多数应用程序和游戏都是我从来没用过的，所以我不能明确告诉你哪些被移除了。我只知道地铁冲浪者是失踪人员之一。你没疯。app 选择部门一直没有进展。看来我们已经倒退了。

如果你进入所有应用和游戏屏幕，你会看到的前六个选项来自于我的对话汤姆系列。对于应用程序，你会发现雅虎邮件、BBC 声音和亚马逊第一方应用程序，如 Kindle 和 Comixology。没有像 Prime Video 和 Amazon Music 这样的应用，因为这些已经在 Windows 上存在了。

旁注:我具体说的是通过亚马逊应用商店提供的应用，这是目前在 Windows 11 上获得 Android 应用的官方途径。这不是获得应用程序的唯一途径。所以如果你想疯狂，让我们疯狂吧。

性能和缺陷

这是我觉得 Android 的 Windows 子系统在 Windows 11 上真正改进的地方。因为坦率地说，它在发布时非常糟糕。是这样的。每当你启动一个 Android 应用程序，不管是亚马逊应用商店还是从那里获得的应用程序，你都必须坐着等待 Android 的 Windows 子系统启动。

这实际上是一个错误。安装 WSA 后，您会在开始菜单中找到一个选项，用于 Android 设置的 Windows 子系统。在那里，您可以选择是否希望 WSA 按需启动，或者是否应该连续运行。这种选择一直存在，但似乎从未奏效。现在好多了。

我总是将它设置为连续，尽管默认情况下它被设置为“按需”。但在 continuous 上，打开一个 Android 应用程序很像打开任何其他类型的应用程序，这才是真正的目标。Windows 上的任何应用程序都应该和其他应用程序一样。用户不必决定在 Windows 上运行哪种平台的应用程序。对于最终用户来说，它们应该只是应用程序。

性能仍有一定的发展空间。在使用一个 app 的时候，你可以立刻感觉到它并不像在 Android 设备上原生运行的那样流畅。

需要澄清的是，我一直在各种硬件上测试 Android 应用，使用英特尔、AMD 和高通的芯片。如果它有一个可转换的铰链，并且是最近一两年制造的，我可能至少会在上面运行 Kindle 应用程序。英特尔和 AMD 处理器使用英特尔桥技术来模拟应用程序，它们在 ARM 上本机运行。尽管如此，性能并不存在。

这可能与为什么地铁冲浪者消失有关。有了延迟，就太难玩了。

结论:我们真的需要 Windows 11 上的安卓应用吗？

围绕这一点，一种相当普遍的情绪是，“谁在乎？”毕竟，Windows 几乎拥有我们想要的所有应用程序，对吗？如果没有原生应用，就有 web 界面，反正我们大部分时间都是在 PC 上的浏览器里度过的。

自从 Astoria 项目以来，我一直对 Windows 上的 Android 应用程序感到兴奋，尽管当时更多的是在 Windows phone 上运行 Snapchat。现在，我只是想要更好的平板电脑应用程序，让可变形电脑成为一个合理的价值主张。当我听说这是通过与亚马逊的合作来实现时，我更加兴奋，因为 Kindle 是我最想要的应用程序。Comixology 也在上面，但亚马逊最近让你在 Kindle 应用程序中阅读你的 Comixology 漫画，所以我现在只需要一个。

我最近写了一篇关于联想 ThinkBook Plus Gen 2 的评论，讲述了我如何坐在机场等待航班，通过在 Windows 11 上运行 Android 应用程序，从笔记本电脑盖子上的 E Ink 显示屏上阅读漫画。很酷。更重要的是，我可以少带一个设备。如果我在火车上通勤，我总是带着 iPad 或 Kindle 阅读。有了像三星 Galaxy Book Pro 360 这样的东西，我就可以把盖子折起来，在我的临时平板电脑上阅读。

目前，没有迹象表明 Android 的 Windows 子系统比预览版到来时更接近公开发布。

与我交谈过的其他人提到了智能家居应用，这是我不太感兴趣的领域。许多智能家电在 Android 上有配套应用，但在 Windows 上没有。如果你想在晚上不在家的时候开灯，或者调节恒温器，你必须用手机来完成。这里的目标是而不是拿出你的手机；是用你已经在用的设备。就连公司刚发给我的联想智能无线耳塞也需要配套的 app 来更新固件。你甚至不能在它自己的电脑上做到这一点。

虽然我认为这个想法很有潜力，但在我们看到它投入生产之前，还有很长的路要走。就像我说过几次的，在过去三个月的测试中没有太大的变化。在亚马逊应用商店里，我们有更少的应用可供选择，而我们应该有更多的。性能是实现这一目标的一个关键因素，但它还没有走得很远。只是现在问题少了。

目前还不清楚微软何时会开始在 Windows 11 的生产版本中支持 Android 应用程序，但这可能会成为定于今年秋天进行的下一次大更新的关键功能。我们已经联系了微软，要求其就 Android 的 Windows 子系统的状态发表声明，一位发言人表示，该公司目前没有什么可以分享的。
