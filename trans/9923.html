Chrome 100 Beta 标志着用户代理字符串终结的开始

原文：https://www.xda-developers.com/chrome-100-beta/#0001-01-01

谷歌 Chrome 和 Mozilla Firefox 都在快速接近版本 100，这有可能破坏一些错误检查浏览器版本的网站。两种浏览器都在研究可能的解决方案，现在 Chrome 100 已经进入测试版，对版本报告和其他新功能进行了更改。

谷歌在一篇 Chromium 博文中写道，“Chromium 100 将是最后一个默认支持未缩减用户代理字符串的版本(以及相关的 navigator.userAgent、navigator.appVersion 和 navigator.platform DOM APIs)。允许网站测试完全缩减的用户代理的 origin 试验将于 2022 年 4 月 19 日结束。在该日期之后，用户代理字符串将逐渐减少。”

几十年来，用户代理字符串一直是 web 浏览器的核心组件，允许站点根据浏览器、浏览器版本、CPU 体系结构和其他数据来改变它们的行为。浏览器已经慢慢开始限制用户代理字符串中可用的信息(例如，所有 iPhone 上的用户代理字符串都不包括 iPhone 型号)，因为它们可以很容易地用于指纹识别。谷歌建议使用更新的用户代理客户端提示 API 代替，比用户代理字符串更安全，更不容易破坏网站。

Chrome Beta 100 还包括一些其他新功能。桌面平台上有一个新的多屏幕窗口放置 API，它允许网站检测连接到您的计算机的显示器，并将不同的窗口放置在特定的屏幕上。也有数字商品 API 和一些新的 JavaScript 特性的原始试用版(基本上是 web 特性的可选测试版)。

你可以在桌面平台上试用 Chrome Beta 版，方法是从谷歌网站下载。Chrome Beta 版也可以通过谷歌 Play 商店在 Android 设备上下载，链接如下。在所有平台上，Chrome Beta 可以与其他版本的 Chrome 一起运行，而不会破坏任何东西。
