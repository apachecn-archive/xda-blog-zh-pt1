Windows 11 build 22567 使 Windows Update 更具可持续性

原文：https://www.xda-developers.com/windows-11-build-22567-windows-update/#0001-01-01

今天是周三，这意味着微软是时候在 Dev 频道为 Windows 内部人士发布另一个版本的 Windows 11 了。本周，我们将获得 Windows 11 build 22567，它带来了一些改进，尽管可以说没有我们过去几周看到的那么大。

更值得注意的变化之一是，微软通过利用清洁能源，使 Windows Update 更加环保。如果你所在的地区有已知的碳强度数据(特别是来自 electricityMap 和 WattTime 的数据)，你的电脑只会在有更多清洁能源可用的时候尝试检查更新。这是一小步，但看到微软这样做还是很有趣的。当然，如果您不想等待，您可以随时手动检查更新。

还是在设置应用中，微软正在对最近向所有 Windows 11 用户推出的你的微软账户页面进行一些更改。现在，您将看到有关当前 Microsoft 365 订阅的更多信息(如果您有)，包括查看和更新您的支付方式、支付的价格以及与谁共享订阅(如果是家庭订阅)。免费用户还会看到 Office 应用程序网页版的链接。

Windows 11 build 22567 中的一个全新功能叫做智能应用控制。这是一项安全功能，旨在保护您的电脑免受潜在的危险和未知的应用程序。当您第一次启用它时，SAC 会尝试了解您的使用模式，以确定它是否有用，而不会过于频繁地碍事。如果可以，它将保持启用，否则，它将自动关闭。如果需要，您可以再次启用它。此功能仅在您执行此版本的全新安装时可用，因此如果您正常升级，将无法获得它。

这种建筑有一些小的改进。我们上周报道的新应用程序选择器现在已经可以使用了，不需要任何修改，但它正在逐步推出。在 build 22557 中引入的新任务管理器现在也可以在 Dev 频道中为每个人所用。微软还改进了一些触摸手势的动画，以便它们更平滑地跟随你的手指，例如当你最小化一个应用程序时。

除此之外，大部分都是些小东西。如果您使用其他手机应用程序，您现在可以在 Windows 11 OOBE 期间链接您的手机。当你在开始菜单中移动一个图标时，你会看到一个小动画，提示你这样做会创建一个文件夹。

除了新功能和改进，Windows 11 build 22567 还包括一系列常见的修复，这应该会带来更好的整体体验。

Windows 11 内部版本 22567 中的修复

【常规】

  • 修正了用户听不到启动声音的问题。

【任务栏】

  • 使用任务栏将一个已经运行的应用程序放到前台应该更可靠了。
  • 修复了一个问题，如果你改变时区，日历弹出按钮不会更新，以反映日期的变化。
  • 使用讲述人键加回车键或讲述人触摸手势现在应该可以激活任务栏中的项目。
  • 缓解了导致 windows 在关闭其中一个任务栏弹出按钮后能够在任务栏顶部绘图的问题。

【平板电脑优化任务栏】

注意:这些修复将仅在您的设备上启用平板电脑优化任务栏的情况下显示，该任务栏开始向 Windows Insiders 推出，但尚未向所有人提供。

  • 任务栏设置已更改为“当此设备用作平板电脑时，为触摸交互优化任务栏”。
  • 我们改进了鼠标点击系统托盘图标的可靠性。

【开始菜单】

  • 修正了一个影响开始菜单打开可靠性的问题。
  • 修复了在固定应用程序页面之间拖动固定应用程序图标不起作用的问题。
  • 解决了更多按钮的对齐问题。

【文件浏览器】

  • 点击中的按钮，新的 OneDrive 弹出按钮现在应该将这些对象带入前景焦点。
  • 星级支持的文件(例如 mp4 文件)现在应该再次工作。
  • 修正了有时打开上下文菜单时可能导致 explorer.exe 崩溃的问题。
  • 改进了在有很多文件的情况下显示回收站内容的性能。
  • 解决了试图从文件资源管理器开始搜索时发生的影响搜索能力的底层崩溃问题。
  • 修正了一个小屏幕用户的罕见问题，文件浏览器中的上下文菜单会向下动画显示，消失，然后立即向上动画显示。

【输入】

  • 改进了使用 WIN + Space 时的输入切换响应。
  • 修正了一个问题，导致一些内部人员在输入切换器中看到黑色背景上的黑色文本。
  • 解决了一个问题，即日本 IME 工具栏可能会变得透明，IME 候选窗口剪辑或根本不显示。
  • 如果您使用带有垂直候选列表的日文 IME，文本现在应该会以正确的方向显示。
  • 更新了当焦点设置为密码字段时用于隐藏触摸键盘中按键视觉效果的图标，以帮助更容易理解。
  • 修正了新的“按退格键”命令在日语语音输入时不工作的问题。
  • 缓解了在插入表情符号并关闭表情符号面板后导致讲述人焦点停留在错误位置的问题。

【设置】

  • 应用列表中应用>已安装应用下的项目不应再相互重叠。
  • 修正了在设置> Windows 更新中使用触摸无法选择“立即重启”按钮的问题。

【开窗】

  • 在显示器之间拖动窗口时，exe 不会再崩溃。
  • 如果您悬停在任务视图中的桌面缩略图上，工具提示现在应该与您悬停的图标相匹配。
  • 修复了一个问题，即当你已经在该区域捕捉到一个窗口时，捕捉助手可能会意外触发。
  • 提高了在屏幕顶部调用快照布局的可靠性。
  • 如果您在任务栏的搜索弹出按钮打开时调用任务视图，而不是调用并立即关闭，任务视图现在应该会成功启动。

【语音接入】

  • 修复了一个问题，导致 Outlook 滚动到您的电子邮件的开头，如果您启用语音访问并开始交谈。
  • 当你改变文本大小时，语音访问中的按钮文本将不再被截断。
  • 修复了阻止在 ARM64 设备上启动语音访问的问题。
  • 除非语音访问设置也已打开，否则在您登录后，语音访问将不再打开。

【直播字幕】

  • 直播字幕现在可以让你更流畅地切换音频设备。
  • 改变直播字幕的大小将不再导致黑暗模式下的白色闪烁。
  • 您现在可以搜索实时字幕，在设置中找到它们的页面。
  • “设置”中的“辅助功能主页”现在会在标题下面列出“实时标题”。
  • 设置弹出按钮现在可以在实时字幕窗口中被淡入淡出。

【旁白】

  • 修复了使用讲述人时导致高 CPU 使用率的问题。
  • 讲述人现在将读出主题以及在设置 > 个性化 > 文本输入 > 主题中的选择状态。
  • 当一个对话框弹出时，讲述人现在会一致地宣布它正在从一个对话框中读取消息。
  • 当使用 Microsoft Edge 的历史页面时，讲述人现在将正确地宣布页面名称和聚焦元素。
  • 修复了在某些应用程序中影响 webview2 中讲述人键盘导航的潜在问题。
  • 更新了设置中的盲文输入和输出表格类型下拉列表，现在有了默认值。
  • 修复了一个问题，导致叙述者挂起时，使用诺顿电源橡皮擦。
  • 如果您在 Excel 中查看 > 检查可访问性，讲述人现在将正确阅读错误、警告、提示&智能服务部分。

【任务管理器】

  • 暂停和效率模式图标不再会在高 dpi 的任务管理器中被剪切。
  • 修正了列首选项没有被保留的问题。
  • 默认的起始页和总是在顶部的设置现在应该可以工作了。
  • 当“…”菜单下没有显示任何内容时，它现在不会显示。
  • 减轻了试图启动任务管理器时发生的崩溃。
  • 修正了一些子进程中效率模式图标丢失的问题。
  • 修正了一些设置选项没有被保留的问题。
  • 修复了在高 DPI 设置的显示器上显示暂停和效率模式图标时被剪切的问题。

【锁屏登录】

  • 修复了导致锁定屏幕上的辅助弹出按钮闪烁的问题。
  • 锁定屏幕位置和间距上的可访问性弹出按钮现在应该居中，并与该屏幕上其他选项的设计保持一致。
  • 修复了锁定屏幕上辅助弹出按钮的边框，以在高对比度模式下正确显示。
  • 解决了使用安全密钥登录时消息文本中的编码问题。
  • 缓解了一些内部人员登录电脑时看到的 explorer.exe 崩溃。

【其他】

  • 通知中心现在会在您启动焦点会话时关闭，并且焦点计时器会打开，而不是您必须关闭它。
  • 修复了当动画被禁用时影响快速设置和通知中心打开能力的问题。
  • 如果您在尝试关闭快速设置和通知中心之前使用了手势来调用它们，那么现在应该可以更可靠地关闭它们。
  • 修正了导致某些应用崩溃的 uiadisconnecprovider API 的问题。
  • 减轻了一个 bug 检查，在过去的几个航班中，内部人员连接了 8 个以上的监视器。
  • 如果你已经将一个应用程序固定在 Windows 沙盒的任务栏上，点击该应用程序来启动它，现在实际上可以工作了。

当然，如果没有一些已知的问题，它就不是一个 Windows Insider 版本。在安装构建版本之前检查它们总是一个好主意，以防它们影响您使用 PC 的能力。

Windows 11 内部版本 22567 中的已知问题

【常规】

  • 运行 Windows 10 的用户尝试在 Dev 通道中直接升级到 build 22563 或更高版本可能会遇到安装失败，错误代码为 0x8007007f。要绕过，请加入测试版频道，在那里安装提供的 Windows 11 build，然后切换到 Dev 频道接收当前的 Dev 频道更新。此问题已被理解，并将在即将到来的版本中得到解决。
  • 在企业版上体验设备设置(OOBE)时，第一次尝试时会跳过网络添加屏幕。作为一种解决方法，当用户看到“命名您的计算机”选项时，请重新启动并重新启动 OOBE。现在，网络添加屏幕将按预期出现。
  • 我们正在调查来自内部人士的报告，由于 explorer.exe 多次崩溃，Windows 混合现实从上一次航班开始就不工作了。
  • 滚动条在某些 Win32 应用程序上无法正常呈现。只有当用户仍然能够按预期点击/拖动滚动条时，这个错误才是可见的。

【任务栏】

  • 如果你右击这个版本中的开始图标或者按 WIN + X，explorer.exe 很可能会崩溃。如果使用此菜单启动任务管理器，请暂时使用 CTRL + Shift + Esc。

【平板电脑优化任务栏】

  • 在 2 合 1 设备上启动应用程序或点击扩展的任务栏外部后，任务栏并不总是自动折叠。
  • 操作系统的某些区域尚未跟踪 2 合 1 设备上扩展任务栏的高度，因此您可能会看到重叠的组件，例如与任务栏重叠的小工具。

【文件浏览器】

  • 在文件资源管理器的搜索框中输入搜索词时，可能无法打开显示的建议结果。
  • 我们正在修复显示 OneDrive 存储的弹出菜单中有关图标大小、视觉错误和文本剪辑的问题。

【微件】

  • 有时，当从提要中锁定时，锁定的小部件被放在顶部，而不是其他锁定的小部件的下面。如果发生这种情况，这将在 30 分钟内自动更正，将最近锁定的小工具移动到预期的默认位置。或者你可以退出你的插件板，然后立即重新登录，应该可以解决这个问题。
  • 在 widgets 面板中重新排列 widgets 后，一些用户会遇到固定部分中的 widgets 呈现不正确的问题。如果发生这种情况，退出你的插件板并立即重新登录应该可以解决问题。

【聚焦】

  • 任务栏图标和工具提示可能与焦点状态不匹配。
  • 在应用程序中配置焦点会话时，时钟应用程序尚未更新 Windows 焦点状态。这将在未来的应用程序更新中解决。

【直播字幕】

  • 全屏模式下的某些应用程序(如视频播放器)会阻止显示实时字幕。
  • 位于屏幕顶部附近并在直播字幕打开前关闭的某些应用程序将在位于顶部的直播字幕窗口后重新启动。当应用程序有焦点时，使用系统菜单(ALT +空格键)将应用程序的窗口进一步下移。
  • 当实时字幕位于顶部时，触摸无法到达最大化应用的最顶部(例如标题栏窗口管理按钮)。

【任务管理器】

  • 当任务管理器配置为黑暗模式时，一些工具提示不能准确显示首选视觉设置。

像往常一样，如果你注册了 Windows Insider 计划的开发频道，你可以通过简单地检查更新来获得这个版本。如果你更愿意使用 ISO 来安装它，微软说它将很快为内部人员提供新的 ISO。
