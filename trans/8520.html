Visual Studio 2022 现在通常只提供 64 位版本

原文：https://www.xda-developers.com/visual-studio-2022-dotnet-6-generally-available/#0001-01-01

微软已经宣布 Visual Studio 2022 和。NET 6 今天已经全面上市。这些更新的工具为开发人员带来了新的功能，旨在加快开发过程，并创建更好的应用程序。

首先， Visual Studio 2022 是继 Visual Studio 2019 之后，两年多来对 Visual Studio 的首次重大更新。最新版本有一个非常重要的变化: Visual Studio 本身现在是一个 64 位应用程序。这对开发人员来说是一个大消息，因为 VIsual Studio 现在可以访问计算机的所有内存，这使得编译和处理大型复杂项目变得更快。事实上，Visual Studio 正在放弃对 32 位机器的支持，但想必没有多少开发人员再使用 32 位 PC 了。

除此之外，Visual Studio 还附带了大量新闻，包括 C++的新版本。NET 6 和 C#10。编辑器也有改进，比如新的多插入符号复制和粘贴体验，新的 IntelliCode 特性，比如 C#项目中的整行完成，等等。值得注意的一点是，Visual Studio 2022 中的扩展平台有一些突破性的变化，因此，除非更新，否则 VS2019 的现有扩展将无法与新版本配合使用。在视觉方面，微软已经改变了应用程序中的图标,现在应用程序中使用了新的卡斯卡迪亚代码字体。你可以阅读发行说明来获得对所有新特性的更全面的描述。

至于。NET 6 ，新闻的列表也是广泛的。性能得到了全面的显著提高，尤其是在文件 I/O 方面。API 得到了更新或引入，包括通过 JSON APIs 和 ASP.NET 核心中新的最小 API 提高了性能，使入门更容易，并提高了 HTTP 服务的性能。这个版本还在 ASP.NET 核心中增加了对 HTTP/3 的支持，以及增强安全性的 OpenSSL 3。

一个。NET 6 推出时没有的功能是多平台应用程序 UI (MAUI)，它允许开发人员为一个在桌面和移动平台上都看起来很现代的应用程序编写代码。然而，微软没有说这一功能何时可用。

。NET 6 是该平台最新的长期支持版本，这意味着它将在未来三年内得到支持，它是所有类型应用程序的统一平台，无论是移动、桌面、云、web 等。它取代了两者。NET 5 和。NET Core 3.1，微软正在敦促开发人员将他们的应用程序从这些旧平台上迁移出来。然而，。NET 6 只在 Windows 和 Mac 的 Visual Studio 2022 中受支持，所以如果你使用的是旧版本，你将无法使用新平台。
