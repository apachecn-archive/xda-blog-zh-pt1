Windows 10 版本 21H2 在打印服务器方面有一个已知问题

原文：https://www.xda-developers.com/windows-10-version-21h2-known-issues/#0001-01-01

微软今天推出了 Windows 10 版本 21 H2,它带有一些专门针对商业用户的功能。这包括支持新的 Wi-Fi 安全标准、新的 Windows Hello 功能等等。然而，与大多数 Windows 10 版本一样，21H2 版本也有其已知问题。

在这种情况下，有一个已知的问题，它与打印服务器有关。事实上，这不是一个会影响大多数用户的问题，但微软指出，一些 Windows 客户端在连接到通过打印服务器共享的打印机时可能会遇到错误。以下是微软对这个问题的描述:

当连接到 Windows 打印服务器上共享的远程打印机时，Windows 打印客户端可能会遇到以下错误:

  • 0x 000006 E4(RPC _ S _ CANNOT _ SUPPORT)
  • 0x0000007c(错误无效级别)
  • 0x00000709(错误无效打印机名称)

虽然可能会遇到这种错误，但微软表示已经缓解了这种情况。您可以解决这个问题，但是在安装打印服务器之前，您的打印客户端需要在 2021 年 1 月之后安装 Windows update。如果满足此要求，则可以设置网络安全设置和 VPN，以允许通过以下端口进行 RPC over TCP 连接:

  • 默认起始端口:49152
  • 默认结束端口:65535
  • 端口范围:16384 个端口

另一个可能有助于缓解这个问题的步骤是为打印作业启用客户端渲染。可以在打印服务器上启用此设置，但是如果打印客户端有阻止此设置的覆盖，则此设置可能不起作用。

虽然这种缓解使您可以解决这个问题，但微软表示，它正在制定一个更明确的修复方案。此修复将允许打印客户端通过 SMB 而不是 TCP 建立到服务器的 RPC 连接。

同样值得注意的是，这个问题影响了之前版本的 Windows 10，以及 Windows 11。目前还没有 Windows 10 版本 21H2 独有的已知问题，考虑到与以前的版本相比变化很小，这应该不会太令人惊讶。如果你想安装最新的 Windows 10 功能更新，请查看我们的指南如何立即获得 Windows 10 版本 21H2】。
