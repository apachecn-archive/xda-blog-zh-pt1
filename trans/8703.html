Windows 11 通过 build 22000.348 获得了对坏应用的另一个修复

原文：https://www.xda-developers.com/windows-11-build-22000-348/#0001-01-01

在本周结束之前，微软为注册了运行 Windows 11 的测试版和发布预览版频道的 Windows 内部人员推出了新的更新。此次更新使 Windows 11 构建了 22000.348，包括对可能导致某些应用停止工作的问题的修复。

具体来说，发现在最近的更新后，Microsoft Installer (MSI)出现了一个问题，这可能会导致应用程序在尝试修复或更新应用程序后停止工作。在其他潜在的受害者中，卡巴斯基应用程序受到了这个问题的影响。本周早些时候，微软将这个问题添加到了 Windows 11 的已知问题列表中，所以很高兴看到修复程序这么快就推出了。

当然，它现在只对内部人员开放，但我们很可能会在月底之前看到这些变化被打包成一个可选的累积更新。这一版将对所有人开放，它还将包括上周向内部人士推出的大量修复。此次更新还包括新的流畅表情符号，该符号在夏季发布后，已经在开发频道测试了几周。

在此之后，所有这些更新也应该捆绑到 12 月的强制更新中，这将是我们进入假期季节的最后一次更新。

微软最近还承认了 Windows 11 中与英特尔智能声音技术驱动程序有关的另一个问题，这可能会导致蓝屏死机。如果您遇到此问题，您需要安装英特尔 SST 驱动程序版本 10.30.00.5714 或 10.29.00.5714 及更高版本(10.30.x 和 10.29.x 等同于此修复程序)。此驱动程序需要由您的计算机的 OEM 提供，如果您尚未升级到 Windows 11，您的升级将被搁置，直到您拥有最新的驱动程序。

如果你还是 Windows 10 的内部人士，微软昨天也发布了发布预览频道的更新。它修复了一个问题，即登录 Azure Active Directory 的支持 Windows Hello 的设备可能无法访问内部资源。
