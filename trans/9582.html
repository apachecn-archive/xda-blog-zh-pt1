GitHub 已经成为软件开发者的乐园

原文：https://www.xda-developers.com/github-sponsor-repositories/#0001-01-01

GitHub 成为开发者的 Patreon，拥有新的赞助商专用库

GitHub 正在为软件开发人员提供创建只有赞助商才能访问的代码库的能力。

微软拥有的 GitHub 是现存的最受欢迎的软件仓库之一，为无数的软件项目提供动力，并为许多产品托管关键的基础设施。GitHub 一直在慢慢鼓励人们从经济上支持他们喜欢的软件项目，现在该公司正在为希望增加项目捐赠的开发者推出另一项功能。

GitHub 已经允许人们在他们的项目中添加捐赠链接，或者开发者可以注册该公司自己的“GitHub 赞助商”功能，这将 GitHub 本身变成了支付服务(就像 Patreon、Ko-Fi 和其他类似的平台一样)。人们可以设定资助目标，项目的资助者将获得一枚特殊的徽章。

GitHub 在周三的一篇博客文章中写道，“启用 GitHub 赞助商的开发者和组织现在可以在他们的每一个赞助商层上附加一个私有库。这将授权各自的资助者访问存储库。这些邀请是由 GitHub 自动管理的，所以设置好之后就没什么可管理的了。这使得很多人手动操作 GitHub 赞助商支持的部分。我们很期待看到您将如何使用它！”

这些年来，将某些内容限制在付费用户范围内一直是 Patreon 等服务的主要吸引力之一，现在 GitHub 也提供类似的功能——仅用于软件开发。GitHub 表示，赞助商专用库可以用于早期访问或 beta 测试，或者只是在一个集中的位置与赞助商交流(GitHub 也有一个类似论坛的讨论功能)。

开源软件项目通常很难获得足够的资金，所以很高兴看到 GitHub 鼓励更多人支持他们喜欢的项目。如果不想使用 GitHub 的平台，软件开发者/存储库所有者也可以保留他们与 Patreon、PayPal 或任何其他支付提供商的链接。
