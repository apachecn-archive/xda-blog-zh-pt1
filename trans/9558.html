通用 Android 解包器更新了新的膨胀软件列表

原文：https://www.xda-developers.com/universal-android-debloater-gui-update-new-bloatware-list/#0001-01-01

Android 上膨胀软件的故事始于运营商将各种无用的应用捆绑到他们网络下销售的智能手机上。从那时起，它迅速扩展到 OEM 厂商自己，推出各种功能重复但质量低劣的应用程序，甚至来自脸书和微软的带有出厂固件的应用程序。虽然有些人可能会发现这些预装的应用程序很有用，但由于不必要的紧密系统集成，缺乏轻松删除它们的选项可能会让那些不喜欢的人感到沮丧。

不过，还是有办法绕过这些限制的。我们有一个精心制作的指南，帮助你从你的 Android 设备上“卸载”任何预装的系统应用，这也不需要 root 访问。以这种方式删除几个臃肿的软件包通常没有问题，但当你不知道删除哪个应用程序安全与否时，事情就变得乏味了。为了填补这一空白，XDA 成员 w1nst0n_fr 提出了Universal Android de bloater项目——一个 shell 脚本，它依赖于社区维护的 OEM 和运营商特定的膨胀软件列表来实现无缝解包。几个月前，开发人员在 Rust 中重写了整个内容，将其转变为一个成熟的跨平台 GUI 工具。现在，最新发布的通用 Android 解包器 GUI 向解包列表添加了许多新的包，带来了多设备支持、可移植性的持久设置以及其他一些改进。

在撰写本文时，最新构建的版本号是 0.4.1，这是 0.4 主版本的修补程序。0.4 更新中的主要改进之一是并行 ADB 命令的异步执行，这极大地增强了 UI 响应能力。“推荐的”去模糊列表也被修改了，因此它不太可能删除你想保留的东西。解禁列表现在包含了几个新的包，分别对应于 Fairphone、高通、Unihertz、小米和英国运营商 EE。最后，对于 Arch Linux 用户来说，通用 Android 解包工具现在可以在 AUR 以universal-android-debloater-bin(二进制)和universal-android-debloater(源代码)的形式获得。

通用 Android 解包器 GUI 0.4(和 0.4.1)的完整 changelog 可以在这里找到。因为这是一个开源项目，你可以关注它的发展，并在 GitHub 上为膨胀软件数据库做出贡献。尝试一下，让我们知道它在您的 Android 设备上运行得如何。
