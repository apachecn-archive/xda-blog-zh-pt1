Android 13 DP1 允许谷歌 Pixel 6 作为虚拟机运行成熟的 Windows 11

Android 13 DP1 解锁了谷歌 Pixel 6 和 6 Pro 上的全部 KVM 功能。现在，您可以在这些手机上启动 Windows 11 和 Linux 虚拟机。

如果你熟悉 Windows 11 ，你几乎肯定听说过安卓的 Windows 子系统。微软在用于 Linux 的 Windows 子系统的基础上构建了其 Android 兼容层，该子系统基于该公司的 Hyper-V 虚拟化平台。谷歌在 Chrome OS 中设计 Linux 应用功能时采用了类似的方法，但他们依赖 Linux 内核的基于内核的虚拟机(KVM)作为虚拟机管理程序。由于 Android 是建立在 Linux 内核之上的，所以可能会出现这样的问题:在任何 Android 设备上，利用 KVM 来运行一个带有另一个操作系统的虚拟机在技术上是否可行？好吧，如果你有一台谷歌 Pixel 6/6 Pro，并且你愿意 flash Android 13 的首个开发者预览版 build，你肯定可以实现这一壮举。

Android 13 中对于谷歌 Pixel 6 有什么秘制酱料？据 XDA 资深会员 kdrag0n 透露，在 Tensor 平台的 Android 13 bootloader 和固件中，谷歌增加了向内核暴露异常级别 2 hypervisor 特权级别的能力，用于其受保护的 KVM 特性(pKVM)。虽然 pKVM 是可选的，并且是基于每个虚拟机启用的，但是人们可以很容易地在不受保护的虚拟机上利用完整的 KVM 功能。这就是为什么开发人员可以引导多个 Linux 发行版并以接近本地速度运行它们的原因。谷歌曾计划在 Android 13 中发布 pKVM，尽管这只是 Android 下一个主要版本的第一个预览版，但目前的实现看起来相当稳定。

请记住，在这个阶段，Android 13 上的直接 KVM 访问需要 root 访问。话虽如此，通过编写虚拟机管理应用程序和请求特殊许可在微程序虚拟机中运行本地服务，非根实现是可能的。有很多这样的虚拟化监视器，像谷歌自己的 crosvm 、 QEMU 或鞭炮，所以他们的 Android 移植应该填补空白。想象一下在你的 Android 智能手机上运行 Windows 11，而不会干扰底层分区，这要感谢 Android 的原生虚拟化支持。
