谷歌的目标是让大屏幕设备的应用程序开发更容易

原文：https://www.xda-developers.com/google-android-12l-app-development-large-screen-devices/#0001-01-01

众所周知，Android 在大屏幕设备上的体验远非完美。那些在过去几年中使用过安卓平板电脑的人会同意，安卓在大显示屏上的扩展性不好。UI 元素没有充分利用所有额外的屏幕空间，大多数应用程序都没有针对外形进行优化。谷歌的目标是用 Android 12L 改变这种状况。为了帮助开发者为更新做好准备，该公司分享了新的工具和指南。

在今天的 Android 开发峰会上，谷歌宣布了新的材料设计指南，以帮助开发者设计在任何屏幕尺寸上都能很好扩展的自适应应用。该指南涵盖了生态系统中普遍存在的常见布局模式，以激励开发人员并帮助他们优化他们的应用程序，使其在小屏幕和大屏幕上都表现良好。此外，指南还概述了开发人员如何在各种屏幕尺寸上提供最佳的导航体验。

在一篇博客文章中，谷歌提到:“推荐的导航模式包括在紧凑型屏幕上使用导航条，在中等宽度和更大尺寸(600dp+)的设备上使用导航条。对于扩展宽度的设备，我们新发布的材料设计指南中有几个关于更大屏幕布局的想法，例如可以使用 SlidingPaneLayout 实现的列表/细节结构。”

它进一步补充道，“虽然更新导航模式和使用 SlidingPaneLayout 是将大屏幕优化布局应用于包含片段的现有应用程序的好方法，但我们知道你们中的许多人都有基于多个活动的应用程序。对于这些应用程序，Jetpack WindowManager 1.0 beta 03 中发布的新活动嵌入 API 可以轻松支持新的 UI 范式，如双窗格视图。”

谷歌还敦促开发者使用 Jetpack Compose 工具包为他们的应用程序构建 UI。该公司解释说，使用 Jetpack Compose 可以帮助开发人员轻松优化 UI，因为所有 UI 都是用代码描述的。这使得在运行时很容易将其调整到可用的屏幕大小。如果你对使用 Jetpack Compose 感兴趣，你可以在 Compose guide 中查看 Google 的 Build adaptive layouts 来开始。

WindowManager APIs

Google 已经引入了新的 WindowManager APIs 来帮助开发人员构建能够适应任何屏幕尺寸的用户界面。WindowManage 库提供了一种向后兼容的方式来处理应用程序中的窗口，并为所有设备构建响应性 UI。它包括:

活动嵌入

活动嵌入通过同时显示多个活动，帮助开发人员利用大屏幕设备上的额外屏幕空间。使用活动嵌入，开发人员可以通过创建 XML 配置文件或调用 Jetpack WindowManager API 来定义他们的应用程序如何显示活动——并排或堆叠。谷歌声称，随着设备的折叠和展开，活动嵌入可以无缝地工作在可折叠、堆叠和拆分活动上。您可以尝试在 Jetpack WindowManager 1.0 Beta 03 和更高版本中嵌入 API 的活动。

窗口大小等级

新的 WindowSizeClass APIs 定义了一组固执己见的视口断点，以帮助开发人员设计、开发和测试可调整大小的应用程序布局。窗口大小类断点分为三类:紧凑、中等和扩展。这些类别旨在平衡布局的简单性，并提供针对大多数用例优化应用程序的灵活性。Jetpack WindowManager 1.1 中将提供 WindowSizeClass APIs。

让应用程序能够识别文件夹

WindowManager 还为窗口功能(如折叠和铰链)提供了一个公共 API 表面。开发人员可以使用该 API 来构建折叠感知应用程序，这些应用程序可以自动适应以避免折叠和铰链，或者利用它们。查看本指南让你的应用了解折叠。

Android Studio 的变化

参考设备

谷歌在 Android Studio 中引入了四种新的参考设备，以帮助开发者构建适应各种屏幕尺寸的响应性应用。这些参考设备可用于开发人员设计、构建和测试其应用的 UI 和布局的工具。这四种参考设计分别代表手机、大型可折叠内置显示屏、平板电脑和台式机。开发人员也可以使用这些参考设备来确保他们的应用程序跨流行的 WindowSizeClass 断点工作。

布局验证

谷歌正在 Android Studio Chipmunk 中开发一种新工具，帮助开发者识别影响大屏幕设备的潜在问题。可视化林挺工具在布局验证中主动显示用户界面警告和建议，并帮助开发人员识别受影响的参考器件。

可调整大小的仿真器

Android Studio Chipmunk 还带来了一个新的可调整大小的仿真器，可以帮助开发人员在运行时测试 UI 缩放。可调整大小的仿真器允许开发人员在四种新的参考设备之间快速切换，并验证他们应用程序的 UI 布局。要使用新的可调整大小的仿真器，请使用 Android Studio 中的设备管理器创建一个新的虚拟设备，并使用 Android 12L (API level 32)选择可调整大小的设备定义。

大屏幕上的 Google Play

最后，谷歌还宣布了针对平板电脑和 Chromebooks 等大屏幕设备的 Google Play 的一些变化。该公司增加了新的检查，根据新的大屏幕应用质量指南评估每个应用的质量。Google Play 将自动推荐通过大屏幕设备评估的应用程序，并在应用程序未针对大屏幕进行优化时发出警告。此外，谷歌将在 Play Store 上推出大屏幕专用应用评级。这些变化将于明年推出。
