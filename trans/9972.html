定制 AOSP GSI 带来了 Android 12L 的任何项目高音功能的设备

原文：https://www.xda-developers.com/android-12l-custom-gsi/#0001-01-01

经过几个月的全面 beta 测试， Android 12L 现已正式发布，作为 Android 13 之前的临时更新。稳定版目前正在推出，以选择谷歌 Pixel 智能手机，而微软、联想和三星的平板电脑和可折叠手机将在今年晚些时候推出。通过 Android 开源项目，底层代码库也变得可用，为 OEM 和开发人员提供了最终确定的 Android 12L 源代码以供使用和编译。这也意味着定制的只读存储器将会随时出现，事实上，它们已经开始出现了。首款基于 Android 12L 的售后 ROM 现以定制通用系统映像(GSI)的形式提供。

谷歌确实在其网站上提供了自己的 Android 12L GSIs，但他们的 GSI 只是为了让开发者可以根据最新的 API 水平测试他们的应用程序。不幸的是，这意味着谷歌制造的 GSI 在一些设备上相当好用，而在另一些设备上就坏了。这就是社区，尤其是 XDA 公认的开发商 phhusson 介入的地方。开发人员构建的自定义 GSI 发行版(又名“Phh-Treble”)旨在尽可能多的设备上运行，并且错误更少。因此，你应该能够在任何支持 Project Treble 的设备上启动 Android 12L，这意味着大多数设备都使用 Android 8.0+。

phhusson 在三星 Galaxy S9 的 Exynos 变种上运行的定制 Android 12L GSI(截图提供:Telegram 用户 FKTDK)

Phh-Treble GSI 项目带有内置的 OTA 更新程序，但标记为 v410 的介绍性 Android 12L 版本将不会作为 OTA 提供。由于构建的前沿性质，人们必须手动下载并刷新它。

v410 版本的完整变更日志可以在下面找到:

  • 安卓 12.1
    • 警告:gapps 变体的启动程序已损坏，您需要手动安装一个
  • March 安全补丁
  • 修复某些三星设备上的触摸屏
  • 修复 Moto One Fusion+上的音频
  • 禁用 LG Stylo 7 上的指纹手势
  • 禁用 Doogee S88 Pro 上的指纹手势
  • 所有索尼设备上的地图相机/对焦按钮
  • 修复 arm32_binder64 gapps 中的设置向导
  • 在小米 Pad 5 上启用 Widevine L1
  • 在 Redmi 9A 上安装调制解调器
  • 通过禁用 WPA3 来修复它
  • 移除圣诞启动动画
  • 在小米 Civi 上固定 120hz/90hz
  • 修复华硕 Zenfone8 上的显示不足指纹
  • 在更新设置中添加一个选项来卸载 OTA

有兴趣试试这个吗？你首先需要的是一个解锁的引导程序。接下来，你可以从下面链接的 GitHub 页面下载 phhusson 的 GSI 的 Android 12L 版本，并获取适合你设备的版本。请务必进行设备外备份，因为刷新过程将要求您对目标设备进行出厂重置。
