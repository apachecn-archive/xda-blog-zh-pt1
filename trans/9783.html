icon 负责一切—从设备发现到无缝上下文切换。但是如果你是一个命令行爱好者，你可能希望看到幕后，并通过 CLI 控制整个转换程序。这正是 Marco Lucidi 所做的，他发明了一个叫做 ytcast 的便捷工具。

什么是 ytcast？

自称为20 多岁的计算机科学爱好者，在 GitHub 上化名 MarcoLucidi01，他将 ytcast 实用程序编码为 cast 按钮的独立替代品。据开发者称，任何人都可以使用该程序直接从命令行界面复制“在电视上播放”按钮的功能。它是用 Go 编写的，可以在 Windows、Linux 和 macOS 上运行。

ytcast 如何工作

在幕后，ytcast 使用由网飞和谷歌开发的 DIAL(发现和启动)协议来处理设备发现和应用程序启动事件。对于不再使用拨号协议(如 Chromecast)的目标，该程序还提供了手动配对选项，由 YouTube on TV 应用程序的“链接和代码功能提供支持。接下来是屏幕识别部分，开发人员依赖于反向工程的 YouTube Lounge API。最后，它将视频 URL 从源设备传递到目标屏幕。

限制

由于这个项目还处于起步阶段，所以有几个缺点。会话处理有时是偶然的，尤其是由于使用了未记录的 API。开发人员还没有找到一种可靠的方法来通过 ytcast 传递 YouTube 播放列表的 URL，但是有一种解决方法涉及到 youtube-dl 可执行文件。最后但同样重要的是，你可能在设置>链接设备菜单下找不到 ytcast。

不用说，ytcast 项目对高级用户非常有用。这是一个巨大的潜力，我们必须记住，在这一点上，它显然是一个测试质量的软件。一旦它达到稳定的里程碑，它可能是任何对脚本和自动化感兴趣的人的最佳选择。
